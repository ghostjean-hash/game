# html-game 표준 변경 이력

# 1. v0.3.1 (2026-07-05)

1.1. 4.7 규칙 1-4의 무테마 골격을 공용 자산 `shared/mobile-shell.css`로 실제 분리. 색·폰트 없이 뷰포트 고정·safe-area·터치·리셋만 담는다. base.css는 이 파일을 `@import`(다크 테마 게임 무영향), 자체 디자인 게임은 직접 링크.
1.2. 변경 사유: 4.7 규칙은 글로 있었으나 구현이 base.css(다크 테마 한 덩어리)에만 있어, 파스텔 게임(노노그램)이 뷰포트 골격 값을 복붙해 중복. 색과 무관한 골격을 무테마 파일로 떼어 링크 재사용이 가능하게 함.
1.3. 검증: 노노그램 main.css 복붙 구간 제거 + mobile-shell.css 링크 전환. browser-shot으로 노노그램(파스텔 배경·맵 정상) + base.css 링크 게임 러시아워(다크 배경·`@import` 정상) 렌더 무변화 확인, pageerror 0.
1.4. 영향: nonogram v0.3 → v0.3.1(복붙 → 링크 전환). tetris/sudoku/rushhour/허브는 base.css가 mobile-shell을 `@import`하므로 무영향(마이그레이션 불필요).

# 2. v0.3 (2026-07-05)

2.1. 4.7 신설: 모바일/태블릿 브라우저 플레이 규칙 7개(viewport meta / 페이지 고정 / safe-area / 드래그 스크롤 차단 / 방향별 레이아웃 / 전체화면 / 보드 크기 산정).
2.2. 변경 사유: 사용자가 아이패드 크롬에서 노노그램 플레이 시 상단 UI가 브라우저 툴바에 잘리고 드래그 시 게임판이 스크롤되는 문제를 지적. 표준에 모바일 플레이 규칙이 전무해 게임별 재량으로 처리되며 편차가 컸음(base.css 미링크 게임은 규칙 누락). 웹게임 공통 규칙으로 명문화.
2.3. 검증: 노노그램에서 아이패드 프로 12.9"·11" 가로/세로 + 구형 아이패드 + 아이폰 6종 뷰포트로 스크롤 0·격자 중앙·safe-area·전체화면 확인, 테스트 20/20. base.css 검증값을 직접 이식(파스텔 자체 디자인이라 base.css 컴포넌트 통째 링크는 부적합).
2.4. 영향: nonogram v0.2 → v0.3 적용. tetris/sudoku/rushhour는 shared/base.css로 1-4를 이미 부분 준수하나 정식 마이그레이션은 사용자 결정(방향별 레이아웃·전체화면 미적용분 존재). lotto는 게임(추천기)이 아니라 대상 제외.
