### 테마(산리오풍, 사용자 확정: 주인공=토끼)
- 차 = 동물 친구. 주인공(target)은 토끼 고정, 친구는 곰/고양이/강아지/병아리/펭귄 순환(`characters.js` 신설, 종류만 정의).
- 캐릭터 얼굴은 흰 베이스 + 눈/볼/코 + 종류별 귀를 `render.js`의 인라인 SVG(viewBox 0~100)로 그림. 색(colors.js)과 종류(characters.js)를 render가 조합.
- 귀/몸이 같은 색이면 차 배경에 묻혀 안 보이던 1차 시안 문제 → 흰 얼굴 베이스 + 흰 귀(안쪽 몸색 포인트)로 재작업해 해소(browser-shot 6종 검증).
- 색을 파스텔로 전환(`colors.js`). 보드 바닥/격자선/집 색은 `.rushhour` 스코프 `--rh-*` 변수로 모음(04 §3.2 예외). 출구 = 🏠 이모지(토끼의 집).

### 화면·캐릭터
- 얼굴 크기 버그: `--cell`(보드 너비 기준 %)을 얼굴에 그대로 써서 차 너비 기준 1/6로 쪼그라들던 것 → 차의 짧은 변(`.car.h`/`.car.v` 분기) 기준 정사각으로 꽉 채움. SVG 내부 head r44로 키움.
- 배경 핑크 거부 → 크림(`--rh-board #f3ead9`). 출구는 집(허접) → 출구 길(골드+흰 점선 차선) + 보드 밖 화살표·별 게이트(SVG, 이모지 font-size가 --cell %를 글자 기준으로 재해석해 점만큼 작아지던 버그 회피).
- 차 안쪽 윤곽선 = 입체용 inset 그림자(흰 하이라이트)가 둥근 모서리 따라 실선처럼 → 부드러운 외부 그림자로 교체. target도 동일.
- 블록 크기 → 동물 고정(`KIND_BY_SHAPE`): 가로2 고양이/가로3 강아지/세로2 병아리/세로3 펭귄, 주인공 토끼. 같은 종류도 표정 4종(normal/happy/wink/surprised)·종류별 파스텔 색(`KIND_COLORS`)·액세서리(리본/나비넥타이/꽃)를 블록 위치 해시로 배정.

### 난이도 밸런스 재설계 (사용자 "60스테이지도 1차원 자명" 지적)
- 원인: 최소수(opt)만으로 곡선 → opt 낮으면 "차 하나 비키면 끝" 자명 퍼즐이 후반에도. 
- 해결: 난이도 2축(최소수 + 얽힘=최적해에서 움직이는 서로 다른 차 수). 한 배치의 도달 가능 전체 상태를 모으고 "풀린 상태 전부"에서 멀티소스 BFS로 정확한 최소수 산출(단일소스 역BFS는 특정 골 상태 거리라 18수로 오측정 → 실제 3수 버그였음, 멀티소스로 정정).
- 후반 "막는 차 2~3대+ & 서로 다른 차 4~5대+" 강제로 자명 배제. 구성당 추출 2개 + 셔플로 인접 유사 퍼즐 방지.
- 곡선: 입문 1~21(1~4수)/쉬움 22~81(5~8)/보통 82~170(9~12)/도전 171~186(13~16). 총 186개(보통 89로 보강). 6x6에서 13수+ 희귀해 도전 16개.
- 퍼즐 생성기는 scratchpad 일회성(gen-bal.mjs). 최소수는 게임 런타임 solver가 계산, 테스트가 전수 검증(주석=실제 일치 verify).
