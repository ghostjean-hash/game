## 2026-07-05 (11차) 10x10 하트 손그림 교체 - 다운샘플 통짜 형태 해소

- 사용자가 클리어 화면 스크린샷과 함께 "10x10 색깔별 하트가 전혀 하트 모양이 아니다, 체크해달라"고 지적했다. 확인해 보니 흰하트를 비롯한 10x10 하트들이 위쪽이 한 덩어리로 꽉 찬 둥근 삼각형이었고, 하트의 특징인 위쪽 두 봉우리와 그 사이 V홈이 없었다.
- 근본 원인은 이 하트들이 Twemoji 하트 이모지 PNG를 10칸으로 자동 다운샘플해 만든 것(build-emoji-puzzles.mjs)이라는 점이다. 10칸 해상도에서는 봉우리 사이 홈이 뭉개져 사라진다. 실제로 갈색·보라·흰·빨강 하트의 grid가 글자 하나 안 틀리고 완전히 동일했다. 반면 손그림인 5x5 튜토리얼 하트는 봉우리와 홈이 살아 정상이었고, 15x15 하트는 칸이 충분해 전부 정상이라 손대지 않았다.
- 처방으로 10x10 하트 11종을 손그림으로 교체했다. 먼저 봉우리 두 개 + V홈 + 아래 뾰족을 갖춘 후보 하트를 solver.verifyPuzzle로 유일해 검증해 통과한 모양(후보 A)을 확정했다. 단색 8종(갈색·노랑·초록·파랑·보라·주황·빨강·흰)은 이 모양으로 색만 바꿔 일괄 교체했다. 다색 3종은 성격이 달라 개별 처리했다 - 반짝하트는 같은 후보 A 형태에 반짝 하이라이트 색만 재배치, 두하트는 큰 하트와 작은 하트가 겹친 형태로 새로 그림, 하트리본은 봉우리 있는 하트 + 아래 리본 꼬리로 새로 그림. 색 배치는 유일해 검증(채움 여부만 판정)과 무관하므로 검증 통과 모양 위에 원본 팔레트로 배치했다.
- 검증 증거: 교체 후 실제 puzzles.js에서 다시 읽어 하트 전체를 verifyPuzzle로 전수 재검증 FAIL 0, 자동 테스트 20/20 PASS(내장 퍼즐 전수 유일해 검증 포함), ASCII 렌더로 11종 모두 봉우리+홈이 살아난 것을 육안 확인했다. two_b(하트 4개 배치) 후보는 "두하트"와 개수가 안 맞아 제외했다.
- 미해결 잔여: (1) 두하트는 원본이 단색이라 큰/작은 하트를 색으로 구분하지 않았다(형태 배치로만 구분, 원본 팔레트 정합 유지). 색 구분을 원하면 팔레트 추가 여지. (2) 실기기 browser-shot 실측은 하지 않았다 - grid에서 렌더는 자명하고 데이터 값만 바뀌어 위험이 낮다고 판단했으나, 사용자가 화면으로 지적한 사안이라 다음에 실제 화면 확인이 이상적이다.
- self-critique: 사용자가 스크린샷으로 지적한 문제였는데도 수정 후 같은 화면을 browser-shot으로 되짚지 않고 ASCII 렌더 + 자동 테스트로 갈음했다. 데이터 변경이라 위험은 낮지만, 지적의 출발점이 실제 화면인 만큼 같은 경로로 확인을 닫는 것이 원칙(verify-real-user-path)에 더 맞았다.
