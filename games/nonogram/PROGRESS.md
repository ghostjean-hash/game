## 2026-07-02 (1차 후속) 컨셉 초안 합의

- 사용자 확정 사항: (1) 타깃 = 초등 4학년 여아, 귀엽고 심플한 캐릭터 선호. (2) 색 방향 = "핑크 배제"가 아니라 "초4 여아 = 핑크 도배" 상투 회피가 지시의 본질. 민트·라벤더·크림 기본 축 + 핑크는 포인트 색 중 하나로만.
- 자비스 초안(사용자 이견 없이 유지): 도감 수집형 노노그램(완성 순간 흑백 격자 → 컬러 픽셀 그림 변신이 핵심 보상), 실패 없음(타이머·목숨 없이 실수 카운트 + 별점 0회=3 / 1-2회=2 / 그 이상=1), 터치 우선(칠하기/X 모드 버튼 + 드래그 연속 입력), 스테이지 맵 + 도감 단독 메타 보상(상점은 1차 범위 제외), 튜토리얼 3판.
- 데이터 전략(사용자가 외부 조언 전달, 자비스가 허브 표준에 맞게 조정 후 합의): 자체 그림 텍스트 격자 제작 → 행/열 힌트 자동 생성 → 솔버가 유일해 + **추측 없이 줄 논리만으로 풀림** 전수 검증(테스트 강제) → 솔버 반복 횟수로 난이도 자동 태깅. puzzlekit은 html-game 표준(외부 라이브러리 금지) 위반이라 미도입, 같은 구조를 src/core/에 직접 구현(rushhour BFS 솔버 전례). WebPBN은 그림 스타일 참고만, 데이터 미사용. 크기 3단 = 5×5(튜토리얼·초급) / 10×10(중급) / 15×15(고급 해금).
- 검증 증거: 골격은 browser-shot 실측 3건(게임 화면 로드 + 테스트 PASS 1/1 + 허브 카드 노출), _registry.json은 JSON.parse 통과, handoff는 gitignore 매칭 확인.
- 미확정 잔여: "이 초안대로 docs/01_spec + 02_data 확정 반영 + 구현 착수" 최종 확정을 사용자에게 질문한 상태에서 checkpoint 발동됨. 다음 세션 첫 행동 = 그 답변 받기.
