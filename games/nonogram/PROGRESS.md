- 자비스 초안(사용자 이견 없이 유지): 도감 수집형 노노그램(완성 순간 흑백 격자 → 컬러 픽셀 그림 변신이 핵심 보상), 실패 없음(타이머·목숨 없이 실수 카운트 + 별점 0회=3 / 1-2회=2 / 그 이상=1), 터치 우선(칠하기/X 모드 버튼 + 드래그 연속 입력), 스테이지 맵 + 도감 단독 메타 보상(상점은 1차 범위 제외), 튜토리얼 3판.
- 데이터 전략(사용자가 외부 조언 전달, 자비스가 허브 표준에 맞게 조정 후 합의): 자체 그림 텍스트 격자 제작 → 행/열 힌트 자동 생성 → 솔버가 유일해 + **추측 없이 줄 논리만으로 풀림** 전수 검증(테스트 강제) → 솔버 반복 횟수로 난이도 자동 태깅. puzzlekit은 html-game 표준(외부 라이브러리 금지) 위반이라 미도입, 같은 구조를 src/core/에 직접 구현(rushhour BFS 솔버 전례). WebPBN은 그림 스타일 참고만, 데이터 미사용. 크기 3단 = 5×5(튜토리얼·초급) / 10×10(중급) / 15×15(고급 해금).
- 검증 증거: 골격은 browser-shot 실측 3건(게임 화면 로드 + 테스트 PASS 1/1 + 허브 카드 노출), _registry.json은 JSON.parse 통과, handoff는 gitignore 매칭 확인.
- 미확정 잔여: "이 초안대로 docs/01_spec + 02_data 확정 반영 + 구현 착수" 최종 확정을 사용자에게 질문한 상태에서 checkpoint 발동됨. 다음 세션 첫 행동 = 그 답변 받기.

## 2026-07-02 (2차) 1차 구현 완료 + 사운드

- 컨셉 초안을 docs에 확정하고 게임을 플레이 가능한 수준까지 구현했다. 사용자 지시 흐름은 /jn 진단으로 nonogram 구현을 다음 작업으로 추천 → "확정 및 구현" → "쭉 끝까지" → "사운드 적용" 순서였다.
- docs 4종 확정: 01_spec은 도감 수집형·실패 없음(실수 카운트 + 별점)·터치 우선·크기 3단·사운드 절, 02_data는 상수·파스텔 팔레트·퍼즐 포맷·저장 스키마(muted 포함), 03_architecture는 의존성 방향(data ← core ← render/input ← main)과 audio 폴더, 04_conventions는 네이밍과 사운드 합성값 매직 넘버 예외.
- 코드 전체 구현: core(hints 힌트 생성 / solver 줄 논리 유일해·무추측 검증 + 난이도 / board 불변 상태 / stars 별점), data(constants / colors / puzzles 20종), render 4뷰(pixel·boardView·mapView·resultView·albumView), input(boardInput 터치 드래그 + 키보드), audio(sound WebAudio 합성, rushhour 패턴 재사용), main 조립.
- 퍼즐 20종: 튜토리얼 3(하트·별·집) + 초급 6 + 중급 6(10×10) + 고급 5(15×15). 모양(채움 패턴)은 solver로 유일해·무추측 전수 검증한 것만 채택하고, 색은 검증에 무관하므로 통과한 모양 위에 파스텔로 자유 배치했다.
- 사운드: 칠하기 / 지우기 / X 표시 / 실수 하강음 / 클리어 팡파레 / 별 반짝. 드래그 연속 입력은 실제로 칸이 바뀐 경우에만 재생. 음소거 토글 + 저장 복원, 백그라운드 절전, 첫 사용자 제스처에서 오디오 unlock(자동재생 정책 대응).
- 허브 반영: registry status를 wip에서 playable로 올리고 applications.md를 1차 구현 완료로 갱신.
