# 05. HUD·전반 퀄리티 향상 계획 (PLAN)

> 상태: **전체(16항목) 구현 완료 (2026-07-03)**. 사용자 "전체를 구현해줘".
> 결정값: 틀린 칸 = 옅게 남김 / 도움 = 별 양보(1회당 별 상한 -1, 최소 1). 맵 썸네일 = 예고 유지.
> 반영은 01_spec/02_data/03_architecture SSOT에 흡수됨. 검증: 테스트 20/20 PASS + 실제 플레이 browser-shot(헤더·별예고·실수칸·완성줄 dim·손가락·되돌리기·도움·순차 변신·칭찬·맵 진행바·데스크톱, pageerror 0).

## 1. 목표와 범위

1.1. 목표: 게임이 "돌아간다" 수준에서 "손에 붙고 예쁘다" 수준으로. 특히 플레이 중 정보 표시(HUD)를 꼼꼼히.

1.2. 범위: 플레이 화면 HUD 최우선. 이어서 반응형 레이아웃, 완성 연출, 맵/도감, 진행 편의. 게임 규칙·퍼즐 데이터·솔버는 이번 범위 밖(이미 검증 완료).

1.3. 비범위: BGM, 온라인, 상점, 신규 퍼즐 제작.

## 2. 현황 진단 (실측 근거, 2026-07-03 browser-shot)

플레이 화면을 모바일(390px)·데스크톱(1280px)·5×5·10×10로 캡처해 확인한 문제.

2.1. **뭘 푸는지 모른다**: 플레이 화면 상단에 "← 맵 / 실수 N / 🔊"만 있고 퍼즐 이름·난이도·진행 위치(예: 초급 7/100)가 없다. 208판 중 어디인지 알 수 없다.

2.2. **어디까지 했는지 안 보인다**: 행/열 힌트 숫자가 회색으로 고정. 그 줄을 다 채워도 힌트가 그대로라, 완성한 줄과 남은 줄이 구분되지 않는다. 노노그램의 기본 편의(완성 줄 흐리게)가 없다.

2.3. **어디를 틀렸는지 모른다**: 정답이 빈칸인 칸을 칠하면 실수 카운트만 오르고, 잘못 칠한 칸이 정상 칸과 똑같이 보인다. "스스로 지워 교정" 설계인데 정작 어디가 틀렸는지 표시가 없어 교정 자체가 어렵다. (실측: 하트에서 빈칸을 칠했더니 실수 1만 뜨고 그 칸은 정상 칠과 동일)

2.4. **별점 동기가 약하다**: 실수에 따라 별이 3→2→1로 정해지는데, 플레이 중에는 지금 몇 별인지 안 보인다. 결과 화면에서야 안다.

2.5. **데스크톱이 휑하다**: 5×5 격자가 화면 상단에 작게 몰리고 그 아래로 빈 공간이 넓다. 세로 중앙 정렬이 안 되고 셀 크기 상한도 낮다.

2.6. **안내문·여백 정리 부족**: 하단 안내문이 항상 떠 있고, 10×10에서 열 힌트 위 여백이 크다.

2.7. **완성 연출이 단출하다**: 격자 전체가 한 번 pop하는 정도. 208판을 도는 게임의 핵심 보상인데 밋밋하다.

## 4. 단계(phase)와 검증

각 phase 끝에 browser-shot으로 실제 플레이 경로를 밟아 확인한다(모바일+데스크톱). 회귀는 테스트 러너로.

4.1. **Phase A - HUD 핵심(P0, 항목 1~4)**: 정보 헤더 + 완성 줄 dim + 실수 표시 + 별점 예고. 이 넷이 체감 개선의 대부분. 완성 줄 판정은 `src/core/`에 순수 함수로 추가하고 테스트를 붙인다(줄 완성 여부는 로직).

4.2. **Phase B - 레이아웃·연출(P1, 항목 5~8)**: 셀 피드백, 데스크톱 중앙 정렬, 여백 정리, 변신 연출 강화.

4.3. **Phase C - 편의·확장(P2, 항목 9~12)**: 중도 저장, 힌트 도움, 맵/도감 다듬기, 접근성.

검증 기준: (a) 모바일·데스크톱 각 화면 browser-shot 육안 + boundingBox 겹침 0 (b) 완성 줄 판정 등 로직은 테스트 PASS (c) 실제 풀이 플로우(칠→실수→완성→결과) 전 구간 오류 0.

## 6. 열린 결정 (착수 전 사용자 선택)

6.1. 범위: P0(Phase A)만 먼저 하고 볼지, P1까지 한 번에 갈지, 전체(P2 포함) 갈지.
6.2. 실수 칸 표시 방식: 잠깐 펄스 후 사라짐 vs 교정 전까지 빨강 지속(초4엔 지속이 더 친절하나 규칙상 "메모"라 논쟁 여지).
6.3. 힌트 도움(항목 10) 포함 여부: 넣으면 쉬워지지만 초4 이탈 방지에 유효.
