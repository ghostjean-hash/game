# 01. 게임 사양 (spec)

> 게임 규칙 / 조작 / 화면 흐름의 SSOT. 컨셉 확정 완료(2026-07-02). 수치는 02_data.md, 색은 colors.js/tokens.css.

## 1. 컨셉

한 줄: 숫자 힌트로 칸을 채워 숨은 픽셀 그림을 완성해 모으는 노노그램.

- 타깃: 초등 4학년 여아. 귀엽고 심플한 캐릭터 그림.
- 톤: 민트·라벤더·크림을 기본 축으로 한 파스텔. 핑크는 포인트 색 하나로만(초4=핑크 도배 상투 회피).
- 핵심 보상: 퍼즐을 다 풀면 흑백으로 채워진 격자가 컬러 픽셀 그림으로 변신한다. 이 "변신 순간"이 재미의 중심.
- 정서 설계: 실패가 없다. 타이머도 목숨도 없다. 틀려도 실수 횟수만 쌓이고, 그 횟수로 별점만 달라진다.

## 2. 핵심 규칙 (코어 루프)

1. 스테이지 선택 → 빈 격자 + 행/열 숫자 힌트 제시.
2. 각 행/열의 숫자는 그 줄에서 "연속으로 칠해진 칸 묶음의 길이"를 왼→오 / 위→아래 순서로 나열한 것.
   - 예: 행 힌트 `3 1` = 그 행에 길이 3 묶음, 한 칸 이상 띄고, 길이 1 묶음.
3. 플레이어는 칸을 "칠하기" 또는 "X(빈칸 확정)"로 표시한다. X는 메모일 뿐 정답 판정에는 무관.
4. 칠한 칸 집합이 정답 그림의 칠한 칸 집합과 완전히 일치하면 클리어.
5. 클리어 순간 흑백 격자가 컬러 그림으로 변신. 그 그림은 스테이지 맵에 컬러 썸네일로 남는다.

정답 판정 기준: "칠해야 하는 칸을 정확히 다 칠했고, 칠하면 안 되는 칸은 칠하지 않았다"(칸의 정답 색은 판정에 무관, 오직 칠함/안칠함).

## 3. 조작

터치 우선 설계. 마우스/키보드는 보조.

- 모드 토글 버튼 2개: **칠하기 모드** / **X 모드**. 현재 모드가 강조 표시된다.
- 탭(클릭): 현재 모드대로 칸 상태를 바꾼다. 단 칠하기 모드에서 이미 맞게 칠한 칸(정답 칠칸)은 잠겨서 지워지지 않고, 틀리게 칠한 칸(느낌표 표시)만 지울 수 있다. 표시(X) 모드는 자유롭게 토글.
- 드래그: 시작 칸에서 정한 동작(칠함/지움)을 이어지는 칸에 연속 적용한다. 방향은 "직선 고정" - 드래그 시작 후 주 이동 방향(가로/세로)을 정해 그 한 줄로만 투영하고, 옆 줄로 손이 흔들려도 침범하지 않는다(저학년 오조작 방지). 동작(칠함/지움)도 첫 칸의 전이로 고정한다.
- 키보드(보조): 방향키 커서 이동, Space=칠하기 토글, X=X 토글.

실수 규칙: "칠하기 모드에서 정답이 빈칸인 칸을 칠함"이 1회 실수. X 표시와 지우기는 실수가 아니다. 잘못 칠한 칸은 정상 칠 색 그대로 칠해지되 그 위에 느낌표(!)가 떠서 어디를 틀렸는지 바로 보이고(처음 한 번 붉은 테두리로 위치를 짚음), 스스로 지우면 사라진다(자동으로 되돌리지 않음). 맞게 칠한 칸은 느낌표가 없으므로 곧 정답이며, 실수로 지워지지 않도록 잠근다(지우기는 느낌표 칸에만 적용). 드래그로 느낌표 칸들을 지울 때 사이에 낀 맞은 칸은 건너뛴다.

되돌리기: 직전 행동(한 번의 칠하기·지우기·드래그 한 묶음, 도움 포함)을 취소한다. 여러 번 누르면 그만큼 거슬러 올라간다. 실수한 칸을 되돌리면 실수 개수도 함께 줄어든다.

도움: 막히면 아직 안 푼 한 줄을 정답대로 열어준다. 도움을 한 번 쓸 때마다 받을 수 있는 별 상한이 하나 줄어든다(최소 1개는 보장). 도움으로 채운 칸은 실수로 세지 않는다.

플레이 화면 정보(HUD): 상단에 난이도·진행 위치(예: 중급 12/100)와 지금 받을 별(실수·도움 반영)을 보여준다. 퍼즐 이름은 정답 그림을 미리 알려주는 스포일러라 플레이 중에는 HUD에 표시하지 않고 클리어 결과 화면에서만 공개한다. 상단은 난이도 배지(난이도별 색: 튜토리얼 라벤더·초급 민트·중급 앰버·고급 코랄) + 진행 위치로만 간결하게 두고, 뒤로/소리는 양끝 아이콘 버튼으로 배치한다. 실수 횟수는 눈에 띄게 크게 보여준다. 한 줄을 완성하면 그 줄의 힌트 숫자가 흐려지고 작게 반짝이며, 여러 줄을 잇달아 완성하면 짧은 응원이 뜬다. 그림을 다 맞추면 칠하기·표시 모드 버튼이 잠긴다(완성 표현 + 오조작 방지). 플레이 화면은 상태(뒤로·난이도·진행·실수·별) / 격자 / 조작(칠하기·표시·되돌리기·도움·소리·전체화면) 3그룹으로 나뉜다. 세로 방향에서는 UI를 위·아래로, 가로 방향에서는 좌(상태)·우(조작)로 분산 배치하고 격자는 항상 화면 중앙에 온다. 격자는 남은 공간(폭·높이)에 맞춰 크기가 자동으로 정해진다. 플레이 중에는 페이지가 스크롤·바운스되지 않도록 뷰포트에 고정하고(드래그해도 격자가 밀리지 않음), 노치·홈바(safe-area)를 피해 배치한다. 지원 기기에서는 전체화면 버튼으로 몰입 모드에 들어갈 수 있다. 맵·결과처럼 목록이 긴 화면은 그 화면 안에서만 스크롤한다.

## 4. 성공 / 실패 / 종료 조건

- 성공: §2.4 일치. 별점 부여(§5).
- 실패: 없음. 종료(포기)해도 페널티 없음, 진행은 저장.
- 종료: 클리어 시 결과 화면(변신 + 별점) → 스테이지 맵 복귀.

## 5. 진행 시스템

- 스테이지 맵: 퍼즐을 난이도 순으로 나열. 튜토리얼 3판 → 초급 → 중급 → 고급(해금).
- 크기 3단: 5×5(튜토리얼·초급) / 10×10(중급) / 15×15(고급, 앞 단계 일정 수 클리어 시 해금).
- 별점: 클리어 시 실수 횟수로 결정. 실수 0회=별 3 / 1~2회=별 2 / 3회 이상=별 1. (경계값은 02_data.md 상수.)
- 수집: 클리어한 그림은 스테이지 맵에 컬러 썸네일로 남아 모으는 재미를 준다(별도 도감 화면 없이 맵이 겸함). 미클리어는 물음표로 가려 정답 모양을 감춘다.
- 상점/재화는 1차 범위 제외.

## 6. 화면 흐름

```
[스테이지 맵] --선택--> [플레이 화면] --클리어--> [결과(변신+별점)] --> [스테이지 맵]
```

- 스테이지 맵: 퍼즐 목록(잠금/별점 표시). 클리어한 퍼즐은 컬러 썸네일, 미클리어는 물음표로 가린다(맵이 도감을 겸함).
- 플레이 화면: 격자 + 행/열 힌트 + 모드 토글 + 실수 카운트 + 나가기.
- 결과 화면: 흑백→컬러 변신 연출 + 별점 + 다음/맵으로.

## 7. 튜토리얼

첫 진입 시 5×5 튜토리얼 3판을 순서대로 안내한다. 1판=힌트 읽는 법, 2판=X 표시 활용, 3판=자유 풀이. 튜토리얼 완료 상태는 저장한다.

## 8. 사운드

Web Audio로 합성한 짧은 효과음(음원 파일 0). 조용하고 귀엽게, 과하지 않게.

- 칠하기: 짧고 밝은 톡. 지우기: 살짝 낮은 톡. X 표시: 부드러운 틱.
- 실수(정답 빈칸을 칠함): 짧은 하강음으로 부드럽게 알림(벌 느낌 아님).
- 클리어: 상승 팡파레 + 별 개수만큼 반짝음 계단식.
- 음소거 토글은 플레이 화면 우상단. 상태는 저장하고, 음소거 중엔 오디오를 재워 배터리를 아낀다.
- 드래그 연속 입력 시 실제로 칸 상태가 바뀐 경우에만 소리(무변화 반복은 무음).

## 9. 비범위 (1차 제외)

상점/재화, 일일 보상, 랭킹, 온라인, 배경음악(BGM), 퍼즐 에디터.
