# 기획서: 스도쿠 (game_ghost / sudoku)

문서 버전: v0.2 / 갱신: 2026-05-01 / 상태: M1+ 구현 반영

# 1. 개요

1.1. **한 줄 정의**: 초등 3~4학년도 시작할 수 있는 진입 난이도부터, 스마트 힌트로 풀이 사고를 학습할 수 있는 모바일 PWA 스도쿠.
1.2. **플랫폼**: game_ghost 허브 안의 웹 게임. PWA, 모바일 우선, 키보드 보조.
1.3. **타깃**:
- 1차: 초등 중·고학년 입문자 (가족 단위 동반 플레이 포함).
- 2차: 일반 라이트 사용자 (출퇴근 5~20분 풀이).
- 3차: 풀이 학습 욕구 있는 중급자.
1.4. **코어 루프**: 난이도 선택 → 퍼즐 풀이(메모/힌트 보조) → 클리어 → 베스트/통계 갱신.
1.5. **세션 길이 목표**: 입문 1~3분, 초급 3~7분, 중급 5~15분, 상급 10~30분.

# 2. 디자인 원칙

2.1. **진입 장벽 0**: 첫 화면에서 1~2 탭 안에 첫 셀을 채울 수 있다.
2.2. **학습이 곧 진행**: 힌트는 "정답 제공"이 아니라 "왜 그런지"를 점진 공개로 가르친다.
2.3. **부기 부담 제거**: 후보 숫자 자동 채움(auto-notes)·자동 정리로 메모 노동 최소화.
2.4. **가벼움**: 보드 면적 최우선, 광고/배너 없음, 오프라인.
2.5. **결정은 사용자**: 자동 보조 기능은 모두 토글, 보조 끄면 정통 풀이.

# 3. 난이도 설계 (4단계, 초등 진입형)

| 단계 | 명칭 | 단서 수 | 풀이 필요 기법 | 한 판 평균 | 진입 안내 |
|---|---|---|---|---|---|
| L1 | 입문 (Starter) | 38~46 | Naked Single만 | 1~3분 | 첫 게임 권장. 자동 메모 ON, 충돌 강조 ON, 스마트 힌트 풀공개 |
| L2 | 초급 (Easy) | 32~38 | Naked Single + Hidden Single | 3~7분 | 자동 메모 기본 ON, 메모 자동 정리 ON |
| L3 | 중급 (Medium) | 28~32 | + Pointing Pair / Naked Pair | 5~15분 | 자동 메모 기본 OFF, 사용자 토글 |
| L4 | 상급 (Hard) | 24~28 | + Box/Line Reduction, Hidden Pair | 10~30분 | 모든 보조 기본 OFF |

3.1. **풀이 기법 기준**으로 난이도를 가른다(단서 수만으로 가르지 않는다).
3.2. **L1은 거의 채워진 보드**. 한 칸씩 보면 답이 보이는 수준. 가족 동반 플레이 진입용.
3.3. 상급 위 단계는 본 기획에서 제외(전문가/지옥 추후).

# 4. 코어 시스템

## 4.1. 보드와 룰
- 9x9, 3x3 박스 9개. 1~9 각 행·열·박스 1회.
- 셀 상태: `given`(고정), `user`(입력), `empty`.
- 위반: 같은 행/열/박스에 같은 값 두 곳 이상이면 양쪽 빨강.
- 클리어 조건: 빈칸 0 + 위반 0.

## 4.2. 입력
- 셀 → 숫자 모델.
- 키보드: `1~9` 입력, `0/Backspace/Delete` 지우기, 화살표 셀 이동, `N` 메모, `U` undo, `R` redo, `H` 힌트, `P/Esc` 일시정지.
- 터치: 숫자 패드 9개 + 보조 버튼(메모/지우기/undo/힌트).

## 4.3. 메모(노트)
- 셀당 1~9 후보 비트마스크.
- 입력 확정 시 같은 행/열/박스 메모에서 그 숫자 자동 제거(옵션, 기본 ON).

## 4.4. auto-notes
- 토글: 모든 빈 셀 후보 자동 표기 / 모든 메모 비움.
- 입문/초급 기본 ON, 중급/상급 기본 OFF.

## 4.5. 힌트 (스마트 힌트, 3단계 점진 공개)

| 단계 | 표시 |
|---|---|
| 1차 (Where) | 적용 가능한 행/열/박스 영역을 약하게 색칠 |
| 2차 (Which) | 정답 셀을 깜빡임 + 사용된 기법명 표시 |
| 3차 (Why) | 정답 숫자 + 한 줄 논리 설명 |

- 힌트 사용 시 그 판은 classic 베스트 후보에서 제외.
- 풀이 가능한 기법이 여러 개면 가장 단순한 것부터.

## 4.6. 실수/오류
- 자동 확인(옵션). ON: 오답 즉시 빨강 + 카운트, OFF: 위반만 빨강.
- 한도(옵션): 무제한 / 3 / 5. 한도 초과 시 게임 오버.
- L1·L2 자동 확인 기본 ON, L3·L4 OFF.

## 4.7. undo/redo
- 입력·메모·지우기 모두 스택. 한도 100.

## 4.8. 타이머/일시정지
- 첫 입력에서 시작. 일시정지 시 보드 가림 + 타이머 정지.

## 4.9. 진행 저장 / 이어하기
- 진행 중 매 입력 + 디바운스 200ms로 자동 저장. 새 판 시작 시 덮어쓰기.

## 4.10. 베스트 기록
- 난이도별 최단 시간. classic(보조 미사용) / assist(보조 사용 또는 힌트 사용)로 분리.
- 1차에는 classic만 노출, assist는 통계.

# 5. 스마트 힌트 명세

## 5.1. 적용 기법(우선 낮음 → 높음)

| 우선 | 기법 | 한 줄 설명 | 노출 난이도 |
|---|---|---|---|
| 1 | Naked Single | 한 셀 후보가 1개 | L1+ |
| 2 | Hidden Single | 행/열/박스에서 어떤 숫자 자리가 1곳 | L2+ |
| 3 | Pointing Pair/Triple | 박스 안 후보가 한 행/열에 몰림 → 그 행/열 박스 밖 후보 제거 | L3+ |
| 4 | Naked Pair | 두 셀이 동일 후보 2개만 → 영역 내 다른 셀에서 제거 | L3+ |
| 5 | Box/Line Reduction | 행/열의 후보가 한 박스에 몰림 → 그 박스 다른 셀에서 제거 | L4 |
| 6 | Hidden Pair | 두 숫자의 후보가 두 셀에만 → 그 두 셀 다른 후보 제거 | L4 |

5.1.1. 직접 채움(1·2)을 우선 제시. 후보 좁힘(3~6)은 그것 없이 진행 못 할 때만.

## 5.2. 힌트 엔진 출력

```ts
type Hint = {
  technique: "naked-single" | "hidden-single" | "pointing-pair" | "naked-pair" | "box-line-reduction" | "hidden-pair";
  scope: { rows?: number[]; cols?: number[]; boxes?: number[] };
  cells: number[];
  value?: number;
  removeCandidates?: { cell: number; values: number[] }[];
  message: { ko: string; en: string };
};
```

# 6. UI 구성

## 6.1. 화면 (모바일 세로)

```
[← 홈] [난이도] [중앙 타이머]   [실수 x/3] [⏸]
┌──────────────────────────┐
│         9 x 9 보드        │
└──────────────────────────┘
[메모][지우기][↶undo][힌트💡]
[1][2][3][4][5][6][7][8][9]
```

## 6.2. 가로 모드 / 태블릿 (orientation landscape & 폭 ≥ 600 또는 폭 ≥ 900)
- 패드를 보드 우측에 사이드로 배치 (3x3 키패드).
- 사이드 폭 240~360px. 버튼 크기 vmin 기반 clamp(상한 80px).
- 보드 한 변은 좌측 영역에서 정사각 최대화.

## 6.3. 톤
- **스도쿠 페이지에 한정해 라이트 톤 적용**(글로벌 다크 토큰을 페이지 스코프에서 오버라이드).
- 셀 강조 톤: related(옅은 회색) → same-num(옅은 파랑) → selected(진한 파랑 + outline).
- 책 위 인쇄 스도쿠에 가까운 차분한 시각.

## 6.4. 셀 시각

| 상태 | 표현 |
|---|---|
| given | 굵은 검정 |
| user | 파랑(`#2563eb`) |
| user 위반/오류 | 빨강(`#d92d20`) |
| 선택 | 진한 파랑 배경 + outline |
| 관련 행/열/박스 | 옅은 회색 |
| 같은 숫자 | 옅은 파랑 |

# 7. 데이터 명세

## 7.1. 핵심 타입

```ts
type CellIndex = number;
type Digit = 0..9;       // 0 = empty
type NoteMask = number;  // bit 1<<(d-1)

type Puzzle = {
  id: string;
  difficulty: 1 | 2 | 3 | 4;
  given: Digit[81];
  solution: Digit[81];
  techniques: string[]; // 풀이 시 필요한 최고 기법
};

type GameState = {
  puzzleId: string;
  difficulty: 1 | 2 | 3 | 4;
  given: boolean[81];
  cells: Digit[81];
  notes: NoteMask[81];
  selected: CellIndex | null;
  errors: Set<CellIndex>;
  mistakes: number;
  hintsUsed: number;
  noteMode: boolean;
  options: { autoCheck: boolean; autoNotesCleanup: boolean; mistakeLimit: 0 | 3 | 5 };
  timer: { startedAt: number; pausedAt: number | null; pausedTotal: number };
  paused: boolean;
  over: boolean;
  cleared: boolean;
  undoStack: Step[];
  redoStack: Step[];
  lastChanged: CellIndex | null; // 클리어 확산 중심
};
```

| 키 | 값 |
|---|---|
| `sudoku.progress` | `GameState` (진행 1개) |
| `sudoku.best.<diff>.classic` | `{ time, date }` |
| `sudoku.best.<diff>.assist` | `{ time, date }` |
| `sudoku.stats.<diff>` | `{ played, cleared, totalTime }` |
| `sudoku.options` | 사용자 토글 |

# 11. 시각/이펙트 명세

## 11.1. 완성 웨이브 (행/열/박스)

| 트리거 | 이펙트 | 컬러 | 셀별 지연 |
|---|---|---|---|
| 행 새로 완성 | 좌→우 웨이브 | 파랑(`pulse-line`) | `col × 50ms` |
| 열 새로 완성 | 위→아래 웨이브 | 파랑(`pulse-line`) | `row × 50ms` |
| 3x3 박스 새로 완성 | 좌상→우하 웨이브 | 보라(`pulse-box`) | `(dr×3 + dc) × 50ms` |

- 셀 ::after 오버레이로 펄스. selected/related 등 다른 강조 클래스와 충돌 없음.
- 펄스 duration: 480~520ms.
- 동일 셀 다중 트리거 시 더 짧은 delay 채택.
- 지우기(0)는 트리거 안 함.

## 11.2. 클리어 확산

- 마지막 입력 셀(`state.lastChanged`)을 기준점으로 체비셰프 거리 × 70ms 지연.
- 기준 셀에는 `pulse-clear-origin` 추가 적용으로 처음부터 강하게 빛남(opacity 100%부터 시작, 980ms).
- 다른 셀은 `pulse-clear`(파랑→보라 그라데이션, 760ms).
- 클리어 진입 시점에 `pulse-line/pulse-box` 즉시 정리 + 한 프레임 대기 후 확산 시작 (시작점이 깔끔하게 보이도록).
- 가장 먼 셀(거리 8)까지 펄스 종료 = `8×70 + 800 ≈ 1410ms` 후 클리어 모달.

# 13. 구현 현황 (M1+)

13.1. **완료**:
- 9x9 보드 렌더 + 셀 선택, 키보드/패드 입력, 룰 검증(빨강 표시).
- 같은 숫자/관련 영역(행/열/박스) 약한 강조.
- 타이머(MM:SS), 일시정지(보드 블러 + 모달), undo, 지우기.
- 클리어 시 모달 + classic 베스트 시간 저장.
- 라이트 톤 페이지(스도쿠 한정 토큰 오버라이드 + 모달도 라이트로).
- 가로 모드/태블릿 사이드 패드(3x3 키패드, vmin 기반 버튼 크기 clamp).
- 행/열/박스 완성 웨이브 이펙트, 마지막 입력 셀 기준 확산 이펙트.
- L1 데모 퍼즐 1개 하드코딩.

13.2. **미완**: M2~M5의 모든 항목, 그리고 M1 범위 중에서도 sound/haptic·접근성 라벨 정밀화는 보류.
