# Sudoku 작업 컨텍스트

이 파일은 sudoku 게임 작업 시 자동 로드되는 컨텍스트다. 작업 지침/핵심 정보/주의점만 둔다. 진행 로그는 `PROGRESS.md`, 상세 기획은 `docs/`.

# 1. 게임 정의

1.1. **한 줄**: 초등 진입형 + 스마트 힌트로 풀이 학습이 가능한 모바일 PWA 스도쿠.
1.2. **상태**: M1+ 완료 (라이트 톤, 사이드 패드, 완성/클리어 이펙트), M2 진입 대기.
1.3. **타깃**: 초등 중·고학년 입문자 / 라이트 사용자 / 풀이 학습 욕구 중급자.

# 3. 핵심 결정 (작업 시 반드시 준수)

3.1. **라이트 톤 단일 운영**. 글로벌 다크 토큰을 페이지 스코프로 오버라이드. 글로벌 테마 토글 없음.
3.2. **가로 모드 사이드 패드**. 보드 정사각 최대화 + 우측 3x3 키패드. 버튼 크기 vmin clamp 상한 80px.
3.3. **클리어 확산 기준 = 마지막 입력 셀**(`state.lastChanged`). 진입 시 라인/박스 펄스 즉시 정리.
3.4. **풀이 기법 기준 난이도**(L1~L4). 단서 수만으로 가르지 않는다.
3.5. **힌트는 정답 제공이 아니라 학습**. 3단계 점진 공개(Where → Which → Why).

# 4. 작업 시 주의

4.1. 셀 색만으로 정보 전달 금지(접근성). 위반/오류는 색 외 보조 표시 고려.
4.2. 힌트 사용 시 그 판은 classic 베스트에서 제외. assist 베스트로.
4.3. 입력 처리 16ms / 힌트 계산 100ms(L4 최악 200ms) 이내 유지.
4.4. 진행 저장은 매 입력 + 디바운스 200ms.
4.5. 풀 기획은 `docs/PLAN.md` 참조. 결정 변경 시 PLAN.md와 본 파일 동시 갱신.
4.6. 진행/완료/다음 작업은 `PROGRESS.md` 참조.

# 5. 비스코프

5.1. 글로벌 다크/라이트 토글.
5.2. 변종(X-Sudoku, Killer 등).
5.3. 풀이 기법 튜토리얼 페이지.
5.4. 업적/뱃지, 멀티플레이/리더보드, 광고/결제.
