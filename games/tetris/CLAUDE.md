# Tetris 작업 컨텍스트

이 파일은 tetris 게임 작업 시 자동 로드되는 컨텍스트다. 작업 지침/핵심 정보/주의점만 둔다. 진행 로그는 `PROGRESS.md`.

# 1. 게임 정의

1.1. **한 줄**: SRS 회전 + 7-bag 기반 정통 테트리스. 모바일 우선 PWA.
1.2. **상태**: playable. 모바일 검증 통과. 입력/HUD 개편 완료.

# 2. 파일 구조

# 3. 핵심 사양

3.1. **회전**: SRS + 월킥(JLSTZ/I).
3.2. **랜덤**: 7-bag.
3.3. **락 딜레이**: 0.5s.
3.4. **점수**: 일반 [100/300/500/800] × 레벨, 10라인마다 레벨업. T-spin Guideline 점수표 적용(`docs/tspin.md`).
3.5. **고스트, 홀드 지원**.
3.6. **T-spin**: 3-corner 룰 + Mini/정식 분기 + TST 격상 + B2B ×1.5. 스프린트 미적용(시간 모드).

# 4. 입력

4.1. **키보드**: ←→↓↑/Space/Shift/P.
4.2. **터치**:
- 좌우 드래그: pan 가로 추적.
- 아래 드래그: 진입 임계(12px) 통과 후 거리 비례 소프트드롭(칸당 +1점, 단방향, 락 진입 안 함). 좌우 pan과 동일 추적 메커니즘.
- 탭: 회전.
- 위 스와이프: 즉시 내리기(하드드롭).
- 길게누름: 무동작(폐기). 홀드는 키보드 Shift/C만.
4.3. **입력 영역**: `.page` 전체. button/a는 `input.js` 가드로 회피.

# 5. HUD

5.1. topbar 인라인 흡수(보드 셀 축소 회피 위해 별도 행 금지).
5.2. NEXT/HOLD 미니 슬롯 + 중앙 큰 점수.
5.3. localStorage 베스트 점수.

# 6. 작업 시 주의

6.1. 게임 오버 후 `tickHud`가 매 프레임 `gameOver` 호출하지 않도록 `overHandled` 플래그 유지.
6.2. 다중 라인 클리어는 `applyClear` 내림차순 splice (인덱스 어긋남 방지).
6.3. 모바일 입력 영역에서 button/a 가드 누락 시 UI 클릭 막힘.
6.4. 진행/완료/다음 작업/미해결은 `PROGRESS.md` 참조.
