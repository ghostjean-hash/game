# Tetris 진행 로그

# 1. 현재 상태

1.1. **상태**: playable. 모바일/PC 실기 검증 통과. 정통 SRS 락 딜레이 룰 정렬 완료.
1.2. **마지막 갱신**: 2026-06-26.

# 2. 완료된 작업

2.1. SRS 회전 + 월킥(JLSTZ/I), 7-bag, 정통 락 딜레이(0.5s + Move/Rotate Reset Limit 15회), 고스트.
2.2. 점수 [100/300/500/800] × 레벨, 10라인마다 레벨업. 소프트드롭 칸당 +1, 하드드롭 칸당 +2.
2.3. 키보드 입력: ←→ 이동 / ↓ 정통 소프트드롭(*20G) / ↑·X 회전 / Z 반대회전 / Space 하드드롭 / Shift·C 홀드 / P·Esc 일시정지.
2.4. 터치 입력: 좌우/아래 드래그 추적(아래는 데드존 12px 후 거리 비례 소프트드롭, 칸당 +1점, 단방향) / 탭 회전 / 위 스와이프 즉시 내리기(하드드롭). 길게누름 매핑 폐기. 입력 영역 `.page` 전체.
2.5. topbar에 NEXT/HOLD 미니 슬롯 + 중앙 큰 점수.
2.6. 모드 시스템: 마라톤(레벨업 + 모바일 L10 / PC L12 캡 + 시작 보정 + "리듬에 들어왔다" 캡 진입 토스트) / 젠(0.5s/라인 고정, 가속 없음) / 스프린트(0.3s/라인, 40라인 클리어 종료) / 배움(초등 수학 학습, 2.0s/라인). URL `?mode=...` 진입로 + 메인 모드 카드 화면 + 마지막 선택 강조 + 모바일 젠 추천 뱃지.

2.13. 배움 모드(초등 수학 학습용, 2026-07-06): 초보(초4 대상)가 좌절 없이 감각을 익히도록 설계. 아주 느린 중력(2.0s/라인, 젠 대비 4배) + 가속·레벨업 없음. 수학 사고력 장치로 "완성까지 남은 빈칸(10의 보수)" 힌트 - 한 줄이 빈칸 1~3개만 남으면 그 빈 칸을 노란 테두리로 강조 + 첫 빈칸에 남은 칸 수 숫자 배지를 표시(고정 블록 기준, 낙하 피스 제외). 라인 성공마다 큰 격려 토스트("잘했어!/멋져!/최고야!/대단해!"). 게임 오버 시 격려 문구 + "다시 하기" 기본 버튼으로 즉시 재도전. 메뉴 카드 "어린이" 뱃지. best.kids 점수 분리 저장. T-spin 미적용(단순화). 메뉴 카드 최상단 배치(사용자 지시). browser-shot로 카드 노출·게임 진입·힌트 렌더(빈칸 3 배지)·게임오버 모달·콘솔 에러 0 검증.

2.14. 사운드 효과음(2026-07-06): Web Audio 합성(음원 파일 0, `sound.js`, nonogram 패턴 차용). 이동·회전·하드드롭·블록 고정·라인 클리어(4줄은 더 화려한 팡파레)·홀드·레벨업·게임 오버 이벤트에 짧은 합성음 연결. 첫 사용자 제스처에서 오디오 잠금 해제(브라우저 자동재생 정책), 유휴 4초 후 오디오 스레드 절전, 백그라운드 탭 자동 suspend. topbar 🔊 음소거 토글(localStorage `muted` 저장). 전 모드 공통 적용. browser-shot 전 이벤트 구동 시 콘솔 에러 0 검증(소리 자체는 무음 환경이라 미청취, 재생 호출 무결성만 확인).
2.7. 모드별 베스트 분리(`best.marathon` / `best.zen` / `best.sprint(ms)`). 게임 오버 시 모드 선택 화면 복귀 + 일시정지 모달 액션(재개/다시 시작/모드 선택/홈) + 기존 단일 베스트 폐기 1회 안내.
2.8. localStorage 베스트 점수.
2.9. 정통 테트리스 top-out: vanish 영역(`y < VANISH=2`) 락 시 즉시 game over.
2.10. 가로(landscape) 모드 + 4줄 동시 클리어 실기 검증 통과.
2.11. T-spin 감지 + 보너스 점수 (기획서 `docs/tspin.md`). 3-corner 룰 + Mini/정식 분기 + TST kick 격상 + B2B ×1.5 + 한글 토스트(T-스핀 더블! / 미니 T-스핀! 등). 마라톤·젠 적용, 스프린트 미적용. 실기 검증 통과.
2.12. 렌더 성능 최적화 (모바일 발열 개선, 2026-06-26): 매 프레임 보드 전체를 다시 그리던 것을 화면에 실제 변화가 있는 프레임에만 그리도록 바꿈(`renderKey` 서명 비교 + 고정 블록 변동 추적 `gridVersion`). 매 프레임 호출하던 배경색 `getComputedStyle`을 1회 캐시(`bgColor`). 점수·레벨·베스트 HUD와 NEXT/HOLD 미니 블록도 값이 바뀔 때만 갱신(`hudCache` / `lastNextType` 등). 시각·조작감 동일, 블록이 가만히 떨어지길 기다리는 idle 프레임의 CPU/GPU 낭비 제거. browser-shot 마라톤 모드 정상 렌더 + 콘솔 에러 0 검증.

# 3. 해결한 주요 버그

3.1. 인접 다중 라인 클리어 시 한 줄 안 사라짐: `applyClear` 내림차순 splice. (커밋 `e32ea4a`)
3.2. 게임 오버 후 빈 화면: `tickHud`가 매 프레임 `gameOver` 호출하던 문제 → `overHandled` 플래그. (커밋 `8ae9750`)
3.3. 모바일 HUD가 보드 가림: 오버레이 → grid 분리 → topbar 인라인. (커밋 `57bb6bb`)
3.4. 메뉴/게임 화면 hidden 충돌: attribute selector(`[hidden]`)로 `display:none` 명시. (커밋 `3d7f2be`)
3.5. 공중 정지(top-out 누락): vanish 영역에 락이 박혀 보이지 않는 잔재로 다음 피스가 막힘. `lockPiece`에서 vanish 영역 락 즉시 game over. (커밋 `53fb456`)
3.6. 공중 락(락 딜레이 룰 누락): 좌우 이동/회전 후 빈 공간 위에서 0.5s 후 락. `refreshLockAfterMove` 추출 + 빈 공간 위면 `locking` 해제, 바닥이면 `lockTimer` 리셋(최대 15회). (커밋 `e1de6a5`)
3.7. softDrop 이중 가속(`*6` + `softDropTick(*20)` 누적) → 단일 `*SOFT_DROP_MULT(=20)`로 정리. (커밋 `e1de6a5`)
3.8. 터치 길게누름 홀드 오발: 보드 본체에서 무의식적 손가락 정지로 홀드 발동되던 문제. 매핑 폐기. (커밋 `249991d`)
