# Tetris 진행 로그

# 1. 현재 상태

1.1. **상태**: playable. 모바일 검증 통과.
1.2. **마지막 갱신**: 2026-05-10.

# 2. 완료된 작업

2.1. SRS 회전 + 월킥(JLSTZ/I), 7-bag, 락 딜레이 0.5s, 고스트, 홀드.
2.2. 점수 [100/300/500/800] × 레벨, 10라인마다 레벨업.
2.3. 키보드 입력(←→↓↑/Space/Shift/P) + 터치 입력(좌우 드래그 추적/탭 회전/위 스와이프 홀드/아래 스와이프 하드드롭/길게누름 홀드).
2.4. 입력 영역을 `.page` 전체로 확장(button/a 가드).
2.5. topbar에 NEXT/HOLD 미니 슬롯 + 중앙 큰 점수.
2.6. localStorage 베스트 점수.

# 3. 해결한 주요 버그

3.1. 인접 다중 라인 클리어 시 한 줄 안 사라짐: `applyClear` 내림차순 splice로 해결. (커밋 `e32ea4a`)
3.2. 게임 오버 후 빈 화면: `tickHud`가 매 프레임 `gameOver` 호출하던 문제 → `overHandled` 플래그. (커밋 `8ae9750`)
3.3. 모바일 HUD가 보드 가림: 오버레이 → grid 분리 → topbar 인라인. (커밋 `57bb6bb`)

# 4. 미해결 / 개선 여지

4.1. T-spin 감지 없음.
4.2. 가로(landscape) 모드 실기 검증 미완료.
4.3. 4줄 동시 클리어 실기 검증 미완료.
4.4. 난이도 1차 패치 코드 적용 완료(곡선 완만화 + 모바일 레벨 10 / PC 레벨 12 캡 + 모바일 시작 1~3레벨 +0.10초 보정 + 캡 진입 카피 "리듬에 들어왔다"). 모바일/PC 실기 검증 대기.
4.5. 난이도 2차 패치 1단계 코드 적용 완료. MODES 시스템(마라톤/젠/스프린트) + URL 쿼리 진입로 `?mode=marathon|zen|sprint` + 젠 0.5s/라인 고정 + 스프린트 0.3s/라인 + 40라인 클리어 종료 + HUD 모드별 분기(LV/L · 젠/L · ⏱/남) + 모드 진입 토스트.
4.6. 난이도 2차 패치 2단계 코드 적용 완료. 메인 화면(모드 카드 3장 + 마지막 선택 강조 + 모바일 젠 추천 뱃지) + 베스트 분리(`best.marathon` / `best.zen` / `best.sprint(ms)`) + 기존 단일 베스트 폐기 1회 안내 + 게임 오버 동선(모드 선택 화면 복귀) + 일시정지 모달 액션 갱신(재개/다시 시작/모드 선택/홈). 실기 검증 대기.
4.7. 터치 입력 재설계 코드 적용 완료. 아래 드래그(진입 임계 12px 통과 후) = 거리 비례 소프트드롭(칸당 +1점, 단방향, 락 진입 안 함). 좌우 pan과 동일 추적 메커니즘. 위 스와이프 = 즉시 내리기(하드드롭). 길게누름 = 홀드(단일 채널화). 키보드 매핑은 그대로(↓ 정통 소프트드롭, Space 하드드롭, Shift/C 홀드). 실기 검증 대기.
4.8. 공중 정지 버그 정공법 fix 코드 적용 완료. `lockPiece`에서 활성 락 셀이 vanish 영역(y < VANISH=2)에 걸쳤으면 즉시 `state.over=true` 처리 → 보이지 않는 잔재가 다음 피스와 충돌해 시각상 "공중 정지"로 보이는 증상 차단(정통 테트리스 top-out 정의 정렬). 라인 클리어/스폰 모두 스킵. 실기 재현 검증 대기.

(공통 미해결은 루트 `PROGRESS.md` 4장 참조.)
