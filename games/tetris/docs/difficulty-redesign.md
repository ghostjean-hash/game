# 난이도 재설계 기획서 (Tetris)

작성일: 2026-05-08
상태: 1차 적용 대상 / 2차 후속

# 1. 문제 정의

1.1. 현행 사양은 정통 테트리스 가이드라인을 따르며 레벨당 지수적 가속 + 영구 가속 구조다.
1.2. 모바일 환경(드래그/탭)에서는 PC 대비 입력 정밀도가 낮다. 같은 속도라도 체감 난이도가 30~50% 높다.
1.3. 결과: 후반 레벨은 모바일에서 사실상 사망 강제 구간이 된다. "오래 버티면 결국 죽는다"는 아케이드 시대 설계가 캐주얼 모바일과 충돌한다.

# 2. 설계 원칙

2.1. 모바일을 1순위로 본다. PC는 호환 환경이지 우선 환경이 아니다.
2.2. "오래 버틸수록 즐겁다"가 기본. "오래 버티면 죽는다"는 선택 가능한 옵션이지 기본값이 아니다.
2.3. 가속은 긴장감 도구지 종료 도구가 아니다.
2.4. 점수 인플레와 속도 인플레를 분리한다. 점수는 무한, 속도는 캡.

# 3. 1차 패치: 마라톤 단일 모드 보정

현 단일 모드를 "마라톤"으로 명명하고 다음 보정을 적용한다.

## 3.1. 속도 곡선

레벨별 한 라인 낙하 시간(초). 캡 적용.

| 레벨 | 현행(가이드라인) | 1차 패치 | 비고 |
|---|---|---|---|
| 1 | 1.000 | 1.000 | 동일 |
| 2 | 0.793 | 0.850 | 완만화 |
| 3 | 0.618 | 0.720 | 완만화 |
| 4 | 0.473 | 0.600 | 완만화 |
| 5 | 0.355 | 0.490 | 완만화 |
| 6 | 0.262 | 0.390 | 완만화 |
| 7 | 0.190 | 0.310 | 완만화 |
| 8 | 0.135 | 0.260 | 완만화 |
| 9 | 0.094 | 0.220 | 완만화 |
| 10 | 0.064 | 0.200 | 모바일 캡 |
| 11+ | 더 빨라짐 | 0.200 고정 | 캡 유지, 점수만 누적 |

3.1.1. 모바일은 레벨 10에서 캡(0.200초/라인).
3.1.2. PC는 레벨 12까지 가속 허용 후 캡(0.150초/라인). 환경 분기 적용.
3.1.3. 후반부 곡선(레벨 7~10) 가속 폭은 현행의 절반 수준으로 평탄화.

## 3.2. 모바일 시작 보정

3.2.1. 모바일에서는 시작 레벨이 PC와 동일하지만 첫 3레벨 구간 속도를 한 단계 낮춘 값으로 시작. 입력 적응 시간 확보.
3.2.2. 시작 레벨 기준값은 표 3.1의 값을 기준으로 하되 모바일은 레벨 1~3에서 +0.10초 추가.

## 3.3. 점수

3.3.1. 캡 도달 후에도 레벨 표기와 점수는 계속 증가. "버틴 시간 자체가 점수"가 되도록.
3.3.2. 점수 공식은 현행 유지([100/300/500/800] × 레벨).
3.3.3. 캡 도달 시점에 화면 어딘가에 작은 표시 1회(세션당 1회). 카피: "리듬에 들어왔다." 해당 세션 내에서는 이후 노출 없음. 다음 세션 진입 후 캡 재도달 시 다시 1회 노출.

# 4. 2차 패치: 모드 분리

마라톤 외 두 모드 추가. 메인 화면에서 모드 선택 후 진입.

## 4.1. 모드 목록

| 모드 | 가속 | 종료 조건 | 점수 기준 | 주 타깃 |
|---|---|---|---|---|
| 마라톤 | 레벨 10/12에서 캡 | 토핑아웃 | 라인 점수 누적 | 일반/긴 세션 |
| 젠 | 없음(고정) | 사용자 종료 | 라인+콤보+T-spin | 모바일/이완 |
| 스프린트 | 없음(고정 중간 속도) | 40라인 클리어 | 클리어 시간(짧을수록 우수) | 숙련/타임어택 |

## 4.2. 젠 모드 사양

4.2.1. 낙하 속도는 0.500초/라인 고정. 레벨 개념 없음.
4.2.2. 게임 오버 외 종료 조건 없음. 일시정지/저장 없이 그냥 그만두면 됨.
4.2.3. 점수는 라인 클리어 + 콤보 + T-spin 보너스만. 레벨 곱연산 없음.
4.2.4. 모바일 메인 화면에서 [추천] 뱃지 노출.

## 4.3. 스프린트 모드 사양

4.3.1. 낙하 속도는 0.300초/라인 고정.
4.3.2. 40라인 클리어 시 종료. 시간 기록.
4.3.3. 베스트 타임 별도 저장. 마라톤/젠 점수와 분리.

## 4.4. 메인 화면 흐름

```
[ 마라톤 ]   라인을 지워라. 끝까지.
[ 젠 ]       가속 없음. 호흡대로.        (모바일: 추천 뱃지)
[ 스프린트 ] 40라인. 최단 시간.
```

4.4.1. 카드 형태. CLAUDE.md 표준 톤(사각 미세 라운드, 원형/pill 금지) 유지.
4.4.2. 진입 시 마지막 선택 모드 기억. 다음 진입 시 해당 모드 카드 강조.
4.4.3. 게임 오버 후 "다시 시작"은 같은 모드 재시작이 아니라 모드 선택 화면으로 복귀한다. 매 세션마다 사용자가 모드를 의식적으로 고른다. 마지막 선택 모드는 카드 강조로 빠른 재선택 지원.

## 4.5. HUD 표기

4.5.1. 인게임 HUD 좌상단 또는 topbar 끝에 모드명 표기. 폰트 크기 작게.
4.5.2. 마라톤은 레벨/점수, 젠은 라인 수/점수, 스프린트는 남은 라인/경과 시간.

## 4.6. 베스트 점수 정책

4.6.1. 모드 분리 적용 시점에 기존 단일 베스트 점수는 폐기. 마이그레이션 없음.
4.6.2. 마라톤은 점수 기준, 젠은 점수 기준, 스프린트는 시간 기준(짧을수록 우수)으로 각각 분리 저장.
4.6.3. 폐기 시점 1회 안내. 카피: "새 모드와 함께 점수도 새로 시작한다." 안내 후 사용자 확인 1회로 종료.
4.6.4. 안내 직후부터 모드별 베스트는 각자 누적. 모드 간 점수 비교 의미 없음을 화면 카피에서 한 번 더 명시.

# 5. 카피 / 톤 가이드

5.1. 마라톤: "라인을 지워라. 끝까지."
5.2. 젠: "가속 없음. 호흡대로."
5.3. 스프린트: "40라인. 최단 시간."
5.4. 캡 진입 안내(마라톤): "리듬에 들어왔다."
5.5. 톤 원칙: 짧게. 명령문 또는 단언문. 감정 과잉 금지. "최고" "최강" "도전" 같은 단어 금지.

# 6. 적용 순서

6.1. 1차(즉시): 3장 마라톤 보정. 곡선 + 캡 + 모바일 보정. UI는 현행 유지.
6.2. 2차(별도 작업 단위): 4장 모드 분리. 메인 화면 개편 동반.
6.3. 카피/톤: 1차에서 마라톤 카피 적용. 모드 카피는 2차에서.

# 7. 검증 기준

7.1. 모바일 실기기에서 레벨 10 도달 후 30초 이상 추가 플레이 가능해야 한다(현행은 사실상 불가).
7.2. PC에서 레벨 12 도달 후에도 의도된 사망이 아니라 실수로 사망해야 한다.
7.3. 젠 모드는 "30분 연속 플레이"가 자연스러워야 한다.
7.4. 스프린트 모드는 숙련자가 60~120초 범위에 도달하는지 확인.

8.3. 베스트 점수: **모드 분리 시점에 기존 단일 베스트 폐기, 모드별 분리 저장, 마이그레이션 없음** (4.6 반영). 결정 2026-05-08.
