# 디오라마 도시 리스트 (ChatGPT 이미지 작업용)

세계 여행 스테이지 배경 = 나라별 프리렌더 2.5D 디오라마 1장. **나라 상징이 아니라 수도(그릴 도시) 기준**으로 그린다.
이 문서는 ChatGPT에 그대로 복사해 넘기는 작업 지시서다(코드는 참조하지 않으므로 필요한 규격을 이 문서 안에 모두 담았다).

기존 `src/data/backgroundPools.js`의 나라별 오버레이(후지산·만리장성 등 나라 상징) 방식은 폐기한다. 수도 기준 디오라마로 일원화한다.

## 1. ChatGPT에 넘기는 법

1.1. 이 문서 전체를 붙여넣는다.
1.2. **이미 완성한 `KR-seoul.png`(서울)를 화풍 레퍼런스 이미지로 함께 첨부**한다. "이 그림과 같은 화풍·같은 카메라 각도·같은 여정 구조로, 아래 표의 각 수도를 그려달라"고 지시한다.
1.3. 한 번에 한 도시씩 뽑는 게 품질이 안정적이다. 표의 "그릴 도시"와 "랜드마크"를 그 도시 것만 골라 준다.

## 2. 공통 규격 (모든 도시 동일)

- **형식**: 세로로 긴 1장 이미지(위아래로 긴 세로 구도). 게임 화면 세로보다 길게 그려 위로 스크롤할 여백을 만든다.
- **시점**: 비스듬한 항공 조감(2.5D 디오라마, 미니어처를 위에서 45도쯤 내려다보는 느낌). 정면 사진 금지.
- **여정 구성**: 화면 **하단 = 도시 출발점**(랜드마크 밀집), 위로 갈수록 도시 외곽 → 강·길 → **상단 = 관문/항구(여정의 끝, 보스 무대)**. 서울 그림의 "경복궁(하단) → 한강 → 항구(상단)" 흐름과 동일한 구조.
- **색**: 저채도·저대비·차분한 톤. 강한 흰/검/네온 금지(그 위에서 총알·적이 잘 보여야 함).
- **비우기**: 세로 중앙 띠(화면 중앙 약 60%)는 하늘·강·열린 공간으로 비운다. 랜드마크·건물 실루엣은 화면 좌우 가장자리와 하단에 배치(전투 가독성).
- **랜드마크 표현**: 사실적 정면 묘사가 아니라 상징적 실루엣·단순화. 원경 중심.
- **통일감**: 모든 도시가 같은 화풍·같은 각도·같은 여정 구조. 도시 차이는 랜드마크·지형·포인트색으로만.

## 3. 파일명 규칙

- 폴더 없이 `assets/diorama/` 바로 아래. 파일명 = `국가코드(대문자)-도시명(소문자).png`.
- 예: `KR-seoul.png`, `JP-tokyo.png`, `CN-beijing.png`.
- 여행지(발리·하와이)는 소속 나라 코드 + 지명: `ID-bali.png`, `US-hawaii.png`.
- `webp` 최적화본은 배포 때 자동 생성되므로 직접 만들 필요 없다(원본 `png`만 넣으면 됨).

## 6. 참고 (코드 연결)

- 완성한 `png`를 `assets/diorama/`에 넣으면, `src/render/view.js`의 `DIORAMA_SRC`에 `'나라이름': '경로'` 한 줄만 추가하면 게임에 붙는다(자비스가 처리).
- 디오라마가 붙은 나라는 그 이미지가 배경 최우선. 아직 안 만든 나라는 우주 배경으로 나온다.
- 나라 이름 키는 `src/data/countries.js`의 `ko` 값과 일치해야 한다(예: `'한국'`, `'일본'`).
