# 01 게임 사양 (Sky Raider)

## 1. 한 줄

세로(종) 스크롤 캐주얼 비행 슈팅. 플레이어는 화면 하단, 적은 위에서 내려온다. 30개 구역의 보스를 격파하면 클리어(11구역부터 신규 적, 21구역부터 완전 다른 형태의 이질 기계 적 - 상세 docs/08).

## 2. 조작

- **난이도 모드(시작 화면 선택)**: 메뉴에서 **일반 모드**와 **어린이 모드** 중 골라 시작한다. 어린이 모드는 (1) 적·보스의 발사 간격을 늘려(`CFG.difficulty.kid.enemyFireMul`) 총알을 덜 쏘게 하고(탄 수·패턴은 동일, 빈도만↓), (2) 메인 총알 2발·꼬리 비행기 1대를 갖춘 채 출발한다(`startFront`/`startTail`, 더 쉽게 시작). 선택은 localStorage에 기억되고 '자동 플레이로 시작'도 마지막 선택 모드를 따른다.
- **이동**: 화면 드래그(상대 이동 - 잡은 지점 대비 델타를 기체에 적용, 손가락 가림 방지). PC는 방향키/WASD.
- **발사**: 자동(0.14초 간격). 정면(위쪽) 기준 화력 단계별 부채.
- **일시정지**: ⏸ 버튼 또는 P 키. 일시정지 중 화면 탭으로 재개.
- **전체화면**: 지원 기기에서 ⛶ 버튼(미지원 시 숨김).
- **자동 플레이**: 게임 중 🤖 버튼으로 켜고 끄거나, 메뉴의 '자동 플레이로 시작' 버튼으로 처음부터 자동 진행한다. 기체가 스스로 적·적탄을 피하고 파워업을 주우며 진행한다(발사는 자동). 데모·밸런스 체감·검증용. 로직은 `core/autopilot.js`. **실력 4단계**(초보/중수/고수/프로, 환경설정에서 선택, 기본 초보) - 반응 지연·조준 오차·예측 지평·동시 고려 위협 수를 인간 실측 기준으로 조절해 사람처럼 움직인다(근거 docs/plans/research-2026-07-09).
- **환경설정·치트**: 메뉴 '환경설정'에서 자동 플레이 실력 선택 + 치트 박스 on/off. 치트 박스(드래그 이동·접기)는 플레이 속도(×1~×8)·무적·아이템 드랍 확률·종류를 조절한다(개발·테스트용). 치트 세부 설정은 localStorage(`cheatCfg`)에 저장돼 재시작·재접속에도 유지되고, 치트 박스가 열릴 때 `syncCheatUI`로 저장값이 UI에 반영된다(사용자 지시 2026-07-10).

## 3. 코어 루프

웨이브(적 무리)를 처치하며 전진 → 웨이브 소진 후 보스 등장 → 보스 격파 시 다음 구역. 목숨 3개가 모두 소진되면 게임오버. 구역이 바뀌면 '구역 N' 배너가 잠시(`stageIntro`) 뜨고 그동안 적이 나오지 않는다(전환 숨돌림).

## 4. 파워 파츠 (4계통)

파워업이 네 갈래로 분리된다. 각 계통은 전용 아이템으로만 성장하고, 피격 시 마지막 얻은 파츠 1개를 잃는다(역순 손실). 모든 발사체는 모양·크기가 커질수록 데미지가 오른다(에너지존만 예외). 상세 설계 → 05_power-parts.md.

- **전방 화력(P)**: 정면 부채탄. 1~8단계는 탄 +1(최대 8발), **개별 탄 크기·모양·색 전부 동일**(성장은 탄 수로만). 9단계부터는 8발 고정, 가운데 탄부터 한 발씩 진화(4티어, 8발 다 바뀌면 다음 티어) - 진화 탄은 아군 차가운색(시안·흰·하늘·민트) + 밝기·글로우 계단으로 즉시 구분되고(적탄과 안 겹침) 데미지 조금↑. 최대 40단계. `core/fire.js`의 `frontSpec(front)`.
- **옵션기(S)**: 좌우에 붙는 부속 비행기. S 획득당 좌우 번갈아 1대씩, 최대 8대(좌4·우4). **8대 전부 레이저**(빠른 직진탄), 옵션 수가 많을수록 굵고 세진다. `core/parts.js`.
- **에너지존(E)**: 플레이어 중심에서 퍼지는 펄스파(파동 링). 레벨 0~5. 링이 주기적으로 바깥으로 퍼지고, **링이 지나가는 순간 링 위에 있는 적·보스에게만** 레벨만큼 피해(상시 장판 아님). 레벨↑ 파동이 더 자주·멀리·두껍게·세게. `core/parts.js`.
- **꼬리 비행기(T)**: 플레이어 뒤에 붙는 유도탄 발사기, 최대 4대(처음부터 유도탄). 전용 아이템 T로 성장 - 먼저 대수(1~4대), 4대 후 1번→2번→3번→4번 꼬리기 순서로 무기 한 단계씩 진화(무기 4단계). 단계 클수록 유도탄이 커지고 데미지↑. 유도탄이 가장 가까운 적을 추적. `core/parts.js`.

## 5. 적

기본 적(1~10구역 주력):
- **drone**: 직진 하강. hp 1. 발사 없음.
- **weaver**: 좌우 사인파로 흔들며 하강. hp 2. 가끔 단발 조준 발사.
- **gunner**: 느리게 하강하며 플레이어 x를 추적, 3발 확산 조준 발사. hp 3.

신규 적(11~20구역 주력, "전혀 다른 적"):
- **splitter(분열체)**: 직하하다 격파되면 조각(shard)으로 쪼개져 좌우로 흩어진다. shard는 작고 빠르며 재분열하지 않는다.
- **shielder(방패병)**: 정면(아래)을 향한 방패로 기본탄·레이저를 막는다. 꼬리 비행기의 유도탄(측면 우회)이나 에너지존(범위)으로만 피해를 줄 수 있어 그쪽 성장을 유도. 단발 조준 발사.
- **rusher(돌격기)**: 잠시 조준하며 천천히 내려오다, 플레이어 방향으로 급강하 돌진한다.

이질 기계 적(21~30구역 주력, "완전 다른 형태" - 각진 기계·결정 계열, 상세 docs/08):
- **turret(부유 포대)**: 거의 정지한 채 포신으로 3방향 조준 연사. 단단하다.
- **prism(결정체)**: 직하하되 피격당할 때마다 사방으로 반사탄을 튕긴다.
- **mine(기뢰)**: 느리게 표류하다 플레이어가 근접하면 자폭(범위 파편탄) - 정면 돌파를 벌한다.
- **warper(공간 왜곡체)**: 짧게 순간이동하며 하강해 위치 예측이 어렵다.

보너스 기체:
- **bonus**: 주기적으로 화면 좌/우에서 등장해 가로질러 간다. 파워업의 주 공급원 - 격파 시 확정 드롭(잡몹은 드롭 없음).

- 적 체력은 구역이 오를수록 상승: `base × (1 + (stage-1)×enemyHpScale)`, 11구역부터 추가 배수(`hardStage.hpMul`). 3계통 화력 성장과 균형.
- 상세 수치 → 02_data.md.

## 6. 보스

- **중보스(1~19구역)**: 작은 정령형 기체. 좌우 유영 + 조준 발사 + 호위 비행체 주기 소환. hp는 구역에 비례. 격파 시 파워업 2개 드롭.
- **기계 중보스(21~29구역)**: 각진 강철 외형(신규). 부채 방사 ↔ 조준 3연발 번갈아(더 조밀) + 기계 잡몹(turret/prism) 호위. 격파 시 파워업 2개.
- **최종보스(30구역)**: 큰 기체. 좌우 유영 + 2패턴(부채 산탄 / 조준 3연발) 번갈아. 고정 hp. 격파 시 파워업 4개 드롭.
- 등장 위치는 상단 체력 바 아래(`boss.spawnTop`)로 내려 바와 겹치지 않는다.

## 7. 파워업 아이템 (6종)

- **P**: 전방 화력 +1 (1~8 탄 수, 9~40 발별 진화). 최대(40) 후엔 점수 보너스.
- **S**: 옵션기 1대 추가 (최대 8, 전부 레이저).
- **E**: 에너지존 레벨 +1 (최대 5).
- **T**: 꼬리 비행기 성장 - 대수 +1(4대까지) → 그 뒤 1~4번 꼬리기 무기 순차 +1(무기 4단계). 최대 후엔 점수 보너스.
- **H**: 목숨 +1(최대 3).
- **B**: 봄 - 화면의 적 전멸 + 보스에 큰 데미지.
- 보스·보너스 기체는 확정 드롭, 일반 잡몹은 낮은 확률(`drop.chance`)로 드롭한다(초반 화력 성장 숨통).
- 피격 시: 목숨 -1 + 마지막 얻은 파츠(P/S/E) 1개 손실.

## 8. 화면 흐름

메뉴 → (시작) → 구역 1 웨이브 → 중보스 → 구역 2 … → 구역 20 최종보스 → 전 구역 격파(승리). 중간 목숨 소진 시 게임오버 모달(다시하기/홈).

## 9. 성공 / 실패 / 종료

- 성공: 20구역 최종보스 격파.
- 실패: 목숨 0.
- 진행 유지: 파워 파츠·목숨·점수는 구역이 넘어가도 유지(초기화는 게임 재시작 시만).
- 점수: 적/보스 처치 누적. localStorage 최고 기록(`gg.flightshooting.best`).
