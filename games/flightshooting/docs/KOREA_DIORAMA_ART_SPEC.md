## 1. 첫 실제 아트 테스트 범위

## 2. 공통 이미지 규격 [프로토타입 기준]

- 청크 원본 1080 × 2048px, 청크 수 2개, 불투명 WebP, 시작 품질 q80, 오버랩 128px, sRGB, 일반 청크 알파 미사용.
- 논리 게임 화면 720 × 1280. 진행 방향 이미지 아래→위. 카메라 Roll 0.
- **두 청크 완전 동일**: 카메라 각도 / 광원 방향 / 건축물 축척.

**원본 관리**: 최종 WebP만 보관하지 않는다 - 고품질 마스터, 레이어 분리 편집본, 경복궁 분리 원본, 오버랩 처리 전/후, 중앙 보호 오버레이본을 유지(후속 수정·재출력 가능해야 함). 원본 포맷은 제작 도구에 맞춰 선택.

## 3. 카메라와 원근

### [아트 고정 규칙]
- 세로 종스크롤용 비스듬한 탑다운. 수평선·넓은 하늘 안 보임. 지면 앞뒤 깊이 명확.
- 건물 정면이 과하게 보이는 측면 시점 금지. 지붕만 보이는 수직 탑다운 금지.
- 모든 수직 구조물 동일 투영. 청크마다 소실 방향 불변. 건물·산·수목·도로 축척이 진행 방향에 따라 임의로 변하지 않음.
- 인트로·플레이는 같은 렌더 소스·같은 각도. 인트로는 거리·포커스만 다르고 장면 자체 동일(별도 그림 아님).

### [아트 테스트 후 확정]
- Pitch는 약 55°를 첫 후보로 쓰되 한 장으로 확정하지 않는다. 제작 전 **저비용 시안 3개 비교**: 후보 A 50° / B 55° / C 60°. 판정 = 미니어처 입체감·건물 실루엣 인식·중앙 가림 정도·세로 스크롤 자연스러움·경복궁 인트로 클로즈업 품질·현대 도시까지 동일 시점 유지 가능성. 시안은 저해상·블록아웃 가능.

## 4. 디오라마 스타일 [아트 고정 규칙]

목표: 고품질 프리렌더 2.5D 미니어처 디오라마 / 통일된 축척 / 재질을 게임에 맞게 단순화 / 형태·실루엣 우선 / 큰 형태 중심(미세 디테일 과다 지양) / 한 국가 안 전통·현대가 같은 렌더 세계에.
금지: 실사 위성사진 / 사진 콜라주 / 평면 관광지도 / 기계적 아이소메트릭 격자 / 저폴리 장난감 블록 / 플라스틱 피규어 질감 / 과도한 셀셰이딩 / 과도한 동화책풍 / GIS식 실축척 재현 / 랜드마크만 거대한 관광 포스터식 / 건물과 산이 다른 화풍인 혼합 스타일.

## 5. 축척 규칙 [아트 고정 규칙]

국가 전체를 실축척으로 재현하지 않는다. 단 한 화면 안에서 경복궁·한옥·현대 건물·수목·도로·차량·강폭·교량·산 언덕의 상대 관계는 일관돼야 한다. 랜드마크는 인식 위해 약간 강조 가능하나 일반 건물 대비 지나치게 거대하게 만들지 않는다. 강조 우선순위: (1) 배치 위치 (2) 주변 여백 (3) 실루엣 (4) 색상 대비 (5) 인트로 포커스 (6) 제한적 크기 강조. 단순 크기 확대는 마지막 수단.

## 6. 중앙 전투 보호 영역

### [프로토타입 기준]
논리 720×1280 기준 보호 영역 X 144~576(중앙 60%). 청크 제작 중 이 영역을 기준 오버레이로 표시. 스크롤 위치별 반복 적용.

### [아트 고정 규칙]
제한 요소: 경복궁 주 건물 / 대형 랜드마크 / 강한 흰 지붕 / 반복되는 밝은 창문 / 작은 원형 조명 / 탄환처럼 보이는 점광원 / 강한 주황·적 점 / 고대비 도로 차선 / 복잡한 지붕 격자 / 강한 반사광 / 매우 밝은 물결 하이라이트 / 적·아이템처럼 보이는 작은 장식.
허용 요소: 낮은 대비 지면 / 단순화 도로 / 톤 억제 일반 건물 / 낮은 수목 / 완만한 그림자 / 큰 면 강·평지 / 전투 오브젝트와 형태가 겹치지 않는 환경 패턴.
**중앙을 완전히 비우는 것이 아니다.** 배경은 존재하되 전투보다 먼저 눈에 들어오면 안 된다.

## 7. 랜드마크 배치 [아트 고정 규칙 + 아트 테스트 후 확정]

경복궁은 중앙 보호 영역 밖. 첫 후보: 좌측형 중심 X 20~32% / 우측형 중심 X 68~80%. **좌우 방향은 실제 시안에서 결정**[아트 테스트 후 확정]. 판정 = 인트로 줌 시 충분히 크게 / 줌아웃 후 중앙 안 가림 / 청크 경계와 안 겹침 / 플레이 중 너무 빨리 화면 밖으로 사라지지 않음 / 포커스 이동이 과도하게 수평으로 흔들리지 않음.
금지 위치: 청크 오버랩 중심 / 화면 정중앙 / 플레이어 기본 이동 중심 / 적 스폰 집중 위치 / UI에 가리는 모서리 / 강한 지형 전환선과 겹치는 위치.

## 8. KR-P01 구성

목적: 경복궁 인트로 줌 검증 / 전통 건축·지형 미니어처 표현 / 랜드마크 레이어 정합 / 전투 보호 영역 / KR-P02 연결부 제공.
필수: 경복궁 핵심 구조(즉시 인식) / 전통 담장·문 / 제한된 한옥 / 수목 / 단순화 도로·보행 공간 / 다음 청크 지형 연결부 / 중앙 보호 영역의 낮은 대비 환경.
제한: 경복궁 전체 단지 실비율 금지 / 건물 수 과다 금지 / 지붕 문양 과세밀 금지 / 관광객·사람 무리 제외 / 차량 제외 가능 / 깃발·조명·입자 제외 / 강한 안개 제외 / 계절 효과 제외 / 산맥 주 피사체 금지.
연결부(KR-P02와 128px 오버랩): 평지·낮은 수목·단순 도로·낮은 담장·반복 가능한 일반 환경만. 경복궁 본체·대형 문·첨탑·고층 현대 건물·강·교량·강한 그림자 경계·큰 색상 변화는 배치 금지.

## 9. KR-P02 구성

목적: 전통→현대 전환 검증 / 서로 다른 건축 양식의 일관성 / KR-P01 경계 검증 / 전투 가독성.
필수: KR-P01에서 이어지는 도로·지형 축 / 낮은 전통 건물 일부 / 중층 현대 건물 / 제한된 고층 / 공원·녹지 완충 / 동일 수목 축척 / 동일 태양·그림자 방향.
전환: 전통이 갑자기 고층으로 바뀌지 않음. 권장 순서 = (1) 전통 담장·낮은 건물 (2) 저층 도시 (3) 중층 현대 (4) 제한된 고층 실루엣. 한강은 필수 아님. 넣더라도 청크 전체를 가르는 거대 밝은 수면 금지, 가독성 방해 없는 짧은 구간만.

## 10. 경복궁 고해상도 레이어

### [프로토타입 기준]
별도 고해상 알파 레이어 / 배경 청크와 동일 월드 좌표·동일 `w2s` 변환 / 인트로 2.4x 확대 대응 / 청크와 **동일 렌더 소스에서 분리**(다른 생성 결과 덧붙임 금지).

### 제작 규칙
포함: 경복궁 핵심 건축물 / 정합용 최소 주변 지면 / 건물 직결 그림자 / 겹침 순서용 일부 수목·담장.
제외: 넓은 하늘 / 청크 전체 지면 / 불필요한 투명 여백 / 무관한 일반 건물 / 별도 색보정 / 별도 조명 / 인트로용 추가 장식 / 별도 안개 / 별도 그림자 방향.

### 선명도 규칙
2.4x 확대에서 지붕선·주요 윤곽 유지. 플레이 상태에서 주변 청크보다 과하게 선명하지 않음. 필요 시 플레이 배율에서 약한 선명도 완화 또는 배경 합성본 전환 검토(**전환 필요 여부는 첫 아트 테스트 후 결정**).

### 포맷 [아트 테스트 후 확정]
첫 후보 우선 알파 WebP / 비교용 PNG. 동일 에셋 두 포맷 출력해 파일 크기·디코딩 시간·가장자리 품질·premultiply 문제·반투명 그림자 품질·모바일 메모리 비교. **최종 포맷은 테스트 후 확정.**

### 유효 해상도 [프로토타입 기준]
논리 표시폭 × 2.4(줌) × dpr 2 ≈ 유효 픽셀(예 표시폭 300 → ≈1440px). 투명 여백 포함 최대 ≈1600px 정사각. 앵커 = 하단 중앙(지면 접점), 정합 기준점 = 배경 청크 지면 라인 y와 일치.

## 11. 조명과 색상 [아트 고정 규칙]

부드러운 주간광 / 태양·그림자 방향·강도 고정 / 정오 과노출 금지 / 강한 역광 금지 / 야간 금지 / 석양 강한 주황 전체 틴트 금지 / 청크마다 다른 시간대 금지 / 인트로만 별도 색보정 금지 / 연결부를 안개로 숨기기 금지.
권장 시작: 중립적 맑은 날 / 약간 차분한 색온도 / 지붕·수목 고유색 식별 / 흰 영역은 순백보다 낮은 명도 / 그림자 검게 뭉개지지 않게 / 현대 유리는 탄환처럼 번쩍이지 않게.

## 12. 전투 가독성 기준 [실제 아트 검수]

실제 아트 적용 후 캡처: (1) 배경만 (2) 플레이어만 (3) 일반 적 (4) 적 탄환 (5) 플레이어 탄환 (6) 이펙트 겹친 전투 (7) 인트로 2.4x (8) 줌 중간 (9) 플레이 1.0x (10) 청크 오버랩 통과.
검수: 플레이어 즉시 인식 / 적 실루엣이 배경 건물과 안 섞임 / 적 탄환이 배경 조명과 혼동 안 됨 / 플레이어 탄환이 흰 지붕 위에서도 보임 / 아이템이 수목·창문과 혼동 안 됨 / 경복궁이 전투보다 먼저 시선 뺏는 시간이 과도하지 않음.
대응 우선순위: (1) 아트 배치 조정 (2) 배경 명도·채도 조정 (3) 디테일 정리 (4) 전투 오브젝트 외곽선·글로우 조정 (5) 마지막으로 약한 런타임 tint 검토. **전체 배경을 무조건 어둡게 만들기 금지.**
측정 방법[제안]: 중앙 보호 영역 픽셀 평균 휘도(0.2126R+0.7152G+0.0722B)·표준편차 산출, 전투 오브젝트 휘도와의 차 ΔL ≥ 0.3 기준 미달 구간 식별.

## 13. 청크 연결 검수 [실제 아트 검수]

오버랩은 단순 복제가 아니라 **의도적 완충 영역**. 검수: 도로 중심선·담장 위치·수목 밀도·그림자 방향·지면 명도·색온도·건물 축척·소실 방향 일치, 경계 밝기 밴드 발생 여부, 블렌드 중 이중 구조물, 스크롤 중 형태 녹음 여부.
밝기 밴드 대응 순서: (1) 알파 베이크 계산 검토 (2) 오버랩 원본 명도 통일 (3) 중복 그림자 제거 (4) premultiply 방식 검토 (5) 필요 시 오버랩 폭 조정. **안개·파티클로 가리기 금지.**
[실측] 프로토타입에서 오버랩 alpha 사전 베이크로 런타임 비용 0에 크로스페이드 동작 확인. placeholder에서 색 대비가 커 밴드가 약간 밝게 보였음 → 실제 아트 명도 통일로 재확인 대상.
