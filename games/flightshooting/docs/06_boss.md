# 06. 보스 설계안 (부위 파괴형 + 다중 스타일)

현재 보스는 중보스 1종·최종보스 1종의 **단일 몸체 + hp 바 하나** 구조다(01_spec 6장). 이 문서는 사용자 지시(2026-07-09)로 보스를 **부위 파괴형**으로 바꾸고 **여러 스타일**을 만드는 설계안이다. 아직 **설계 단계**이며(구현은 확정 후 착수), 이 문서가 구현의 SSOT가 된다.

## 0. 왜 바꾸나 (문제 인식)

- 지금 보스는 모습이 딱 2가지(중보스/최종보스)뿐이라 1~19구역 중보스가 전부 똑같이 생겼다. 19번 반복되는 같은 얼굴이 단조롭다.
- hp 바 하나를 0까지 깎는 단순 구조라, 화력이 세지는 후반엔 "그냥 오래 쏘면 죽는" 소모전이 된다. 플레이어가 **어디를 노려야 하는지**가 없다.
- 부위 파괴형은 (1) 노릴 표적을 주고 (2) 부위를 깰 때마다 보스의 공격이 약해지는 체감(공략의 진행감)을 주며 (3) 스타일별로 부위 구성을 달리해 **같은 시스템으로 여러 보스**를 뽑아낸다.

## 1. 부위 파괴형 - 핵심 규칙

보스는 **코어(core) 1개 + 부위(part) 여러 개**로 구성한다.

1. **부위(part)**: 각자 독립 hp를 가진다. 위치는 코어 기준 오프셋(`ox, oy`). 역할은 둘 중 하나.
   - `weapon`(포탑): 자기 공격 패턴을 가진다. 파괴되면 **그 패턴이 영구 정지**한다(보스 화력이 단계적으로 약해짐 - 공략의 보상).
   - `shield`(방어구): 공격은 안 하지만 **코어를 가린다**. 살아있는 동안 코어는 피해를 받지 않는다.
2. **코어(core)**: 보스의 본체 hp. `exposed`(노출) 상태일 때만 피해를 받는다.
   - `shield` 역할 부위가 **모두 파괴되면** `core.exposed = true`가 된다(방어구 없는 스타일은 처음부터 노출).
   - 코어 hp가 0이 되면 보스 격파(기존 `defeatBoss`).
3. **타격 우선순위**: 아군 발사체는 겹친 대상 중 **살아있는 부위 → (노출된) 코어** 순으로 1개에만 피해를 준다. 부위가 코어 앞을 가리므로, 방어구를 먼저 부수지 않으면 코어를 못 때린다.
   - 예외: 에너지존(범위기)·유도탄(측면 유도)은 부위와 무관하게 사거리 내 가장 가까운 대상을 때린다 - 기존 적(shielder) 공략과 일관. 정면 화력(메인/사이드)만 위 가림 규칙을 받는다.
4. **부위 파괴 연출**: 부위 hp 0 → `dead=true` + 그 자리에 폭발 파티클 + 잔해가 잠깐 남는다. 부위는 사라지고 그 패턴은 멈춘다.
5. **hp 표시**: 상단 바는 **코어 hp**만 표시하되, 방어구가 남아 있으면 바를 "보호 중"으로 흐리게(또는 자물쇠 아이콘) 보여 "아직 코어를 못 때린다"를 알린다. 부위별 hp는 각 부위 위에 작은 게이지.

## 2. 데이터 구조 (구현 가이드)

```
boss = {
  style: 'battleship' | 'bio' | 'orbiter' | 'sentinel',
  kind:  'mini' | 'final',           // 중보스/최종보스 (드롭 수·hp 규모 구분, 기존 유지)
  x, y, rx, ry, color, t,
  entering: bool, targetY,           // 등장 이동(기존)
  core:  { hp, maxHp, exposed: bool },
  parts: [
    { id, ox, oy, r, hp, maxHp, dead:false,
      role: 'weapon' | 'shield',
      pattern: 'fan' | 'aim3' | 'ring' | null,  // weapon만
      fireTimer, color, shape },      // shape = 렌더 형태(포탑/촉수/조각/날개)
  ],
}
```

- `parts`가 비거나 shield가 없으면 `core.exposed` 초기값 true.
- 부위 위치는 코어 유영(`bob`)에 함께 딸려 움직인다(오프셋 고정, 절대 좌표는 매 프레임 `boss.x/y + ox/oy`).
- 스타일별 부위 구성·패턴·색은 `data/bosses.js`(신설) 또는 `numbers.js`의 `bossStyles` 테이블에서 정의(매직넘버 0 원칙).

## 3. 보스 스타일 (4종 제안)

같은 부위 시스템으로 외형·부위 구성·공략 포인트를 달리한 4종. 구역대별로 배정해 "이 구역 보스는 처음 보는 모습"을 만든다.

### 3.1. 함선형 (battleship)
- **구성**: 중앙 코어 + 좌우 **포탑 2개(weapon)**. 방어구 없음(코어 처음부터 노출).
- **패턴**: 좌 포탑 = 아래 방향 부채 산탄(`fan`), 우 포탑 = 조준 3연발(`aim3`). 코어는 가끔 느린 유도탄.
- **공략**: 포탑을 먼저 부수면 탄막이 확 줄어 코어전이 쉬워진다. 가장 직관적 - **입문용(초반 구역)**.

### 3.2. 생체형 (bio)
- **구성**: 중앙 **눈 코어** + 이를 감싼 **촉수 팔 3~4개(shield)**. 촉수가 코어를 가린다.
- **패턴**: 촉수가 번갈아 휘둘러 짧은 근접 판정 + 코어 눈이 조준 레이저 예고 후 발사. 촉수는 shield라 공격 약하고 방어 위주.
- **공략**: 촉수(방어구)를 다 부숴야 코어 눈이 노출된다. 방어구 철거형 - **중반 구역**.

### 3.3. 위성형 (orbiter)
- **구성**: 코어 + 코어 주위를 **회전하는 실드 조각 4~6개(shield)**. 조각이 원을 그리며 돌아 코어를 간헐적으로 가린다.
- **패턴**: 조각이 도는 틈으로만 코어가 잠깐 열린다(타이밍 사격). 조각 각각이 가끔 바깥으로 탄 발사(약한 weapon 겸용도 가능).
- **공략**: 조각을 하나씩 떼어내면 코어 노출 시간이 늘어난다. 타이밍·집중 사격형 - **후반 구역**.

### 3.4. 파수꾼형 (sentinel, 최종보스급)
- **구성**: 큰 몸체에 **머리(weapon)·양 어깨 포(weapon 2)·가슴 방어판(shield)** + 코어. 부위 5개.
- **패턴**: 부위별 다른 패턴(머리 링 탄막 `ring` / 어깨 부채·조준 / 방어판은 코어 보호). 부위를 부술수록 남은 부위가 격화(광폭화)된다.
- **공략**: 방어판 → 코어. 부위 많고 광폭화까지 있는 종합형 - **최종보스(20구역)**.

## 4. 구역 배정 (확정 - 30구역 기준 2026-07-10)

| 구역 | kind | style | 비고 |
|---|---|---|---|
| 1~10 | mini | battleship | 입문 - 좌우 포탑 파괴 학습(방어구 없음) |
| 11~20 | mini | bio | 방어구(촉수) 철거 도입 |
| 21~29 | mini | orbiter | 회전 실드 - 타이밍 사격(기계 강철 톤) |
| 30 | final | sentinel | 종합·광폭화 |

- 한 style을 여러 구역이 공유하되 hp·부위 hp·색조를 구역에 비례해 키운다(같은 모습이어도 1구역 battleship < 10구역).
- 색조: 1~20은 정령 톤(따뜻/차가운 파스텔), 21~29 orbiter는 기존 기계 중보스 강철 톤(강철 청 + 붉은 코어)을 이어 심우주 이질감 유지.

## 5. 바뀌는 파일 (구현 단계, 착수 시)

- `data/numbers.js` - `bossStyles` 테이블(스타일별 부위 오프셋·hp 비율·패턴·반경), 코어/부위 hp 산출 계수. `miniBoss`/`finalBoss`를 스타일 참조로 재구성.
- `data/colors.js` - `boss.styles`(스타일별 코어·부위·포탑 색), 부위 파괴 잔해 색.
- `core/spawn.js` - `spawnBoss`가 구역→style 매핑으로 `core`+`parts` 조립.
- `core/world.js` - `updateBoss`를 부위별 패턴 구동으로, 충돌 판정을 **부위 우선 → 코어(노출 시)**로, `defeatBoss`는 코어 0에서. shield 전멸 시 `exposed` 갱신.
- `render/view.js` - `drawBoss`를 스타일별 외형 + 부위(포탑/촉수/조각/날개) + 부위별 미니 hp 게이지 + 파괴 잔해 렌더로 확장.
- `main.js` / HUD - 코어 hp 바에 "보호 중(방어구 남음)" 표시.
- `tests/` - 부위 우선 타격(방어구 살아있으면 코어 hp 불변), shield 전멸 후 코어 노출, weapon 파괴 시 패턴 정지, 코어 0에서 격파.
- `docs/01_spec.md` 6장 - 부위 파괴형으로 갱신, `02_data.md` - bossStyles 스키마 추가.

## 6. 확정 (구현 2026-07-10, 사용자 "전면 개편" 선택)

미확정 5건을 아래로 확정하고 구현한다.

1. **스타일 4종 그대로**: battleship(포탑 2)·bio(촉수 방어구 3)·orbiter(회전 실드 4)·sentinel(부위 5). 역할·개수 유지.
2. **구역 배정**: 4장 표 확정(1~10 battleship / 11~20 bio / 21~29 orbiter / 30 sentinel).
3. **광폭화 채택**: sentinel(최종)만. 부위 하나 깰 때마다 남은 weapon 부위 발사 주기가 짧아진다(`enrage`).
4. **부위별 미니 hp 게이지 노출**: 각 살아있는 부위 위에 작은 바. 상단 코어 바는 방어구(shield)가 남아있으면 "보호 중"으로 흐리게.
5. **부위 파괴 시 부분 점수 채택, 부분 드롭 없음**: 부위를 깨면 점수(`partScore`)를 준다. 아이템 드롭은 기존대로 코어 격파에서만.

**hp 예산**: 총 hp(기존 miniBoss/finalBoss 공식)를 코어와 부위에 비율(`coreRatio` + 부위 `hpRatio` 합 = 1)로 나눈다. 부위를 다 부수는 총 타격량은 기존 단일 hp와 비슷하게 유지(순삭·과다 방지, dev 훅 실측 후 미세 조정).

**타격 규칙(1장 재확인)**: 정면 화력(메인·사이드)은 살아있는 부위 → (노출된)코어 순으로 겹친 1개만 때린다. 방어구(shield)가 살아있으면 코어는 무피해. 에너지존·유도탄은 부위 무시하고 사거리 내 대상 직격(기존 shielder 공략과 일관).

## 7. 강화판 (2026-07-16, 사용자 지시)

각 10구역 묶음의 **뒤 5개 구역**(6~10 · 16~20 · 26~29)은 앞 5개(1~5 · 11~15 · 21~25)의 **강화판**이다. 스타일(함선/생체/위성)은 같되 다음을 강화한다.

1. **부위 추가**(`bossStyles.<style>.upgrade.extraParts`): 함선형 = 상단 포탑 1개(ring), 생체형 = 아래쪽 무기 촉수 1개(fan), 위성형 = 상단 포탑 1개(fan). 겉모습이 더 위협적이고 화력이 는다.
2. **발사 주기 단축**: 부위 발사 주기 ×`fireMul`(0.7), 코어 자체 공격 주기 ×`coreMul`(0.7) - 더 자주 쏜다(코어 주기는 강화판만 조절되게 `boss.coreEvery`에 저장).
3. **체력 정규화**: 부위가 늘어도 총 hp가 커지지 않게 (코어 + 전체 부위) 비율 합으로 나눠 총량을 유지한다(`spawnBoss`, hpRatio 합이 1을 넘어도 자동 정규화).
4. **판정**: `bossUpgradeFrom`(=5) - `(stage-1) % 10 >= 5`면 강화판. 등장 배너에 "(강화)" 표시. **30(최종 sentinel)은 강화 대상 제외**(isFinal).
