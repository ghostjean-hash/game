# 09. 친구 비행기 (친구 동행 토글)

홈 '친구 동행' 토글로 켜는 동료 비행기 설계. 난이도와 무관하게 어느 난이도든 켤 수 있다(2026-07-16 개편으로 옛 '어린이 모드 전용'에서 독립 토글로 분리). 사용자 지시(2026-07-14~16) 기반. SSOT - 코드보다 이 문서가 진실.

## 1. 개요

1.1. 홈에서 **친구 동행** 토글을 켜고 시작하면 화면 옆에서 친구 비행기가 말풍선으로 인사하며 날아 들어와 함께 싸운다.
1.2. 토글을 끄면 등장하지 않는다(`game.friend` = null). 켬 여부는 localStorage(`friendOn`)에 기억되고, 난이도 선택과 독립이다.
1.3. 목적: 혼자가 아니라 동료와 함께 싸우는 재미 + 옆에서 구경하는 재미(내 것과 완전히 다른 회색 직선 레이저를 쏘는 걸 본다).
1.4. 생김새: **키위새**(통통한 갈색 몸 + 위로 뻗은 긴 부리 + 작은 날개 + 큰 눈). 플레이어(흰 정령)·토토로(회색 옵션기)와 색·형태로 구분(`COLORS.friend.body` 갈색, `beak`).

## 2. 등장 연출

2.1. 어린이 모드 시작 시 화면 안 왼쪽(`startXRatio`)에서 자기 자리로 날아온다. 시작 위치가 화면 밖으로 나가지 않는다(사용자 지시).
2.2. 들어오는 동안(`enterTime`) 말풍선을 순서대로 띄운다: "안녕!" → "난 친구야" → "같이 게임하자!"(`CFG.friend.speech`, 각 `speechEach`초).
2.3. 말풍선은 캔버스에 친구 위쪽 둥근 사각형으로 그린다(HUD 아님). 3개 다 지나가면 사라진다.
2.4. 등장(enter) 중에는 발사하지 않는다. 자리 잡은 뒤 자동발사 시작.
2.5. 인트로 인사가 끝난 뒤에도 플레이 중 상황에 따라 이야기한다(사용자 지시 - 같은 말 반복 금지, 다양한 표현). 상세는 아래 4.5.

## 3. 이동 (플레이어 자동조종과 같은 빔서치 AI, 사용자 지시)

3.1. 플레이어와 무관하게 완전 독립으로 움직인다. 플레이어를 따라다니지 않는다.
3.2. 이동 판단은 **플레이어 자동조종과 같은 로직**(`autopilot.decideTarget`, 여러 수 앞을 보는 빔서치)을 공유한다. 적탄·적을 피하면서 적 아래로 정렬해 조준한다.
3.3. 단, 친구는 무기 발사가 목적이라 두 가지를 조정한다(회피 판단 구조 자체는 플레이어와 동일):
   - `clearWhenSafe: false` - 안전할 때 빈 구석으로 도망가지 않고 적을 조준하러 간다(플레이어는 true).
   - 짧은 예측 지평(`aiSim` 0.7초, 플레이어 1.5초) - 먼 적은 위협으로 안 보고 조준을 유지하며 코앞 위협만 회피한다(hp 5개라 좀 더 대담하게). 조준 정렬률이 크게 오른다(재현상 ~40% → ~70%).
3.4. 자기 세로 밴드(`homeYRatio` 0.80, 화면 하단부 = 적 경로 끝자락)에서 논다. 너무 위(적 경로 한복판)면 늘 위험 판정이라 회피만 하고 조준을 못 한다.
3.5. 조작 불가(플레이어만 조작). 친구는 항상 AI 자율.
3.6. **플레이어와 좌우 분담**(협력 AI, 사용자 지시): 화면을 좌우로 나눠 플레이어와 **다른 쪽 적**을 주로 맡고(같은 반쪽 적은 후순위), 플레이어가 가장 가까이서 노리는 **아이템은 양보**한다(플레이어가 먹으러 가면 키위새는 다른 걸 먹으러 간다). 구현은 `decideTarget`의 `opts.mate`(분담 상대 = 플레이어). **보스전(`game.boss` 존재)에서는 분담을 풀고** 둘 다 보스에 집중한다(`friend.js`에서 보스전엔 `mate=null`).

## 4. 무기 (회색 직선 발사체 - 키위새 부리 모양)

4.1. 친구는 사이드 총알·꼬리 비행기·에너지존이 없다. **직선 발사체만** 있다.
4.2. 발사체는 강화 10단계(`level` 0~10). **부채 확산 없이 곧게 위로** 쏜다(사용자 지시). **강화해도 개별 총알 외형(길이·색·크기)은 변하지 않고**, 발 수와 데미지(파워)만 증가한다(사용자 지시).
   - 개수 = `min(shotsMax, shotsBase + level)`(1~4발, `laneGap` 간격으로 나란히 평행).
   - 길이 = `beamLen`(고정), 폭(충돌 반경) = `beamW`(고정), 발당 데미지 = `dmgBase + level*dmgGrow`(증가).
   - **발사 속도는 플레이어 메인 총알과 동일**(`CFG.bullet.speed` 참조, 별도 값 없음 - 사용자 지시).
4.3. **만렙에서도 직선 발사체를 유지**한다(전면 레이저 벽으로 바꾸지 않는다 - 사용자 지시 번복). 만렙(10)은 4발 직선(길이 최대).
4.4. 모양은 **키위새 부리와 같은 실루엣**(위로 뾰족한 가느다란 삼각, `drawFriendShot` - `drawFriend`의 부리와 동일 형태). 색은 **어두운 회색**(`COLORS.friend.shot`) + 절제된 회색 코어(`shotCore`, "너무 밝다" 지시로 눈부심 억제). 플레이어(차가운 시안)·적탄(빨간 원)과 색·형태로 구분.
4.5. 친구의 강화 정보는 HUD에 따로 표시하지 않는다(옆에서 발사체가 길어지는 걸 눈으로 볼 뿐).
4.6. **합체 발광**(사용자 지시): 플레이어 메인 총알과 친구 발사체가 화면에서 겹치면(`mergeDist` 이내) 두 총알 사이에 은은한 시안-흰 융합 발광을 얹어 "살짝 합쳐진" 모습을 낸다(`view.drawShotMerge`, 렌더만 - 총알 로직·궤적은 불변).

## 4.5. 대사 (같은 말 반복 금지 - 상황별 다양한 표현, 사용자 지시)

같은 말만 반복하지 않도록 상황마다 여러 표현을 두고 매번 다르게 고른다(각 8종, `numbers.js CFG.friend`).
- **맞았을 때**(`hitMsgs`): 친구가 피격되면(기절이 아닐 때) `hitMsgTime` 동안 말풍선("아야!", "아파아파!" 등).
- **살려줬을 때**(`reviveMsgs`): 기절한 친구를 회복(H)으로 부활시키면("고마워, 살려줘서!" 등).
- **잘했을 때**(`praiseMsgs`): 짧은 시간(`praiseWindow`) 안에 적을 `praiseKills`마리 이상 잡으면 칭찬("우와 잘한다!", "적들이 쓸려나가!" 등). 처치 신호는 `world.js` 적 처치 지점의 `notifyFriendKill`.
- **평상시 잡담**(`chatter`): 인트로 후 `chatterEvery`(+jitter)마다 한 줄("좋아 좋아!", "우리 팀 최고!" 등).
- 우선순위: 맞았을 때 > 살려줬을 때 > 잘했을 때 > 등장 인사 > 평상시 잡담(`stepSpeech`).

## 5. 아이템 공유 (서로서로 강화)

5.1. 친구와 플레이어는 아이템을 공유한다. 친구가 먹어도 플레이어가 강화되고, 플레이어가 먹어도 친구가 강화된다.
5.2. 아무 강화 아이템(P·S·E·T)이든 누가 먹든:
   - 플레이어는 그 아이템 계통이 오른다(기존 gainFront/Option/Zone/Tail 그대로).
   - 친구 메인 총알 `level`도 +1(최대 10). = 모든 아이템 공유.
5.3. 친구가 만렙(10)이어도 플레이어 계통이 오르면 정상 강화. 둘 다 무의미할 때만 점수 보너스(`maxedBonus`).
5.4. 아이템 획득 판정: 플레이어 또는 친구가 아이템에 닿으면 1회 획득(양쪽 동시 닿아도 한 번).

## 6. HP / 피격

6.1. 친구는 자기 하트 5개(`maxHp`). 플레이어 목숨과 별개 풀.
6.2. **총알·충돌 피해는 각자 개별**: 친구가 맞으면 친구 hp만 -1(플레이어 목숨 불변), 플레이어가 맞으면 플레이어만(기존 그대로). 피격 후 짧은 무적(`inv`).
6.3. 친구 hp 0 → 기절(`down`, 사라짐). 게임오버 아님 - 플레이어 혼자 계속.
6.4. **회복(H)은 공유**: H를 먹으면 플레이어 목숨 +1(상한 내) + 친구 hp +1(상한 내, 살아있을 때).
6.5. 친구가 기절한 상태에서 H를 먹으면 **부활**(hp 풀 회복 + 옆으로 재등장, `reviveMsg` 말풍선). 강화 `level`은 유지(모아둔 성장은 사라지지 않음).
6.6. **표정**(2026-07-16): 바푸리와 같은 규칙으로 총알에 맞으면 우는 표정, 아이템을 먹으면 웃는 표정(`view.drawFace` 공용, `emo`/`emoT`). 키위새는 위 보는 자세라 눈만 표정이 바뀐다(입은 부리). 상세 01_spec §2.5.
6.7. **죽는 연출**(2026-07-16): hp 0으로 기절할 때 몸·글로우·부리색 파편을 겹쳐 크게 터뜨린다(3중 폭발).

## 7. 점수

7.1. 점수는 원래 하나(`game.score`)라 자동 공유. 친구 총알로 잡은 적도 같은 점수에 합산.

## 8. 구현 위치 (매직넘버 0 / core 순수)

8.1. 수치: `src/data/numbers.js` `CFG.friend`. 색: `src/data/colors.js` `COLORS.friend`.
8.2. 순수 로직: `src/core/friend.js` - `spawnFriend`/`stepFriend`(빔서치 이동·말풍선·발사)/`gainFriendLevel`/`friendTakeHit`/`reviveFriend`/`notifyFriendKill`(연속 처치 칭찬)/`fireFriend`(직선 레이저)/`stepSpeech`(대사 우선순위). DOM/Canvas 미의존, `game.sfx`로만 사운드 신호.
8.3. 이동 AI: `src/core/autopilot.js` `decideTarget`을 공유한다. `actor`(친구 기체)·`opts.clearWhenSafe`·`tier.sim` 파라미터로 친구용 동작을 낸다. 플레이어 호출은 이 인자들을 안 넘겨 기존과 동일하게 동작(회귀 없음).
8.4. 월드 연동: `src/core/world.js` - `stepWorld`에서 `stepFriend` 호출, `checkCollisions`에서 친구 피격·아이템 공유(`grabItem`)·연속 처치 신호(`notifyFriendKill`) 처리.
8.5. 렌더: `src/render/view.js` - `drawFriend`(키위새 몸·부리·hp 점·말풍선)/`drawFriendShot`(어두운 회색 직선 레이저)/`drawBullets`에 `kind:'fmain'` 분기.
8.6. 생성: `src/main.js` - 어린이 모드 startGame 시 `spawnFriend`. 일반 모드는 null 유지.
8.7. 총알 `kind: 'fmain'`. 데미지·충돌은 기존 정면 화력과 동일 경로(부위·코어 타격, 방패병에 막힘) - 별도 특례 없음.
