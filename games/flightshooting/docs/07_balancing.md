# 07. 밸런싱 종합

게임 밸런스 수치를 한눈에 보는 문서. 무기·적·보스·드롭·성장 곡선을 축별로 정리한다. **실제 값의 SSOT는 `src/data/numbers.js`**이고, 이 문서는 그 값의 의미·의도·조정 이력을 설명한다(값이 바뀌면 이 표도 함께 갱신).

넘버링 근거: 무기 상세 설계 → 05_power-parts.md, 게임 규칙 → 01_spec.md, 수치 스키마 → 02_data.md.

## 0. 난이도 모드 (시작 화면 선택)

| 모드 | `enemyFireMul` | 효과 |
|---|---|---|
| 일반 | 1 | 기본 발사 빈도 |
| 어린이 | 2.2 | 적·보스 발사 간격 2.2배 = 총알 절반 이하로 덜 쏨(탄 수·패턴은 동일) |

`CFG.difficulty`. `world.js`가 적·보스 `fireTimer` 재설정 시 `game.enemyFireMul`을 곱한다. 어린이 모드는 저연령 플레이어가 적탄에 덜 시달리게 하는 배려용(사용자 지시 2026-07-09).

## 1. 플레이어·무기 화력

### 1.1. 메인 총알 (전방 화력 P, 내 비행기)

| 항목 | 값(numbers.js) | 의미 |
|---|---|---|
| 발사 쿨타임 | `player.fireEvery` 0.154초 | 자동 발사 간격. 작을수록 초당 화력↑ |
| 총알 속도 | `bullet.speed` 496 | 위로 나가는 속도(px/s) |
| 탄 수 성장 | 1~8단계 = 1→8발 | 단계당 +1발, 개별 탄 크기·색 동일 |
| 탄 수 데미지 | 1,1,2,2,3,3,4,4 | `1 + floor((shots-1)/2)` |
| 발별 진화 | 9~40단계 = 티어 1~4 | 8발 고정, 가운데부터 순차. 외형은 레이저 빔 유지, 티어↑ 길고 굵고 밝게(`mainBeams`) |
| 진화 데미지 | `front.shapeDmg` 1/티어 | 진화 탄만 티어당 +1 |
| 강화 표현 | 레이저 빔 강화(사용자 지시) | 각진 도형 폐기. 티어↑ w(반폭)·len(길이)·core(흰코어)·glow 상승 |
| 나란히 간격 | `front.laneGap` 11 | 여러 발 가로 간격(부채 아님, 직진) |

### 1.2. 사이드 총알 (옵션기 S, 사이드 비행기)

| 항목 | 값 | 의미 |
|---|---|---|
| 최대 대수 | `option.maxPerSide` 4 (좌4·우4 = 8) | S 획득당 좌우 번갈아 1대 |
| 발사 쿨타임 | `option.laserEvery` 0.176초 | 사이드 총알 발사 간격 |
| 총알 속도 | `option.laserSpeed` 880 | 메인보다 빠른 직진 빔 |
| 총알 크기 | `option.laserR` 1.1 + 대당 `laserRGrow` 0.15 | 총알 굵기(=히트박스). 대수↑ 굵어짐 |
| 기본 데미지 | `option.laserDmg` 1 + 대당 `laserDmgGrow` 0.5 | 대수 많을수록 세짐 |
| 부채 각(기본) | `option.laserDiagBase` 6도 | 안쪽(slot0) 비행기 벌어짐 각 |
| 부채 각(증분) | `option.laserDiagStep` 5.5도/slot | 바깥으로 갈수록 더 벌어짐 |
| 진화 | 8대 후 32단계 | 원→타원→빔→링 4티어, 안쪽부터 순차. tier↑ 데미지 +tier |

### 1.3. 유도탄 (꼬리 비행기 T)

| 항목 | 값 | 의미 |
|---|---|---|
| 최대 대수 | `tail.maxCount` 4 | 처음부터 유도탄 발사 |
| 무기 단계 | `tail.weaponMax` 4 | 4대 후 1~4번 순차 무기 진화 |
| 발사 쿨타임 | `tail.missileEvery` 2.7초 | 유도탄 발사 간격(느린 대신 단발 강함) |
| 시각 | 로켓 실루엣(`view.drawRocket`) + `tailMissileByStage` 색 | 노즈+몸통+뒤핀. 단계↑ 핀·발광 강화(꼬리불 별도) |
| 크기 | `tail.missileR` 2.4 + 단계당 `missileRGrow` 0.6 | 무기 단계↑ 커짐(이전 3.5/1.1에서 축소 - 사용자 "너무 크다") |
| 데미지 | `tail.missileDmgBase` 3 + 단계당 `missileDmgGrow` 1.5 | 단발 위력 큼(방패병 공략) |
| 유도 | `missileTurn` 3.2, `missileAccel` 520 | 가장 가까운 적 추적 |

### 1.4. 에너지존 (E)

| 항목 | 값 | 의미 |
|---|---|---|
| 레벨 | 0~5 (`levelMax` 5) | E 획득당 +1 |

## 2. 적 밸런스

| 적 | hp | 속도 | 점수 | 특징 |
|---|---|---|---|---|
| drone | 1 | 150 | 100 | 직진 하강, 발사 없음 |
| weaver | 2 | 120 | 150 | 사인파 흔들, 가끔 단발 조준 |
| gunner | 3 | 70 | 250 | x 추적, 3발 확산 |
| bonus | 5 | 130 | 500 | 가로질러 감, 확정 드롭원 |
| splitter | 4 | 105 | 300 | 격파 시 shard 2조각 분열 |
| shard | 1 | 250 | 50 | 작고 빠름, 재분열 없음 |
| shielder | 3 | 85 | 350 | 정면 방패(유도탄·존으로만 피해) |
| rusher | 2 | - | 320 | 조준 후 급강하 돌진 |

- **구역별 체력 상승**: `hp = ceil(base × (1 + (stage-1)×enemyHpScale))`, `enemyHpScale` 0.28 → 10구역 ≈ 3.5배.
- **11구역 이후 추가 배수**: `hardStage.hpMul` 1.35 (신규 적 구간 난이도 가속).

## 3. 보스 밸런스

| 보스 | hp | 크기(rx×ry) | 점수 | 드롭 |
|---|---|---|---|---|
| 중보스(1~19) | `baseHp` 66 + 구역×`hpPerStage` 34 | 30×26 | 900 | 2개 확정 |
| 최종보스(20) | `finalBoss.hp` 1500 | 50×44 | 12000 | 4개 확정 |

- 봄(B) 사용 시 보스에 `maxHp × 0.15` 즉시 피해.
- **주의**: 현재 보스 hp는 4계통 동시 성장 DPS를 완전 실측하기 전 값이다(05 §5.1). 부위 파괴형(06_boss.md) 도입 시 코어+부위로 hp가 재분배되므로 이 표는 그때 전면 재산정한다.

## 4. 드롭·성장 곡선

- **드롭 종류 가중치**(합 1): P 0.24 / S 0.22 / E 0.16 / T 0.16 / H 0.12 / B 0.10.
- **잡몹 드롭 확률**: `drop.chance` 0.11 (초반 성장 숨통). 보스·보너스는 확정 드롭.
- **보너스 기체**: `bonusShip.every` 10초마다 등장, 격파 시 1개 드롭.
- **만렙 후**: 파츠·목숨 최대 상태에서 파워업 획득 시 `maxedBonus` 300점.
- **손실**: 피격 시 목숨 -1 + 마지막 얻은 파츠 1개(역순).

## 5. 변경 이력

- **2026-07-09 후속2(사용자 지시)**: 메인 총알 레이저식 → **발별 진화 재전환**(각진 세트, 사이드와 외형 구분) + 강화1단계(다이아) 크기 절반. 존 → **펄스파**(닿을 때만 피해, 5단계 강화). 유도탄 쿨타임 0.9→2.7초 + 외형을 메인과 통일. 하트 대칭 곡선. HUD를 아이템 배지(P·S·E·T)로.
- **2026-07-09 밸런싱 조정(사용자 지시)**:
  - 사이드 총알 **크기 절반**: `laserR` 2.2→1.1, `laserRGrow` 0.3→0.15. (히트박스·굵기 동반 축소, 데미지 값은 유지)

## 6. 조정 시 유의

- 모든 값은 `numbers.js` 한 곳에서만 바꾼다(매직넘버 0). 이 문서는 뒤따라 갱신.
- 화력을 바꾸면 보스·적 hp와의 균형이 흔들린다 - DPS 실측(dev 훅 `?dev=1`) 후 보스 hp를 맞춘다(05 §5.1).
- "발사체 크기 = 데미지" 원칙: 크기를 줄이면 데미지도 함께 낮추는 게 원칙이나, 이번 사이드 크기 축소는 **시각·히트박스만** 줄이고 데미지는 별도 지시가 없어 유지했다(추후 데미지도 낮출지는 플레이 확인 후).
