# 05. 파워 파츠 시스템 (4계통)

메인 화력을 하나의 수치로 두지 않고 성격이 다른 네 갈래로 나눈다. 각 계통은 전용 아이템으로만 자라고, 피격하면 가장 최근에 얻은 파츠 하나를 역순으로 잃는다. 이 문서가 무기 사양의 SSOT이며, 코드(`core/fire.js` `core/parts.js` `data/numbers.js` `render/view.js` `main.js`)와 충돌하면 코드가 진실이다.

## 0. 핵심 원칙 (현재 사양, 2026-07-10 확정)

무기 강화의 근간은 **발별 순차 진화**다. 강화 아이템을 하나 먹을 때마다 총알이 **한 발씩** 다음 단계로 오른다(8발 전체가 한꺼번에 오르는 일괄 진화가 아니다). 이 원칙이 메인 총알과 사이드 총알에 똑같이 적용되고, 꼬리 유도탄은 발 대신 기체 단위로 같은 구조를 따른다.

1. **탄 수 먼저, 진화 나중**: 계통마다 먼저 발수(또는 대수)를 최대로 채우고, 그 뒤에 얻는 아이템이 개별 총알을 하나씩 진화시킨다.
2. **진화 티어 0~10**: 무강화가 tier 0, 최고가 tier 10이다. 티어마다 총알의 **형태 자체가 뚜렷이 다른 패턴**으로 바뀌어(색이 아니라 모양으로) 단계가 한눈에 읽힌다.
3. **형태는 view가 tier로 직접 그린다**: 렌더 형태는 `render/view.js`의 그리기 함수가 tier 값으로 분기해 즉석에서 그린다. `numbers.js`에 `shapes`나 `mainBeams` 같은 형태 배열은 없다. 색만 `colors.js`의 11칸 배열(tier 0~10)에서 뽑는다.
4. **발사체 데미지 = 크기·티어 비례**: 티어가 오른 총알일수록 데미지가 조금씩 오른다. 진화한 발만 세지고 아직 기본인 발은 그대로다. 같은 발사 안에서도 가운데 탄이 바깥 탄보다 셀 수 있다. 에너지존만 발사체가 아니라 이 원칙의 예외다(레벨 = 데미지).
5. **속도도 티어로 오른다**: 모든 총알 속도는 `base × (1 + floor(tier/3) × 0.15)`다(`fire.js speedMul`). 강화 세 단계마다 한 계단씩 빨라져, 초반 저티어 탄은 느리게 나가고 후반일수록 빨라진다.
6. **총알은 글로우 없음**: 총알·적탄은 매 프레임 수십 개가 그려지므로 발열을 막기 위해 그림자 글로우를 쓰지 않고 밝은 색 자체로 표현한다. 글로우는 플레이어·보스 같은 단일 요소에만 준다.

## 1. 네 계통

### 1.1. 전방 화력 (front, 아이템 P) = 메인 총알

내 비행기가 정면(위쪽, -y)으로 직접 쏘는 **메인 총알**이다. 강화 단계는 `front` 정수 하나(1~88)로 탄 수와 발별 진화를 모두 표현한다.

- **발사 형태**: 부채로 벌어지지 않고 곧게 직진한다(vx=0). 여러 발일 때는 `laneGap`(12px) 간격으로 가로로 나란히 평행하게 올라간다. 대각선 확산은 사이드 총알이 담당한다.
- **V자 대형**: 여러 발일 때 바깥 탄일수록 조금 뒤(아래)에서 출발시켜(`front.vStagger` 0.42) 은은한 화살촉 대형으로 흐르게 한다. 8발이 같은 높이에 일렬로 뭉쳐 격자처럼 보이던 것을 막는다. 발사 위치와 히트박스는 일치한다.

**1~8단계 (탄 수 성장)**: 강화할 때마다 탄이 한 발씩 는다(`shots = min(front, 8)`). 이 구간에서는 **개별 탄의 크기·모양·색이 전부 같다**(1단계 탄 = 8단계 탄). 성장은 오직 탄 수로만 보인다. 히트박스 크기는 고정이다(`front.rGrow = 0`). 데미지는 완만하게 오른다(`baseDmg = 1 + floor((shots-1)/2)` = 1, 1, 2, 2, 3, 3, 4, 4).

**9단계~ (발별 진화)**: 탄 수는 8발로 고정되고, **가운데 탄부터 한 발씩** 진화한다. 진화에 투입된 P 수는 `evoSteps = front - 8`이다.

- **진화 순번(rank)**: 8발은 좌우 대칭이라 정중앙 단일 탄이 없다. 순번은 중앙선에 가까운 탄부터, 거리가 같으면 왼쪽을 먼저로 매긴다(중앙좌 → 중앙우 → 그 바깥좌 → 바깥우 → …). rank 0이 가장 안쪽이다(`fire.js centerRanks`).
- **발별 티어 공식**: rank가 `i`인 탄의 티어는 `evoSteps >= i+1 ? min(floor((evoSteps - (i+1)) / 8) + 1, 10) : 0`이다.
  - 검산: front=9(evoSteps=1) → rank 0만 tier 1, 나머지 0. front=16(evoSteps=8) → 8발 전부 tier 1. front=17(evoSteps=9) → rank 0만 tier 2, 나머지 tier 1. front=88(evoSteps=80) → 8발 전부 tier 10(최종). tests가 이 검산점을 케이스로 쓴다.
- 한 티어가 8발 전체로 퍼지는 데 8단계가 걸린다. 티어 10까지면 진화 구간이 80단계이고, 탄 수 8단계를 더해 `front.max = 88`이다.
- **탄별 데미지**: `baseDmg + tier × shapeDmg`(shapeDmg=1). 진화한 탄만 위력이 오른다.

**외형(레이저 빔, tier 0~10)**: 메인 총알은 각진 도형이 아니라 처음부터 끝까지 **레이저 빔(빛줄기)** 형태다. 티어가 오를수록 빔 몸통의 무늬 패턴 자체가 다른 종류의 레이저로 바뀌고(`view.drawMainBeam`이 tier로 그린다), 길이·굵기도 는다(`len = mainLenBase 13 + tier × 1.2`, 반폭 `mainWBase 1.3 + tier × 0.11`, 사용자 지시로 절반 축소된 값). 색은 `colors.mainTier` 11칸(tier 0~10)에서 뽑는다(시안 → 민트 → 하늘 → 파랑 → 보라 → 자홍 계열로 냉색 스펙트럼을 돌아 최종 흰빛, 적탄의 빨강·노란 코어와 절대 겹치지 않게). 티어별 빔 무늬는 이렇다.

| tier | 빔 무늬 |
|---|---|
| 0 | 가는 실선(3px 폭 막대) |
| 1 | 굵은 실선 |
| 2 | 마디(빔을 다섯 칸으로 끊은 점선) |
| 3 | 구슬 체인(원 여러 개를 세로로 꿴 줄) |
| 4 | 마름모 체인 |
| 5 | 물결(사인파로 굽이치는 줄) |
| 6 | 톱니(좌우로 뻗은 지그재그) |
| 7 | 이중 빔(가는 막대 두 줄) |
| 8 | 나선(두 가닥이 꼬인 이중나선) |
| 9 | 화살촉 체인 |
| 10 | 플라즈마(넓은 오라 + 굵은 빔 + 흰 코어, 최종) |

### 1.2. 옵션기 (option, 아이템 S) = 사이드 총알

플레이어 좌우에 붙는 부속 비행기(**사이드 비행기**)가 쏘는 **사이드 총알**이다. S를 얻을 때마다 좌우 번갈아 한 대씩 늘어 최대 8대까지 붙는다(좌 4, 우 4, `maxPerSide` 4). n번째(0-based) 옵션은 `side = n%2==0 ? 왼 : 오`, `slot = floor(n/2)`(안 0 ~ 밖 3)로 배치된다. 8대 전부 레이저이며 유도 미사일은 없다(유도탄은 꼬리 비행기가 담당).

- **발사 형태(부채 확산)**: 각 비행기의 대각선 각도는 `laserDiagBase 6도 + slot × laserDiagStep 5.5도`다. 안쪽(slot 0) 비행기는 살짝, 바깥(slot 3)으로 갈수록 크게 벌어진다. 왼쪽 비행기는 왼쪽 위로, 오른쪽 비행기는 오른쪽 위로 쏜다. 렌더 형태는 진행 방향으로 회전한다.
- **굵기·데미지는 대수 비례**: 사이드 비행기가 많을수록 굵고 세다. 굵기 `laserR 1.1 + n × laserRGrow 0.15`, 데미지 `laserDmg 1 + (n-1) × laserDmgGrow 0.5`(n = 옵션 수)에 진화 티어만큼 `+tier`가 더해진다. 연사 주기 `laserEvery` 0.176초, 기본 속도 `laserSpeed` 640.
- **발별 진화(8대를 채운 뒤)**: 8대를 다 채운 뒤 얻는 S는 사이드 총알을 안쪽(rank 0)부터 한 발씩 진화시킨다. 진화도는 `game.optionEvo`(0 ~ 80)이고, 티어 공식은 메인과 같다(`evo >= rank+1 ? min(floor((evo-(rank+1))/8)+1, 10) : 0`). 8대 × 10티어 = 80단계다. 진화 tier가 오르면 데미지·형태·속도가 오른다.
- **외형(둥근 형태, tier 0~10)**: 메인의 각진 빔과 대비되게 사이드 총알은 둥근 형태 세트로 진화한다(`view.drawSideShape`). 색은 `colors.bulletShapeTier` 11칸에서 뽑는다(시안 계열에서 시작해 보라·자홍을 거쳐 최종 흰). 티어별 형태는 이렇다.

| tier | 형태 |
|---|---|
| 0 | 작은 원(이슬) |
| 1 | 큰 원(구슬) |
| 2 | 원 |
| 3 | 세로 타원 |
| 4 | 긴 타원(방울) |
| 5 | 고리(속 빈 링) |
| 6 | 이중 링 |
| 7 | 4각 별 |
| 8 | 6각 별 |
| 9 | 십자 스파클 |
| 10 | 태양(12방향 광선 + 흰 코어, 최종) |

### 1.3. 에너지존 (zone, 아이템 E) = 펄스파

플레이어 중심에서 링이 주기적으로 바깥으로 퍼지고, **링이 지나가는 순간 링 위(두께 판정 폭 안)에 있는 적·보스에게만** `dmg = level` 피해를 준다. 상시 깔려 있는 장판이 아니라 파동이므로 언제 때리는지가 눈에 보인다. 한 파동은 같은 대상을 두 번 때리지 않도록 파동별로 맞은 대상을 기록한다(`parts.js tickZone`).

강화 레벨은 0~5(`zone.levelMax` 5)이고, 레벨이 오를수록 파동이 더 자주·더 멀리·더 두껍게·더 세게 나간다.

| 레벨 | 주기 period(초) | 최대 반경 maxRadius | 링 두께 thick | 데미지 |
|---|---|---|---|---|
| 1 | 1.4 | 62 | 12 | 1 |
| 2 | 1.15 | 82 | 15 | 2 |
| 3 | 0.9 | 104 | 19 | 3 |
| 4 | 0.7 | 126 | 23 | 4 |
| 5 | 0.55 | 150 | 28 | 5 |

파동 확장 속도는 레벨과 무관하게 200px/s 공통이다. 발사체가 아니라 범위기라 발별 순차 진화·크기 비례 데미지 원칙에서 예외다(레벨이 곧 데미지). 존은 보스 부위를 관통해 코어를 직격한다.

### 1.4. 꼬리 비행기 (tail, 아이템 T) = 유도탄

플레이어 뒤(아래)에 세로 일렬로 따라붙는 유도탄 발사기다. 처음부터 유도탄을 쏜다. 방패병(shielder)처럼 정면을 막는 적을 측면·후방에서 잡는 역할을 한다.

- **4대 먼저 채운다**: T로 꼬리기를 1 → 4대까지 늘린다(`tail.maxCount` 4). 이 구간 모든 꼬리기 무기는 기본(weapon 1). 대수가 많을수록 발사량이 는다.
- **4대 후 무기 진화**: 4대를 채운 뒤 얻는 T는 **가장 낮은 무기 단계의 꼬리기부터 한 단계씩** 올린다(가장 앞 꼬리기부터 순차). `weapon`은 1(무강화) ~ 11(`weaponMax`), 티어는 `tier = weapon - 1`(0~10)이다. 4대가 다 오르면 다시 앞부터 다음 단계로 간다.
- **크기·데미지·속도**: 무기 단계가 오를수록 유도탄이 커지고 세지고 빨라진다. 크기 `missileR 1.2 + (weapon-1) × missileRGrow 0.3`(사용자 지시로 절반 축소), 데미지 `missileDmgBase 3 + (weapon-1) × missileDmgGrow 1.5`, 속도는 `speedMul(tier)` 배수. 발사 주기 `missileEvery` 2.7초.
- **유도**: 발사된 유도탄은 가장 가까운 적/보스를 향해 선회하며 가속한다(`parts.js homeMissiles`, 선회율 `missileTurn` 3.2, 가속 `missileAccel` 520). 표적이 없으면 위로 가속한다.
- **배치·움직임(뱀 꼬리 체인)**: 꼬리기는 4대가 한 몸으로 움직이지 않고 각자 자기 앞 개체를 개별 추종한다(1번은 플레이어를, 2번은 1번을, …). 추종 지연이 쌓여 방향을 틀면 뱀 꼬리처럼 출렁이며 따라온다(`tail.gap` 10, `tail.follow` 10). 구현은 뒤 개체부터 갱신해 앞 개체의 직전 위치를 따르게 한다.
- **외형(로켓·미사일 진화, tier 0~10)**: 유도탄은 형태가 진화하고(`view.drawMissile`), 몸체 색은 `colors.tailMissile3` 3색을 `tier % 3`으로 순환한다(꼬리 계통 녹·청 계열). 티어별 형태는 이렇다.

| tier(=weapon-1) | 형태 |
|---|---|
| 0~1 | 점 → 작은 원 |
| 2~3 | 삼각(크기 점증) |
| 4 | 화살촉 |
| 5 | 삼각 로켓(뾰족한 몸통 + 핀) |
| 6 | 로켓(원통 몸통 + 핀 + 뒤 꼬리불 + 노즈 흰 발광) |
| 7~8 | 로켓 + 중앙핀 보강 |
| 9~10 | 미사일 + 다탄두(양옆 보조 탄두) |

## 2. 성장·손실

획득 순서를 `game.partHistory` 스택에 계통 이름으로 기록하고, 피격 시 스택을 pop해 해당 계통을 한 단계 되돌린다(역순 손실, `parts.js loseLastPart`).

| 스택 항목 | 획득 시 | 피격 손실 시 |
|---|---|---|
| `front` | P 획득(탄 수·진화 무관) | `front = max(1, front-1)` |
| `option` | 사이드 비행기 대수 증가 | 마지막 옵션기 1대 제거 |
| `optionEvo` | 8대 후 사이드 진화 | `optionEvo -= 1` |
| `zone` | 존 레벨 상승 | 존 레벨 -1 |
| `tail` | 꼬리기 대수 증가 | 마지막 꼬리기 1대 제거 |
| `tailWeapon` | 4대 후 꼬리기 무기 진화 | 가장 최근 오른 꼬리기 무기 -1 |

- **강등 시 진화 자동 정합**: `front`가 정수 하나라, 피격으로 8 밑으로 떨어지면 `shots < 8`이 되어 `evoSteps = 0`, 진화 티어가 자동으로 사라진다. 재성장으로 8발을 다시 채우면 9단계부터 진화가 처음(가운데 1발 tier 1)부터 다시 시작한다. 별도 상태 없이 유령 진화 탄이 남지 않는다. 꼬리기도 대수가 줄면 그 무기 진행이 함께 소멸한다.
- **만렙 후 처리**: 파츠가 이미 최대(front 88, 사이드 진화 80, 존 5, 꼬리 무기 전원 11)일 때 해당 아이템을 더 얻으면 성장 없이 `maxedBonus` 점수로 처리한다(`world.grabItem`).
- 회복(H)·봄(B)은 별도 계통으로 유지한다.

## 3. 드롭

- 아이템 6종: P(전방화력) / S(옵션기) / E(에너지존) / T(꼬리기) / H(회복) / B(봄).
- 가중치(합 1): P 0.24 / S 0.22 / E 0.16 / T 0.16 / H 0.12 / B 0.10 (`drop.weights`).
- 일반 잡몹은 처치 시 `drop.chance`(0.11) 확률로 드롭, 보스·보너스 기체는 확정 드롭이다.

## 4. HUD 표기 (강화 단계)

강화 단계는 별(★) 없이 로마숫자(티어)와 아라비아 숫자(서브스텝)로만 보인다(`main.js`). 마스터(최대 강화) 도달 시 주황색(`.mastered`)으로 표시한다.

- **카운트 채우는 중**: 발수·대수·레벨을 채우는 동안(또는 진화 전)에는 카운트 숫자를 그대로 보인다.
- **진화 시작 후**: `로마숫자(티어) + ·서브스텝` 형식으로 보인다. 예를 들어 3티어를 3발째 진행 중이면 `III·3`, 서브스텝이 0이면 로마숫자만(`III`). `tierText` 함수가 담당한다.
- **계통별 매핑**:
  - 메인(P): `setEvoHud(front, 8, front-8, 10)`. front 1~7은 탄 수, 8은 탄 수 만렙, 9~88은 진화(evo = front-8). 티어는 `floor((evo-1)/8)+1`로 대략 표시.
  - 사이드(S): `setEvoHud(옵션수, 8, optionEvo, 10)`.
  - 존(E): `setPartHud(level, 5)`. 서브스텝 없는 단순 레벨 카운트.
  - 꼬리(T): `setTailHud`. 4대 미만이면 대수, 4대 후엔 최저 무기 단계(`minW-1`)를 로마숫자로, 다음 단계로 오른 대수를 서브스텝으로 보인다.

## 5. 밸런스 기본값 (플레이 조정 대상)

수치는 전부 `data/numbers.js`가 SSOT다. 아래는 현재 값의 요약이다.

- **메인(front)**: 탄 수 구간 데미지 `1 + floor((shots-1)/2)`(1~4), 진화 구간 탄별 데미지 `baseDmg + tier × 1`. 히트박스 고정(rGrow 0). 빔 길이·굵기만 티어로 증가(절반 축소 값). 색 `mainTier` 11칸.
- **사이드(option)**: 데미지 `1 + (옵션수-1) × 0.5 + tier`, 굵기 `1.1 + 옵션수 × 0.15`, 부채 `6도 + slot × 5.5도`, 속도 640 × speedMul, 연사 0.176초. 색 `bulletShapeTier` 11칸.
- **에너지존(zone)**: 위 1.3 표(레벨별 주기·반경·두께·데미지), 확장 속도 200 공통.
- **꼬리(tail)**: 최대 4대, 무기 1~11. 크기 `1.2 + (weapon-1) × 0.3`, 데미지 `3 + (weapon-1) × 1.5`, 발사 2.7초, 선회 3.2, 가속 520. 색 `tailMissile3` 3색 순환.
- **속도 공통**: `base × (1 + floor(tier/3) × 0.15)`, base speed 360(메인).

### 5.1. 성장 도달 곡선

- **실 도달 구간**: 각 계통 만렙(front 88, 사이드 진화 80, 꼬리 무기 44단계)은 30구역 게임 전체를 통틀어도 최상위 플레이 전용이다. 대부분 플레이어가 실제로 보는 건 저~중 티어다. 고티어는 성장 상한 여지로 두되, 밸런스·시각 완성도의 초점은 저~중 티어에 둔다. 고티어가 데드 콘텐츠가 되지 않게 하려면 진화 속도(티어당 필요 아이템)를 실측 후 낮추는 것도 선택지다.
- **4계통 동시 성장 DPS**: 메인 진화 + 사이드 8대(데미지 성장) + 꼬리 유도탄 4대(무기 성장) + 존이 겹치면 후반 DPS가 크게 오른다. 보스 hp(`finalBoss` 1500, `miniBoss` baseHp 66 + 구역 × 34)는 이 4계통 성장을 감안한 값이며, dev 훅(`?dev=1`)으로 각 계통 티어를 강제해 단위 DPS와 후반 구역 클리어 시간을 실측하며 조정한다.

## 6. 확정·미확정

**확정(사용자 지시)**:
- 무기 강화 = 발별 순차 진화 10단계(강화 아이템마다 총알 한 발씩 진화, 일괄 아님).
- 메인 1~8 개별 탄 균일(크기·모양·색 동일), 9~88 발별 진화.
- 메인 외형은 레이저 빔 무늬 세트, 사이드는 둥근 형태 세트로 계통 구분.
- 사이드 8대 후 발별 진화, 꼬리 4대 후 무기 순차 진화.
- 유도탄·메인 빔 크기 절반 축소.
- HUD는 별 없이 로마숫자(티어) + 아라비아(서브스텝), 마스터 시 주황.
- 총알 글로우 없음(발열), 단일 요소만 글로우.

**미확정(구현 중 조정 가능)**:
1. 각 계통 고티어의 실 도달 여부와 진화 속도(티어당 필요 아이템) 조정 폭(5.1).
2. 진화 순번(중앙 근접 순, 동거리 좌 우선)이 실제 플레이에서 자연스러운지.
3. 4계통 동시 성장 DPS 실측 후 보스 hp 재산정 폭(5.1).
