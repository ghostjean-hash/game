# 05. 파워 파츠 시스템 (4계통)

기존 단일 `power` 화력을 여러 갈래 파츠로 분리한다. 사용자 확정(2026-07-07): 파츠별 전용 아이템으로 성장, 피격 시 마지막 얻은 파츠 1개 손실. 2026-07-08 재설계로 계통이 4개(전방 화력 P / 옵션기 S / 에너지존 E / 꼬리 비행기 T)가 되고, P 탄 진화와 S 발사체 규칙이 아래처럼 바뀐다.

## 0. 재설계 요지 (2026-07-08, 사용자 확정)

이번 개정에서 바뀌는 것만 먼저 요약한다. 상세는 각 절.

1. **P 1~8단계 개별 탄 완전 균일화**: 지금은 화력이 오를수록 개별 탄이 커진다(1단계 탄 < 8단계 탄). 이 크기 성장을 없앤다 - **1~8단계는 탄 수만 1→8발로 늘고, 개별 탄의 크기·모양·색은 전부 동일**(1단계 탄 = 8단계 탄). 겉보기 급증 금지.
2. **P 탄 진화 = 발별 순차 + 형태 진화**: 9단계부터 8발 고정, **가운데 탄부터 한 발씩** 다음 티어로 진화하고, 8발이 다 진화하면 그 다음 티어로 넘어가 다시 가운데부터. 진화 탄은 티어마다 **형태가 원→타원→긴형→링으로 바뀌어**(핵심 표지) 단계가 한눈에 읽히고, 색·글로우는 아군 차가운색 계열 안에서 보조로 계단식 구분한다(적탄 빨강+노란 코어와 절대 안 겹치게). 진화 티어는 4단계.
3. **S 발사체 통일**: 5~8번째 옵션기가 유도탄(미사일)을 쏘던 것을 없애고, 8대 전부 레이저로 통일한다. 유도탄 역할은 신규 꼬리 비행기로 옮긴다.
4. **T 꼬리 비행기 신설**: 플레이어 뒤에 붙는 최대 4대의 유도탄 발사기. 처음부터 유도탄. 전용 아이템 T로 성장 - **4대 먼저 채운 뒤, 1번→2번→3번→4번 꼬리기 순서로 무기가 한 단계씩 진화**(P 발별 진화와 같은 구조, 무기 4단계).
5. **발사체 데미지 = 모양(크기) 비례**: 모든 발사체는 티어·크기가 오를수록 데미지가 "조금씩" 오른다(P 탄 티어 / S 레이저 굵기 / T 유도탄 무기 단계). 에너지존만 발사체가 아니라 예외(레벨 = 데미지 유지).

## 1. 네 계통

### 1.1. 전방 화력 (front, 아이템 P) = 메인 총알

**메인 총알**(내 비행기가 직접 쏘는 총알)을 담당한다. 화력 레벨은 이제 1~40이고, 두 구간으로 나뉜다.

- **발사 형태(사용자 확정 2026-07-09)**: 메인 총알은 **직진**한다. 부채로 벌어지지 않고, 여러 발일 때는 `laneGap` 간격으로 가로로 나란히 평행하게 곧게 올라간다(양쪽 대각선 확산은 사이드 총알이 담당). 각 탄의 가로 위치는 `xOff`(중앙 기준).
- **1~8단계 (탄 수 성장)**: 단계당 탄 +1(1~8발). 이 구간에서 **개별 탄의 크기·모양·색은 전부 동일**하다(1단계 탄 = 8단계 탄, 기본 세로 타원·기본 시안). 성장은 오직 탄 수로만 보인다. 데미지는 완만 상승(1,1,2,2,3,3,4,4).
- **9단계~ (탄 진화)**: 탄 수는 8발로 고정하고, 개별 탄을 가운데부터 한 발씩 진화시킨다. 최대 40단계(8발 기본 + 진화 4티어 × 8발).

측면·확산·유도는 사이드 총알·유도탄이 담당한다.

#### 1.1.1. 탄 발별 순차 진화 (사용자 확정 2026-07-08)

만렙(P8, 8발) 이후 P를 더 먹으면, **8발 중 가운데에 가까운 탄부터 한 발씩** 다음 티어로 진화한다.

- **9단계**: 가운데 탄 1발만 티어 1로 진화. (나머지 7발 기본)
- **10단계**: 가운데에서 2번째 탄까지 티어 1. …
- **16단계**: 8발 전부 티어 1 (티어 1 완성).
- **17단계**: 다시 가운데 탄 1발만 티어 2로 진화. (나머지 7발 티어 1)
- …이 패턴을 티어 4까지 반복. **40단계**에서 8발 전부 티어 4(최종).

즉 한 티어가 8발 전체로 퍼지는 데 8단계가 걸리고, 티어 4개면 진화 구간이 32단계다.

- **가운데 순번 규칙**: 8발은 부채꼴로 좌우 대칭이라 정중앙 단일 탄이 없다. 진화 순번은 **중앙선에 가까운 탄부터, 거리가 같으면 왼쪽 먼저**로 매긴다(중앙좌 → 중앙우 → 그 바깥좌 → 바깥우 → …). 이 순번은 플레이 확인 후 조정 가능하다.
- **발별 티어 산출 공식 (루프 3, 구현·테스트 명문화)**: `TIER_MAX`=4(진화 티어 수)로 두고,
  - `evoSteps = max(0, front - 8)` (진화에 투입된 P 수).
  - 순번 `rank`(0~7)는 위 가운데 순번 규칙으로 8발에 부여(rank 0 = 가장 안쪽).
  - rank `i`인 탄의 티어 = `evoSteps >= i+1 ? min(floor((evoSteps - (i+1)) / 8) + 1, TIER_MAX) : 0`.
  - 검산: front=9(evoSteps=1) → rank0만 티어1, 나머지 0. front=16(evoSteps=8) → 전원 티어1. front=17(evoSteps=9) → rank0 티어2·나머지 티어1. front=40(evoSteps=32) → 전원 티어4. ✓
  - tests는 이 네 검산점을 그대로 케이스로 쓴다.
- **형태 구분(핵심 표지) - 원→타원→긴형→링 (사용자 확정 2026-07-08: "색만이 아니라 모양이 바뀌어야")**: 진화 탄은 티어마다 **형태가 뚜렷이 바뀌어** 한눈에 단계가 읽힌다. 색·글로우는 보조 표지다. 색은 적탄과 절대 안 겹치는 아군 차가운색(시안·흰·하늘·민트) 안에서만 쓰고, 적탄색(노랑·주황·빨강)은 배제한다.
  - 기본(진화 전): **세로 타원**, 옅은 시안, 글로우 없음
  - 티어 1: **원**(굵은 구슬), 순백 코어 + 시안 글로우(중)
  - 티어 2: **타원**(원을 세로로 늘인 럭비공), 밝은 하늘색 + 글로우(강)
  - 티어 3: **긴형**(레이저빔처럼 길쭉), 형광 민트 + 글로우(더 강)
  - 티어 4: **링**(가운데 뚫린 발광 고리, 최종), 흰빛 시안 + 대형 글로우(최대)
  - 형태가 갈수록 커지거나 길어지고 링만 형태 축을 바꿔 최종·최강을 표시한다. 색·글로우 계단은 보조. 같은 발사에서 가운데 몇 발이 어느 단계인지 형태로 즉시 보인다.
  - **피아 식별 확실 (루프 2)**: 진화 탄은 전부 차가운색 + 시안 글로우 상시라 적탄(뜨거운색)과 절대 안 겹친다. 아군탄 아래→위 / 적탄 위→아래 이동 방향도 보조 단서. (구체 색값 → §5 진화 색)
- **크기·형태**: 티어별 렌더 형태·배율은 `bullet.shapes`(rx/ry 배율, 링은 `ring`=뚫린 안쪽 비율)로 정의한다 - 원→타원→긴형→링. 히트박스(`r`)는 P8 기준 고정(밸런스 안정, 렌더 형태만 변화).
- **탄별 데미지**: 각 탄의 데미지 = `P8 기준 데미지(4)` + `그 탄의 티어 × shapeDmg`(shapeDmg=1, 조금씩). 진화한 탄만 위력이 오르고 아직 기본인 탄은 그대로다. 같은 발사에서도 가운데 탄이 바깥 탄보다 세다.
- **HUD**: 전방 화력 칸은 P8 도달 시 `★`, 진화 진행도에 따라 현재 최고 티어를 `★1`~`★4`로 함께 보인다.

### 1.2. 옵션기 (option, 아이템 S) = 사이드 총알

좌우에 붙는 부속 비행기(**사이드 비행기**). S 획득당 좌우 번갈아 1대씩, 최대 8대(좌4·우4, `maxPerSide` 4).

- **8대 전부 레이저(= 사이드 총알)로 통일**(2026-07-08 변경). 기존 슬롯 2·3의 유도 미사일을 제거한다 - 유도탄은 꼬리 비행기(1.4)로 이관.
- n번째(0-based) 옵션: `side = n%2==0 ? 왼(-1) : 오(+1)`, `slot = floor(n/2)`(안→밖 0~3). 슬롯은 위치·발사체 굵기만 정한다.
- **발사 형태(사용자 확정 2026-07-09)**: 사이드 총알은 **부채로 퍼진다**. 각 비행기의 대각선 각 = `laserDiagBase`(12도) + slot × `laserDiagStep`(11도)로, 안쪽(slot 0) 비행기는 살짝·바깥(slot 3)으로 갈수록 크게 벌어진다. 왼쪽(side -1)은 왼쪽 위, 오른쪽(side +1)은 오른쪽 위. 직진은 메인 총알이 담당. 렌더는 진행 방향으로 막대를 회전.
- 사이드 총알 = 흰-보라 가는 빔(라벤더 외곽 + 흰 코어). **굵기·데미지가 옵션 수에 비례 상승**(발사체 크기 = 데미지 원칙). 사이드 비행기가 많을수록 굵고 세다.
- 위치는 플레이어 기준 대칭 오프셋, 부드럽게 따라온다.

### 1.3. 에너지존 (zone, 아이템 E)

플레이어 주변 원형 오라. 레벨 0~5, 레벨↑ 반경↑(`zone.radius[]`). `tick`(0.5초)마다 존 반경 내 모든 적·보스에게 `dmg = level` 피해. **발사체가 아니라 범위기라 "모양 비례 데미지" 원칙의 예외**(레벨 = 데미지 유지). 존 내 적탄을 바깥으로 미는 밀어내기도 유지.

### 1.4. 꼬리 비행기 (tail, 아이템 T) - 신규

플레이어 **뒤(아래쪽)**에 일렬로 따라붙는 유도탄 발사기. **처음부터 유도탄을 쏜다.** 전용 아이템 T로 성장하며, 두 구간으로 나뉜다(P의 탄 수 → 발별 진화 구조와 동일).

- **1~4단계 (대수 성장)**: T 아이템으로 먼저 꼬리기를 1→4대까지 늘린다. 이 구간에서 모든 꼬리기 무기는 기본(1단계) 유도탄. 대수가 많을수록 발사량↑.
- **5단계~ (꼬리기별 무기 진화)**: 4대를 다 채운 뒤 얻는 T는 **1번 → 2번 → 3번 → 4번 꼬리기 순서로** 무기를 한 단계씩 올린다. 4대가 다 오르면(한 바퀴) 다시 1번부터 그 다음 단계로. **무기 4단계**.
  - **5단계**: 1번 꼬리기 무기 2단계. (2·3·4번 1단계)
  - **8단계**: 4대 전부 무기 2단계.
  - **9단계**: 1번 꼬리기 무기 3단계. … 반복.
  - **20단계**: 4대 전부 무기 4단계(최종). 총 T 성장 = 대수 4 + 무기(4대 × 3단계 상승) 12 = **16단계**(무기 4단계 기준).
- **무기 단계 표현**: 유도탄이 단계마다 **작은 것 → 큰 것**으로 커지고(단조 증가, 절대 작아지지 않음), 색·광량으로도 구분. 단계 클수록 **데미지·폭발 반경 상승**(발사체 크기 = 데미지 원칙).
- 유도탄은 가장 가까운 적/보스를 추적(기존 옵션 미사일 유도 로직 재사용). 방패병(shielder) 같은 정면 방어 적을 측면에서 잡는 역할.
- **배치·움직임 (사용자 확정 2026-07-08)**: 꼬리기는 플레이어 **뒤(아래)로 세로 일렬**로 붙는다. 4대가 한 몸처럼 동시에 움직이지 않고, **각자 자기 앞 개체를 개별 추종**한다 - 1번은 플레이어를, 2번은 1번을, 3번은 2번을, 4번은 3번을 따라온다. 추종에 개체별 지연이 쌓여 플레이어가 방향을 틀면 뱀 꼬리처럼 출렁이며 따라오는 그림이 된다. 간격은 `tail.gap`(앞 개체 반경 + gap), 추종 속도는 `tail.follow`(클수록 덜 늘어짐). 구현은 뒤 개체부터 갱신해 앞 개체의 직전 위치를 따르게 한다(즉시 전파 대신 지연 체인). 세로로 길어지지만 gap이 작아 화면 하단 여백 안에 들어온다.

## 2. 성장·손실

- 획득 순서를 `game.partHistory` 스택에 계통 이름으로 기록: `'front'` / `'option'` / `'tail'`(꼬리기 대수) / `'tailWeapon'`(꼬리기 무기 진화) / `'zone'`.
  - P는 단계(탄 수·진화 무관)마다 `'front'` 하나씩 push. 기존 `'shape'` 항목은 폐기하고 `'front'` 단일로 통합한다(front가 1~40 정수 하나로 탄 수와 진화를 모두 표현하므로).
  - T도 마찬가지로 대수 구간은 `'tail'`, 무기 진화 구간은 `'tailWeapon'`을 push(꼬리기 진행 정수 하나로 표현).
- 피격 시 스택 pop → 해당 계통 1단계 되돌림:
  - `'front'`: `front = max(1, front-1)`. 진화 구간이면 가장 최근 진화한 탄이 한 티어 후퇴, 탄 수 구간이면 탄 -1.
  - `'option'`: 마지막 옵션기 1대 제거.
  - `'tail'`: 마지막 꼬리기 1대 제거.
  - `'tailWeapon'`: 가장 최근 무기가 오른 꼬리기가 한 단계 후퇴.
  - `'zone'`: 존 레벨 -1.
- front 시작값 1은 스택에 없음(P로 2 이상 올릴 때만 push).
- 회복(H)·봄(B)은 기존 유지.
- **피격 강등 시 진화 자동 정합 (루프 3)**: front·꼬리기 진행이 각각 정수 하나라, 피격으로 `front`가 8 밑으로 떨어지면 `shots = min(front,8) < 8`이 되어 **진화 티어는 자동으로 사라진다**(evoSteps=0). 재성장으로 다시 8발을 채우면 9단계부터 진화가 처음(가운데 1발 티어1)부터 다시 시작한다. 별도 상태 없이 front 하나로 일관 처리되므로 강등 엣지케이스에서 유령 진화 탄이 남지 않는다. 꼬리기도 동일(대수가 줄면 그 꼬리기 무기 진행도 함께 소멸).
- **만렙 후 처리 (루프 3)**: P가 40(전원 티어4), T가 무기 만렙(4대 전원 4단계)에 도달한 뒤 얻는 P·T는 성장 없이 `maxedBonus` 점수로 처리한다(목숨·기존 파츠 만렙과 동일 규칙, world.grabItem).

## 3. 드롭 (spawn.dropItem)

- 아이템 종류가 **6종**으로 늘어난다: P / S / E / **T(신규)** / H / B.
- 가중치 재분배 제안(합 1): **P 0.24 / S 0.22 / E 0.16 / T 0.16 / H 0.12 / B 0.10**. 플레이 조정 대상.
- 일반 잡몹 처치 시 `chance` 확률 드롭 / 보스·보너스 기체는 확정 드롭(기존 유지).

## 5. 밸런스 기본값 (플레이 조정 대상)

- **front**: 탄 수 구간 dmg `1 + floor((L-1)/2)`(L 1~8). 진화 구간 탄별 dmg `4 + tier × shapeDmg`(shapeDmg=1). 부채폭 `min((min(front,8)-1)*7, 84)`. 히트박스 `rBase + baseDmg*rGrow`에서 **rGrow=0(개별 탄 크기 고정)**, 진화 티어 렌더 배율은 세로만 미세(예: `[1.0, 1.1, 1.2, 1.3]`) + 글로우.
- **진화 색 (차가운색 팔레트, `colors.bulletShapeTier`)**: 기본(진화 전) 시안 `#22d3ee`. 티어 1 순백 `#ffffff` / 티어 2 밝은 하늘 `#7dd3fc` / 티어 3 형광 민트 `#5effd0` / 티어 4 흰빛 시안 `#c7f9ff`. 글로우는 모두 시안 계열(`#22d3ee`)로 티어당 반경·강도 계단식 증가. 노랑·주황·빨강(적탄색)은 배제. 구체 색값은 플레이 확인 후 미세 조정.
- **진화 순번**: 중앙 근접 순 + 동거리 좌 우선(1.1.1).
- **option 레이저**: dmg `laserDmg + (옵션수-1) × laserDmgGrow`, 굵기 `laserR + 옵션수 × laserRGrow`, 속도 1.4배, 연사 0.16초. (미사일 수치 삭제)
- **tail**: 최대 4대. 무기 4단계 = 반경 `missileR + (무기단계-1) × missileRGrow`, dmg `missileDmgBase + (무기단계-1) × missileDmgGrow`. 단계별 색·크기로 구분. 발사 연사·유도 선회·가속은 기존 옵션 미사일 값 계승 후 조정.
- **zone**: 반경 `[0,34,52,70,88,106]`, tick 0.5초, dmg = level.

### 5.1. 성장 도달 곡선 (루프 1 - 데드 콘텐츠·화력 총량 경계)

- **실 도달 구간 인지**: P는 이론상 40단계지만 20구역 게임에서 P를 40까지(9단계 이후 P 32개 추가) 올리는 건 최상위 플레이 전용이다. **대부분 플레이어가 실제로 보는 건 진화 티어 1~2(9~24단계)**다. 따라서 티어 3·4는 "성장 상한 여지"로 두되, **밸런스·시각 완성도의 초점은 티어 1~2에 둔다**(티어 1↔2의 흰/노랑 대비가 확실히 보이게). 티어 3·4가 데드 콘텐츠가 되지 않게 하려면 진화 속도(티어당 필요 P)를 실측 후 낮추는 것도 선택지.
- **4계통 동시 성장 DPS 재산정**: 현재 보스 hp(finalBoss 1500, miniBoss baseHp 66+구역×34)는 **3계통 기준**으로 잡힌 값이다. P 진화 + S 레이저 8대(데미지 성장 신규) + T 유도탄 4대(무기 성장) + E존이 겹치면 후반 DPS가 크게 오른다. 4계통 만렙 근처 DPS를 dev 훅으로 실측해 **보스 hp를 재산정**한다(순삭 방지). 특히 S 레이저에 `laserDmgGrow`가 새로 붙어 옵션 8대 화력이 기존보다 세짐.
- 검증: dev 훅(`?dev=1`)으로 P 티어 1~4·T 무기 1~4·S 8대를 각각 강제해 단위 DPS를 재고, 후반 구역(11·13·20) 클리어 시간을 측정한다.
