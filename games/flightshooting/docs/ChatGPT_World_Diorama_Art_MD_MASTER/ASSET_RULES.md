# ASSET RULES

## 문서 목적

이 문서는 국가별 배경 아트의 파일 종류, 버전 관리, 검수 및 청크 분할 원칙을 정의한다.

기술 구현 규격이 아직 확정되지 않은 경우 해상도, 압축 방식, 메모리 제한, 청크 개수 등을 임의로 고정하지 않는다.

---

## 기본 원칙

- 아트 원본과 파생본을 분리한다.
- 원본을 덮어쓰지 않는다.
- 수정본은 새 버전으로 저장한다.
- 파일명은 영문, 숫자, 하이픈만 사용한다.
- 국가 코드, 시간대, 파일 종류, 버전을 파일명에 포함한다.
- 최신 승인 기준 이미지를 명확히 구분한다.
- 승인되지 않은 시안을 production으로 취급하지 않는다.
- AI가 임의로 최종 파일 또는 승인본을 결정하지 않는다.

---

## 권장 파일명

```text
[COUNTRY]-[TIME]-[TYPE]-[VERSION].[EXT]
```

예시:

```text
KR-day-concept-v01.png
KR-day-production-v01.png
KR-day-review-v02.png
KR-day-chunk-00-v01.png
KR-night-production-v01.png
```

## 이미지 종류

### concept

초기 방향 확인용 이미지다.

검토 항목:

- 카메라
- 전체 구성
- 국가 흐름
- 중앙 이동축
- 랜드마크 순서와 위치
- 인트로 랜드마크
- 교육 포인트

최종 품질이나 미세한 재질보다 전체 방향 확인이 목적이다.

concept는 production이 아니며, 사용자의 명시적 승인 없이 최종 기준으로 사용할 수 없다.

### production

최종 승인용 고품질 원본 이미지다.

조건:

- 승인된 concept의 카메라와 구성을 유지한다.
- 최신 승인 기준 이미지와 동일한 시각 언어를 사용한다.
- 실제 청크 분할과 인트로 확대의 기준이 된다.
- 사용자가 명시적으로 승인한 경우에만 production 승인본이 된다.

### review

비교와 검수에 사용하는 파생본이다.

- 필요하면 축소본을 사용한다.
- 원본과 동일한 구도를 유지한다.
- 중앙 보호 영역, 청크 경계, 랜드마크 위치 등의 가이드를 표시할 수 있다.
- review 파일을 production 원본으로 대체하지 않는다.

### chunk

승인된 production 이미지를 실제 스크롤 배경으로 분할한 파일이다.

- 국가 전체 production 이미지가 승인된 뒤에만 만든다.
- 청크가 아트 구성을 결정하지 않는다.
- 모든 청크의 폭은 동일하게 유지한다.
- 경계 전후의 지형, 색감, 조명 흐름을 유지한다.
- 핵심 랜드마크를 청크 경계에 배치하지 않는다.

---

## 청크 분할 원칙

- 국가 전체 동선과 아트를 먼저 설계한다.
- 승인된 전체 production 이미지를 기준으로 청크를 나눈다.
- 핵심 랜드마크를 청크 경계에 두지 않는다.
- 경계에서 강, 도로, 해안선, 지형, 건물, 조명이 끊기지 않아야 한다.
- 필요한 경우 오버랩 또는 블렌딩 여유를 확보한다.
- 청크 단위로 서로 다른 시간대나 색감을 적용하지 않는다.
- 무한 반복은 해당 스테이지에 실제로 필요한 경우에만 별도 설계한다.
- 무한 반복을 위해 국가 흐름과 랜드마크 구성을 훼손하지 않는다.

---

## 검수

모든 배경은 다음 상태에서 확인한다.

1. 전체 스테이지 축소 보기
2. 인트로 랜드마크 확대 보기
3. 실제 플레이 배율
4. 중앙 보호 영역 표시
5. 플레이어, 적, 탄환 합성
6. 실제 세로 스크롤
7. 청크 경계
8. 목표 기기의 화면 비율
9. 주간 또는 야간의 점광원 가독성
10. 핵심 랜드마크의 식별 가능 여부

---

## 수정 원칙

- 승인된 카메라와 구성을 임의로 바꾸지 않는다.
- 수정 요청 범위 밖의 요소를 재설계하지 않는다.
- concept, production, review, chunk를 명확히 구분한다.
- 승인된 production 파일을 직접 덮어쓰지 않는다.
- 새 수정본은 버전을 올려 저장한다.
- 사용자가 요청하지 않은 장식이나 랜드마크를 추가하지 않는다.
- 최신 승인 기준 이미지가 변경된 경우 CHANGELOG.md에 기록한다.
