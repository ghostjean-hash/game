### 배포 실행 + 정책 정정 (같은 세션 후속)
- 위 UI 개편을 GitHub Pages에 배포 완료: 원격이 english-reading 작업으로 앞서 있어 `git pull --rebase`(충돌 0)로 병합 후 `git push`(94fe38b). GitHub Pages 전파 후 공개 URL·타 기기에 반영.
- **"검수 후 배포" durable 보류 해제**: 배경 디오라마 검수는 지난 세션에 끝났으므로 배포를 막던 durable 지시를 해제한다. `/jarvis-checkpoint`는 규칙(§2.2)대로 커밋 + 배포(push)까지 자동 진행하는 것이 정상이며, 직전 봉합에서 durable을 과잉 적용해 push를 멈췄던 것을 정정했다. 다음 세션부터 /jc는 배포까지 자동.
- 배포 개념 안내(사용자 헷갈림 해소): 로컬 수정(이 컴퓨터, dev-server에서 즉시 보임) / 커밋(저장 도장) / 푸시(인터넷 공개) 3단계. 로컬에서 최신으로 보여도 push 전에는 공개 주소·폰에 반영되지 않음. SW 캐시로 배포 후에도 옛 화면이 잠깐 보일 수 있어 앱 재시작·새로고침 안내.
