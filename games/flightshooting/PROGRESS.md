- **뒤로가기 이탈 버그**(index.html + main.js): 근본 원인은 게임 화면 좌상단 `←`가 메뉴 화면과 동일한 순수 링크(`<a href="../../">`)라 게임 중에도 클릭 즉시 허브로 이탈. history 관리(popstate)도 전무. 게임 화면 ←에 `id="game-home"` 부여 후 click → `history.back()`. startGame에서 `history.pushState({screen:'game'})`로 진입 지점을 쌓고, `popstate`에서 playing/paused면 `backToMenu(true)`로 모드 선택 전환. 화면 ←와 폰 백버튼이 같은 경로(back→popstate→backToMenu)로 통일. backToMenu(fromPop) 인자로, 게임오버 모달 '홈' 등 직접 호출 시엔 쌓인 game 지점을 소비해 다음 시작의 push가 정상 동작하게 정리. 메뉴 화면 ←는 현행(허브로) 유지.
- **보스 유영 속도 절반**(numbers.js): `bobFreq` 0.3→0.15. 진폭(`bobAmp`)·가속 시간(`bobRamp`)은 불변이라 등장~최대까지 전 구간 각속도가 정확히 절반. bobFreq는 world.js updateBoss 위상 누적 1곳 전용(발사 타이밍 등 타 로직 무영향).
- **자동 플레이 AI 판단 구조 교체**(autopilot.js): 근본 진단 - 기존은 '한 걸음 앞'만 봄(후보로 가서 sim초 사는지만 보고 최선 1개 선택). 도착 뒤를 안 봐 곧 갇히는 막다른 구석으로 진입. 지금까지의 수정(몬테카를로 후보·SIM_T·보너스 가중치)은 전부 이 틀 안 상수 조정이라 체감 없었음. 처방 = 판단 방식 자체를 '여러 수 앞 계획'(2단계 빔서치)으로. simulate에 시작 시각 오프셋 `tOff` + 끝위치 반환 추가. decideTarget: 각 후보로 seg=simT/DEPTH초 굴려 도착점·첫 수 생존 계산 → 첫 수 완전 생존 상위 BEAM(6)개만 그 도착점에서 다음 수까지(tOff=seg) 이어 굴려 미래 생존 합산 → 두 수 총 생존 + 이득(조준/획득/여유공간)으로 점수. 첫 수만 살고 다음에 갇히는 목표는 낮게 매겨져 자진 회피. 실제 이동은 계획의 첫 수만(receding horizon). 실력 티어 sim을 2수로 분할. 나머지 함수(collectThreats/clearanceAt/gauss/autopilotStep)는 유지.
  - 방법 선택 경위: 사용자에게 3안(외부 리서치 검증 / 파이썬 강화학습 / JS 구조 교체) 제시. 사용자가 토큰 최소 기준으로 문의 → 구조 교체가 autopilot.js 한 파일만 고쳐 토큰 최소 + 근본(그리디) 직접 해결이라 답변, 사용자가 구조 교체로 결정. 파이썬 RL은 게임 환경 재현·학습·이식 다단계라 토큰 최대 + 결정론 테스트 체계 충돌로 이 규모엔 과투자로 판단(자비스 과거 'RL 오버킬' 즉흥 판단이 미세조정 반복→실패의 뿌리였음을 시인, 이번엔 판단 구조 자체를 교체).
  - **후속(사용자 "프로로 테스트했고 정말 멍청" 지적 → 재진단)**: 구조 교체만으론 여전히 멍청. 원인 2겹. (1) 자동 플레이 기본이 초보 티어(sim 0.5·threats 4·react 0.38)라 예측 시간·시야가 낮음 → 실력 티어 전반 상향(numbers.js autopilot.tiers: beginner sim0.5→0.9·threats4→7, pro sim1.7→2.1·threats999, react·aimDeg도 하향, 실력 단조 증가 유지). 사용자 선택 B안(실력 선택 살리되 각 값 상향). (2) **빔서치 편향 버그**: 두 번째 수까지 확장할 상위 후보를 (생존+이득) 순으로 골라, 조준·획득 이득이 붙은 목표(=적 근처)만 미래 평가를 받고 순수 회피·정지 목표는 저평가 → 항상 적 근처로 붙어 "일부러 총알 근처 가서 죽는" 증상. 수정 3건: 빔 선정을 순수 생존(_s1)으로(이득 제외), 미확장 안전 후보 미래를 seg로 낙관(0으로 안 깎아 회피 목표 저평가 방지), 위험 시 조준·획득 이득 ×0.25 억제 + 빈 공간(clearance) 선호를 위협 3개 이상일 때 상시 약(×0.35) 반영(탄 밀집 조준 억제). BEAM 6→10. 검증용 dev 훅 `ap`(실력 지정) 추가(localhost 한정).
