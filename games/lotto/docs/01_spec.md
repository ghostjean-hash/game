# 01. 게임 사양

## 1. 게임 한 줄

통계 데이터 + 캐릭터 시드 기반 한국 6/45 로또 번호 추천 RPG. 당첨 확률 향상이 아니라 선택 행위의 서사화.

## 2. 코어 루프

1. 캐릭터 선택 (또는 첫 진입 시 생성).
2. 회차 / 운세 확인.
3. 추천 카드 생성 (본번호 6 + 보너스 1).
4. 구매 인증 (선택, 실제 구매 후 사용자가 토글).
5. 다음 회차 결과 발표 후 자동 매칭.
6. 캐릭터 전적 / Luck 갱신.

## 3. 조작

3.1. 터치 우선 (모바일 PWA).
3.2. 키보드 보조: Tab 이동, Enter 확정, Esc 모달 닫기, 방향키 캐릭터 카드 전환.
3.3. 마우스는 데스크톱에서 터치와 동일 동작.

## 4. 화면 흐름 - 5탭 모델

```
[추첨 탭] (기본)
  ├─ 헤더 (Blessed Lotto 타이틀만, 부제 없음)
  ├─ Hero 카드 (카운트다운 + 추천 통합 박스)
  │   - 상단: 다음 추첨 카운트다운 (제N회 / YYYY-MM-DD 추첨 예정 / 일·시·분·초 1초 갱신)
  │   - 가로 구분선
  │   - 하단: 번호 패널 (본번호 6 + "+" + 보너스 1 / 라벨이 가로선에 박힌 형태)
  │   - 영역 라벨 ("추천번호" / "보너스번호")
  │   - 운세 흉/대길 시 카드 외곽 톤 변화
  ├─ 캐릭터 슬롯 (목록 / 전환만, 추가/삭제는 설정 탭으로 이동 - T1)
  ├─ 전략 가로 스크롤 탭 (11개 버튼 한 줄, 폭 초과 시 좌우 스크롤) + 활성 전략 설명
  └─ 캐릭터 카드 압축 (띠 / 별자리 / 일주 / MBTI / Luck 바 / 사주 행운 번호 7개 - T3)

(추첨 탭 면책 푸터는 제거. 면책은 첫 진입 모달과 설정 탭에서만 노출.)
(회차 nav 폐기. 추천 회차 = 카운트다운 회차 = `최신 + 1`로 자동 고정. 결정론 검증은 전적 탭의 history.)

[통계 탭] sticky 헤더 + 본번호/보너스/동시출현/Cold + 자동·명시 갱신.
[역추첨 탭] 사용자가 6개 골라 1~N회 매칭 → 최고 등수 + 회차 + 등수별 카운트 (S2-T1).
[전적 탭] 활성 캐릭터의 추천 / 적중 이력.
[휠링 페이지] 다구좌 OFF면 활성화 안내. ON이면 풀 / 휠 / 검증. 진입 동선은 설정 탭에서만 (S2-T2 백 버튼으로 복귀).
[설정 탭] 캐릭터 관리(목록 / 추가 / 삭제 / 활성) / 옵션 필터 / 다구좌 토글 + 휠링 진입 / 면책 / 데이터 메타 / 전체 초기화.

[하단 탭 바] 추첨 / 통계 / 역추첨 / 전적 / 설정 (5개, fixed bottom). 휠링은 다구좌 모드 ON 시 설정 탭에서만 진입.

[시트 / 모달]
  ├─ 첫 진입 면책 안내 (1회)
  ├─ 캐릭터 추가 폼 (+ 버튼)
  ├─ 전략 시트 (전략 바 클릭, 11개 그리드 + 1줄 설명)
  └─ 다구좌 윤리 안내 (첫 활성화)
```

4.1. **첫 진입**: 면책 모달 → 캐릭터 생성 폼(직접) → 추첨 탭.
4.2. **이후 진입**: 면책 스킵 → 추첨 탭 직행.
4.3. **추가 캐릭터**: 슬롯의 `+` → 모달 안 폼 → 추가 후 자동 활성화.
4.4. **캐릭터 전환**: 슬롯 클릭 → 활성 변경 → 즉시 추천/카드 갱신.
4.5. **활성 캐릭터 삭제**: 슬롯의 `×` → 확인 → 첫 캐릭터로 활성. 마지막 1명일 때 비활성.
4.6. **탭 전환**: 하단 탭 클릭 → `state.currentTab` 갱신 → 본문만 다시 렌더 (탭 / 헤더는 유지).
4.7. **전략 변경**: 추첨 탭의 가로 스크롤 전략 탭에서 직접 클릭 → 즉시 활성 변경 + 캐릭터 `lastUsedStrategy` 저장 + 재렌더. 시트 모달은 폐기.

4.7.1. **시각**: 11개 전략 버튼이 한 줄에 노출. 폭 초과 시 좌우 스크롤. 활성 전략 설명은 탭 줄 아래에 표시.
4.7.2. **스크롤바**: 완전 숨김 (`scrollbar-width: none` + `::-webkit-scrollbar { display: none }`). 모바일 PWA 톤 통일.
4.7.3. **fade affordance**: 컨테이너 양옆 24px fade gradient(`mask-image`)로 "더 있음" 암시. 끝 도달 시 해당 쪽 fade 자동 제거 (`.is-start` / `.is-end` 클래스).
4.7.4. **scroll-snap 미사용**: 초기엔 `scroll-snap-type: proximity`를 적용했으나 활성 탭 클릭 시 폭 변동 + snap 자동 보정으로 매번 미세 스크롤이 발생해 제거. 자유 스크롤이 더 안정적. 활성/비활성 탭 모두 동일 `font-weight: 600`로 폭 변동 자체도 방지.
4.7.5. **스크롤 위치 보존 + 활성 탭 잘림 보정**: 전략 클릭으로 재렌더되어도 `scrollLeft` 모듈 변수로 위치 복원. 단, 활성 탭이 좌/우 가장자리 fade 영역(24px) 안에서 잘려있으면 fade 안쪽으로 보이도록 자동 보정. 완전히 보이는 탭은 변동 0. 보정 후 위치는 다시 모듈 변수에 기억.
4.7.6. **PC 휠 → 가로 변환**: 모바일은 터치 swipe로 자연 동작. PC는 스크롤바 숨김 + 마우스 휠 기본이 세로라 스크롤 수단이 부재. `wheel` 이벤트에서 `|deltaY| > |deltaX|`인 경우(마우스 휠)만 `e.preventDefault()` + `scrollBy(deltaY)`로 변환. 트랙패드 가로 스와이프는 그대로 두어 충돌 없음.
4.8. **다구좌 활성화**: 휠링 탭 진입 시 OFF면 안내 + 켜기 버튼 → 윤리 모달 → ON 후 휠링 본문. 또는 설정 탭에서도 토글 가능.

## 5. 메커닉

### 5.1. 캐릭터

- 캐릭터 = **정체성** (이름 + 시드 + 띠 + 별자리 + Luck + 이력). 생성 시 1회 결정.
- 시드: 사용자 입력(생년월일 / 이름 / 행운의 단어) + 자동 계산된 별자리(02_data.md 2.6) 합쳐 결정론적 해시.
- Luck: 가중치 분산도 조절기 (0 = 균등, 100 = 시드 6개로 극도 쏠림). 당첨 확률은 동일.
- 운세 사이클: 회차별 컨디션 등급 (대길 / 길 / 평 / 흉). 캐릭터 12간지(생년 기반) + 회차 일진의 관계로 분포 결정 (5.1.1) + 캐릭터 일주(생년월일) + 회차 일주의 천간 오행 관계로 보정 (5.1.4 참조).

### 5.1.3. 추첨 전략 (캐릭터와 분리)

- 전략 12종: 축복받은 자 / 통계 추첨 / 보너스볼 사냥 / 미출현 회귀 / 짝꿍 번호 / 별자리 행운 / 최근 트렌드 / 직감 / 균형 조합 / MBTI 행운 / 별자리 4원소 / 일주 오행.
- **전략은 캐릭터 속성이 아닙니다.** 추첨 시 사용자가 매번 선택. 같은 캐릭터로 다른 전략 시도 가능.
- 마지막 선택은 캐릭터별로 `lastUsedStrategy`(단일 모드) / `lastUsedStrategies`(다중 모드)에 캐시 (캐릭터 전환 시 자동 복원).
- 메인 화면 상단의 전략 picker UI에서 변경.
- 카테고리 분류 (docs/02_data.md 1.5.2): 통계 5 / 운세 매핑(서양) 3 / 사주(동양) 1 / 랜덤 3.

#### 5.1.3.1. 다중 전략 모드 (S3-T1, 2026-05-02 신설)

- 활성: 설정 탭 "다중 전략 모드" 토글 ON (`options.multiStrategy = true`). 기본 OFF (라이트 사용자 비노출).
- 동작: 전략 탭이 단일 활성 변경에서 **토글**(추가/제거)로 동작. 1~6개 선택 가능. 만선 시 비활성 탭 회색 표시.
- 분배: 6 / N 균등 + 나머지 첫 N에 +1 (docs/02_data.md 1.5.4).
- 출처: 추천 카드 본번호 아래 카테고리 색 dot (통계 파랑 / 운세 매핑 분홍 / 사주 황 / 랜덤 회색).
- 의도: 사용자가 "전략 추출 신뢰 없음" 의심을 즉시 시각화로 해소. 어느 번호가 어느 전략에서 나왔는지 한눈에.

#### 5.1.3.2. 5세트 동시 추천 모드 (S4-T1, 2026-05-02 신설)

- 활성: 설정 탭 "5세트 동시 추천" 토글 ON (`options.fiveSets = true`). 기본 OFF.
- 의도: "한 회차에 캐릭터가 시도해볼 수 있는 5가지 다른 결과"를 한 번에 노출. 결정론을 깨지 않으면서도 서사를 풍부하게.
- 데이터 / 알고리즘: docs/02_data.md 1.5.5 (시드 변형 룰).
- UI: 추첨 탭 hero에 메인(`#1`) 카드 1장 + 컴팩트 4장(`#2~#5`) 세로 스택. `#1`은 기존 hero 카드와 동일 스펙(5.2.1). `#2~#5`는 헤더 / 배너 없이 번호줄 + 보너스 + 작은 인덱스 라벨만(높이 ~1.6배 압축).
- 다중 전략 모드 호환: `multiStrategy` ON+OFF 모두 동작. 둘 다 ON이면 5장 모두 카테고리 dot 표시.
- 이력 / Luck / 의식 보너스 영향: 모두 `#1`만. `#2~#5`는 표시 전용. 백캐스트(`backfillRecommendations`)는 단일 시드로 결정론 유지(영향 없음).
- 면책: 컴팩트 카드 묶음 아래에 "5세트는 '한 회차의 다양한 시도'를 보여주는 콘텐츠입니다. 5장 구매 권유가 아니며, 당첨 확률 변화도 없습니다." 카피 1회 노출.
- 사행성 회피: 라벨 "5세트 동시 추천" / "다른 시도". "5장 사면 5배" 류 비교 절대 금지(CLAUDE.md 6.3 일관성).
- **#2~#5 과거 매칭 chip (S5-T1, 2026-05-02)**: 컴팩트 카드 우측에 작은 chip `과거 최고 N등 (M회)` 표시. `core/reverse.js` `reverseSearch` 재활용. `bestRank=null`인 세트는 "미적중 이력만". #1은 hero 카드에 정보 과밀이라 chip 미표시. 면책 카피에 "과거 매칭은 미래 적중률과 무관" 추가.

#### 5.1.1. 운세 분포 (12간지 관계별)

| 관계 | 의미 | 대길 | 길 | 평 | 흉 |
|---|---|---|---|---|---|
| same | 동일 띠 | 30% | 40% | 25% | 5% |
| sahap | 삼합 (±4 / ±8) | 20% | 40% | 30% | 10% |
| chung | 충 (±6, 정반대) | 5% | 25% | 30% | 40% |
| normal | 그 외 | 10% | 35% | 40% | 15% |

5.1.1.1. 캐릭터 띠는 생년에서 자동 계산되어 영구 보존.
5.1.1.2. 회차 일진: `drwDate`가 있으면 1984-02-02 갑자일 기준 일자 차이로 정밀 산출. 없으면 `drwNo mod 12`로 fallback.
5.1.1.3. 등급 결정은 시드 + 회차 mix 후 mulberry32 1샘플로 분포 가중. 같은 입력 = 같은 등급.

### 5.2. 추첨 절차

1. 시드 + 클래스 정책 → base weight (1~45).
2. 사용자 토글된 통계 필터 적용.
3. Luck 분산도로 weight 스케일.
4. 비복원 가중 추출로 본번호 6개.
5. 보너스볼 가중치(클래스별) → **본번호 6개를 풀에서 제외**한 후 보너스 1개 추출. 6/45 룰상 본번호와 보너스는 7개가 모두 서로 다른 번호여야 함.
6. 추천 카드 + 번호별 근거 메타데이터.

5.2.0.1. **6/45 룰 보너스 제약**: 보너스는 본번호 6개와 겹칠 수 없다. `core/recommend.js` `weightedSample(weights, count, seed, exclude)`의 `exclude` 인자에 본번호 Set을 전달해 풀에서 사전 제외. 테스트는 50개 시드 + 11개 전략 모두에서 `bonus ∉ numbers` 검증.

### 5.2.1. 추천 카드 시각 규격 (동행복권 결과 페이지 호환 패널)

추천 카드의 번호 패널은 동행복권 결과 페이지와 시각적으로 동일한 레이아웃을 채택합니다. 단, 라벨은 추천의 의미를 보존합니다.

5.2.1.1. **번호공**: 결과 페이지 변종 5색 (docs/02_data.md 2.4). 본번호와 보너스볼 **완전 동일 크기**. **입체감 / 그림자 / 그라디언트 일체 없음**. 단색 평면 원 + 흰 글자.
5.2.1.2. **레이아웃**: 본번호 6개 → "+" → 보너스볼 1개를 한 줄 그리드(6fr : auto : 1fr)로 정렬. 보너스볼 위 라벨 없음. 본번호 칸 폭과 보너스 칸 폭이 동일하므로 공 크기도 자동 동일.
5.2.1.3. **구분선**: 번호공 줄 위 가로선 **없음**. 라벨이 가로선의 일부로 박힌 패턴(legend-in-line)만 사용.
5.2.1.4. **영역 라벨**: 본번호 영역 아래 "추천번호", 보너스볼 영역 아래 "보너스번호". 각 라벨은 좌우로 가로선이 뻗어 라벨 글자가 선의 가운데를 끊는 형태(`::before` / `::after` 1px 라인). 라벨 글자만큼만 흰 공간으로 유지.
5.2.1.5. **사행성 표현 회피**: 라벨에 "당첨번호" 사용 금지. 추천 카드는 본질적으로 추천이므로 "추천번호"로 표기 (CLAUDE.md 6.3 일관성).
5.2.1.6. **+ 기호**: SVG plus 아이콘 (docs/04_conventions.md 8장). 진한 회색(#444), stroke-width 3 굵게.
5.2.1.7. **폰트**: 한글은 `Noto Sans KR` 웹폰트 최우선(index.html에서 Google Fonts 로드). 시스템 폴백 `Apple SD Gothic Neo` / `Malgun Gothic` / system-ui. 번호 숫자는 700 굵기.

### 5.2.2. 다음 추첨 카운트다운 카드

5.7.6. **면책 카피**: "로또 6/45는 매 회차 독립 시행이므로 과거 매칭 횟수가 미래 적중률을 보장하지 않습니다. 참고용."
5.7.7. **위치**: `src/core/reverse.js` (순수 로직), `src/render/reverse-page.js` (UI), `tests/suites/reverse.test.js`.
5.7.8. **동률 다회차 펼치기 (S5-T3, 2026-05-02)**: `bestRankCount > 1` (최고 등수 매칭이 2회 이상)인 경우 결과 카드 아래 "전체 보기" 버튼 노출. 클릭 시 `core/reverse.js` `findMatchingDraws(numbers, draws, rank)`로 해당 등수의 모든 매칭 회차를 모달로 표시(회차 번호 + 추첨일 + 발표 번호 + 일치 강조). 기본 결과는 가장 최근 1건 유지(현재 동작 호환).

## 6. 성공 이벤트 / 실패 이벤트 / 종료 조건

6.1. **성공 이벤트**: 4등 이상 적중. 카드 글로우 이펙트 + 캐릭터 전적 갱신 + Luck 보너스.
6.2. **실패 이벤트**: 없음. 미적중은 정상 흐름.
6.3. **종료 조건**: 없음. 캐릭터 삭제 외 영속.

## 7. 진행 시스템

7.1. 회차별 추천 / 이력 자동 누적.
7.2. Luck 스탯 성장: 등수 매칭이 처음 결정될 때 1회 보너스 (1등 +20 ~ 5등 +2). 출석 / 미당첨 누적 / 길일 보너스는 후순위(M5 3단계 이후).
7.3. 캐릭터 전적: 누적 적중률, 최고 등수, 이력 카드.
7.4. 마일스톤은 회차 누적 또는 컬렉션 도달도. 강제 종료 없음.

### 7.5. 백캐스트 (Luck 부트스트랩)

추천 회차는 항상 **다음 추첨(미래)**이고 발표 전이라 매칭이 영원히 안 일어난다 (4.x). 이를 보강하기 위해 캐릭터 첫 추첨 탭 진입 시 최근 N회(`BACKFILL_RECENT_COUNT`, docs/02_data.md 1.16)에 대해 결정론적 추천을 history에 backfill + 매칭한다.

7.5.1. 위치: `core/history.js` `backfillRecommendations(character, draws, strategyId, stats, lastN)`.
7.5.2. 결정론: 같은 캐릭터 + 같은 draws + 같은 통계 = 같은 백필 결과 (같은 시점 통계를 모든 회차에 사용).
7.5.3. idempotent: 이미 history에 있는 회차는 건너뜀. 매 진입마다 호출해도 부담 없음.
7.5.4. 매칭은 즉시. 적중된 회차는 `applyLuckGrowth`가 다음 단계에서 luck 보너스 적용.
7.5.5. 사용자가 다음 추첨 회차를 1회 보면 backfill 30회 + 다음 회차 1개 = history 31개.
7.5.6. 통계 정확도 vs 단순성 트레이드: 회차별 그 시점의 통계가 아니라 현재 시점 통계를 모든 백필에 사용. 의미는 "이 캐릭터의 현재 능력으로 과거 N회를 시뮬한다면". 회차별 시점 통계 백캐스트가 필요해지면 별도 모듈로 분리.
