# 01. 게임 사양

## 1. 게임 한 줄

통계 데이터 + 캐릭터 시드 기반 한국 6/45 로또 번호 추천 RPG. 당첨 확률 향상이 아니라 선택 행위의 서사화.

## 2. 코어 루프

1. 캐릭터 선택 (또는 첫 진입 시 생성).
2. 회차 / 운세 확인.
3. 추천 카드 생성 (본번호 6 + 보너스 1).
4. 구매 인증 (선택, 실제 구매 후 사용자가 토글).
5. 다음 회차 결과 발표 후 자동 매칭.
6. 캐릭터 전적 / Luck 갱신.

## 3. 조작

3.1. 터치 우선 (모바일 PWA).
3.2. 키보드 보조: Tab 이동, Enter 확정, Esc 모달 닫기, 방향키 캐릭터 카드 전환.
3.3. 마우스는 데스크톱에서 터치와 동일 동작.

## 4. 화면 흐름

```
[홈] (메인)
  ├─ 캐릭터 슬롯 (목록 / 전환 / + 추가 / × 삭제)
  ├─ 활성 캐릭터 카드 (클래스 / 이름 / 운세 / Luck)
  ├─ 추천 카드 (회차 / 본번호 6 + 보너스 1 / 근거)
  ├─ 회차 이동 (‹ 이전 / 다음 ›)
  └─ 면책 푸터

[모달]
  ├─ 첫 진입 면책 안내 (1회)
  └─ 캐릭터 추가 폼 (+ 버튼 클릭 시)

[통계 탭]                  → 본번호 / 보너스 / 동시출현 (M3 예정)
[이력 탭]                  → 회차별 추천 / 적중 (M4 예정)
[설정]                     → 토글 / 면책 안내 / 데이터 초기화 (M5+ 예정)
```

4.1. **첫 진입**: 면책 모달 → 캐릭터 생성 폼(직접) → 메인.
4.2. **이후 진입**: 면책 스킵 → 메인 직행 (캐릭터 있으므로).
4.3. **추가 캐릭터**: 슬롯의 `+` 버튼 클릭 → 모달 안 폼 → 추가 후 자동 활성화.
4.4. **캐릭터 전환**: 슬롯 클릭 → 활성 변경 → 카드 / 추천 즉시 갱신.
4.5. **활성 캐릭터 삭제**: 슬롯의 `×` 버튼 → 확인 → 첫 캐릭터로 활성 전환. 마지막 1명일 때 비활성.

## 5. 메커닉

### 5.1. 캐릭터

- 캐릭터 = **정체성** (이름 + 시드 + 띠 + 별자리 + Luck + 이력). 생성 시 1회 결정.
- 시드: 사용자 입력(생년월일 / 이름 / 별자리 / 행운의 단어)을 결정론적 해시.
- Luck: 가중치 분산도 조절기 (0 = 균등, 100 = 시드 6개로 극도 쏠림). 당첨 확률은 동일.
- 운세 사이클: 회차별 컨디션 등급 (대길 / 길 / 평 / 흉). 캐릭터 12간지(생년 기반) + 회차 일진의 관계로 분포 결정 (5.1.1) + 캐릭터 일주(생년월일) + 회차 일주의 천간 오행 관계로 보정 (5.1.4 참조).

### 5.1.3. 추첨 전략 (캐릭터와 분리)

- 전략 11종: 축복받은 자 / 통계학자 / 2등의 별 / 회귀주의자 / 짝궁추적자 / 점성술사 / 추세추종자 / 직감주의자 / 균형주의자 / MBTI / 별자리 원소.
- **전략은 캐릭터 속성이 아닙니다.** 추첨 시 사용자가 매번 선택. 같은 캐릭터로 다른 전략 시도 가능.
- 마지막 선택은 캐릭터별로 `lastUsedStrategy`에 캐시 (캐릭터 전환 시 자동 복원).
- 메인 화면 상단의 전략 picker UI에서 변경.

#### 5.1.1. 운세 분포 (12간지 관계별)

| 관계 | 의미 | 대길 | 길 | 평 | 흉 |
|---|---|---|---|---|---|
| same | 동일 띠 | 30% | 40% | 25% | 5% |
| sahap | 삼합 (±4 / ±8) | 20% | 40% | 30% | 10% |
| chung | 충 (±6, 정반대) | 5% | 25% | 30% | 40% |
| normal | 그 외 | 10% | 35% | 40% | 15% |

5.1.1.1. 캐릭터 띠는 생년에서 자동 계산되어 영구 보존.
5.1.1.2. 회차 일진: `drwDate`가 있으면 1984-02-02 갑자일 기준 일자 차이로 정밀 산출. 없으면 `drwNo mod 12`로 fallback.
5.1.1.3. 등급 결정은 시드 + 회차 mix 후 mulberry32 1샘플로 분포 가중. 같은 입력 = 같은 등급.

### 5.2. 추첨 절차

1. 시드 + 클래스 정책 → base weight (1~45).
2. 사용자 토글된 통계 필터 적용.
3. Luck 분산도로 weight 스케일.
4. 비복원 가중 추출로 본번호 6개.
5. 보너스볼 가중치(클래스별) → 보너스 1개 추출.
6. 추천 카드 + 번호별 근거 메타데이터.

### 5.3. 통계 축 (가중치 소스)

- 본번호: 누적 빈도 / Hot·Cold(최근 10/30/100) / 동시출현 매트릭스 / 미출현 갭.
- 보너스볼: 별도 빈도 / Hot·Cold(최근 30) / 보너스-본번호 동반.
- 비율 필터(토글): 번호합, 홀짝, AC값, 연속수, 끝수합. 다수 영역 수치는 docs/02_data.md.

### 5.3.1. 통계 페이지 갱신 흐름

5.3.1.1. **메타 라인**: 헤더 아래에 "{최신 회차}회까지 반영 · 최근 추첨 YYYY-MM-DD" 표시. 사용자가 한눈에 데이터 신선도 판단.
5.3.1.2. **자동 갱신**: 페이지 진입 시 `syncDrawsIfNewer()` 백그라운드 호출.
- 미러 `latest.json` 한 방 GET → cached max drwNo와 비교.
- 새 회차 있을 때만 정적 `draws.json` 다시 fetch + saveDraws + 재렌더 + 토스트("새 회차 N회 반영됨").
- 새 회차 없으면 무동작 (사용자에게 메시지 없음).
5.3.1.3. **명시 갱신 버튼**: 헤더 우측 "↻ 갱신". 클릭 시 같은 흐름이지만 결과를 항상 토스트로 통보.
- 새 회차 있음 → "새 회차 N회 반영됨 (YYYY-MM-DD)".
- 이미 최신 → "이미 최신 데이터입니다".
- 미러 도달 실패 → "미러 연결 실패. 잠시 후 다시 시도하세요".
- CI 지연 (미러는 새거지만 정적 번들 미갱신) → "미러는 새거지만 정적 데이터가 아직 갱신되지 않았습니다".
5.3.1.4. **트리거 비용**: latest.json은 ~360B 단일 GET. 진입마다 호출해도 부담 없음.
5.3.1.5. **새 회차 없을 때 무동작 원칙**: 사용자가 "날짜 기준 새 정보 있을 때만"이라 명시 (2026-05-01 결정).

### 5.4. 결정론성

- 같은 캐릭터 + 같은 회차 + 같은 토글 = 같은 결과.
- 모든 가중치 토글 OFF = 균등 랜덤.

### 5.1.4. 사주 정밀화 (오행 보정)

캐릭터 생년월일 → 일주(천간 + 지지) 자동 계산. 회차 발표일 → 일주. 두 일주의 천간 오행 관계(self / generate / beGenerated / overcome / beOvercome / normal)에 따라 5.1.1 분포에 ±보정 가산 후 정규화.

5.1.4.1. 보정 강도는 ±0.07 이내 (분포 자체가 뒤집히진 않음).
5.1.4.2. 회차 `drwDate` 없으면 (페치 전) 보정 미적용 - 띠 관계 분포 그대로.
5.1.4.3. 사주 정밀화는 운세 분포의 풍부함 강화. 당첨 확률 영향 없음.
5.1.4.4. 자세한 매핑은 docs/02_data.md 1.12 참조.

### 5.1.2. 길일 / 흉일 UI

- 흉일(`bad`): 캐릭터 카드 외곽 빨간 톤 + 추천 카드 상단 "흉일. 방어 모드 권장" 배너.
- 대길일(`great`): 캐릭터 카드 외곽 골드 톤 + 추천 카드 상단 "대길" 배너.
- 길(`good`) / 평(`neutral`)은 추가 톤 없음 (기본 카드).

### 5.5. 휠링 (다구좌 모드, 옵션)

5.5.1. **목적**: 부분 당첨 보장 도구. 1등 확률 향상 아님.
5.5.2. **활성화**: `options.advancedMode` 토글 ON. 첫 ON 시 윤리 안내 모달 강제.
5.5.3. **노출 정책**: 라이트 사용자(기본 OFF)에는 비노출. 토글 ON 시 메인 화면에 "휠링 보기" 버튼 출현.
5.5.4. **휠 종류**:
- **Full Wheel**: 풀의 모든 6 조합 `C(N,6)`. 4-if-4 자동 보장.
- **Abbreviated Wheel**: 시드 기반 결정론적 N장 선택. 4-if-4 보장은 자동 검증 후 통과 / 미통과 표시.
5.5.5. **풀 권장 크기**: 8~10개 (티켓 28~210장). 12개 초과 시 비용 폭증 경고.
5.5.6. **티켓 비용**: 1티켓 = 1,000원 (실제 로또 가격). 풀 크기별 예상 비용 표시.
5.5.7. **금지 표현**: "확률 향상" / "필승" / "1등 보장 (조건 없는)". 허용: "부분 당첨 보장", "참고용", "풀의 K개 일치 시 N등 M장 이상".

## 6. 성공 이벤트 / 실패 이벤트 / 종료 조건

6.1. **성공 이벤트**: 4등 이상 적중. 카드 글로우 이펙트 + 캐릭터 전적 갱신 + Luck 보너스.
6.2. **실패 이벤트**: 없음. 미적중은 정상 흐름.
6.3. **종료 조건**: 없음. 캐릭터 삭제 외 영속.

## 7. 진행 시스템

7.1. 회차별 추천 / 이력 자동 누적.
7.2. Luck 스탯 성장: 등수 매칭이 처음 결정될 때 1회 보너스 (1등 +20 ~ 5등 +2). 출석 / 미당첨 누적 / 길일 보너스는 후순위(M5 3단계 이후).
7.3. 캐릭터 전적: 누적 적중률, 최고 등수, 이력 카드.
7.4. 마일스톤은 회차 누적 또는 컬렉션 도달도. 강제 종료 없음.

## 8. 모바일 지원

8.1. 모바일 우선 PWA. 가로 / 태블릿 모드는 좌측 캐릭터 카드 + 우측 통계 미니뷰 동시 표시.
8.2. 오프라인: SW 캐시. 회차 데이터 없으면 마지막 캐시로 동작.
