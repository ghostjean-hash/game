# 01. 게임 사양

## 1. 게임 한 줄

통계 데이터 + 캐릭터 시드 기반 한국 6/45 로또 번호 추천 RPG. 당첨 확률 향상이 아니라 선택 행위의 서사화.

## 2. 코어 루프

1. 캐릭터 선택 (또는 첫 진입 시 생성).
2. 회차 / 운세 확인.
3. 추천 카드 생성 (본번호 6 + 보너스 1).
4. 구매 인증 (선택, 실제 구매 후 사용자가 토글).
5. 다음 회차 결과 발표 후 자동 매칭.
6. 캐릭터 전적 갱신. (Luck 자산 S089에서 전면 폐기, 2026-05-17 사용자 명시.)

## 3. 조작

3.1. 터치 우선 (모바일 PWA).
3.2. 키보드 보조: Tab 이동, Enter 확정, Esc 모달 닫기, 방향키 캐릭터 카드 전환.
3.3. 마우스는 데스크톱에서 터치와 동일 동작.

4.1. **첫 진입**: 면책 모달 → 캐릭터 생성 폼(직접) → 추천 탭.
4.2. **이후 진입**: 면책 스킵 → 추천 탭 직행.
4.3. **추가 캐릭터**: 슬롯의 `+` → 모달 안 폼 → 추가 후 자동 활성화.
4.4. **캐릭터 전환**: 슬롯 클릭 → 활성 변경 → 즉시 추천/카드 갱신.
4.5. **활성 캐릭터 삭제**: 슬롯의 `×` → 확인 → 첫 캐릭터로 활성. 마지막 1명일 때 비활성.
4.6. **탭 전환**: 하단 탭 클릭 → `state.currentTab` 갱신 → 본문만 다시 렌더 (탭 / 헤더는 유지).
4.7. **전략 변경**: 추천 탭의 카테고리별 그룹 전략 탭에서 직접 클릭 → 즉시 활성 변경 + 캐릭터 `lastUsedStrategy` 저장 + 재렌더. 시트 모달은 폐기.

4.7.1. **시각 (S9 + S10 + S11, 2026-05-02)**: 10개 전략 버튼을 카테고리별로 3줄에 묶어 노출 (S34 짝꿍 폐기 후 10종). 줄 순서 = 운세(3) → 랜덤(3) → 통계(4). 캐릭터 정체성/콘텐츠 우선, 객관 통계는 후순위(S11). 줄 앞에 카테고리 라벨 chip(카테고리 색). 한 줄 폭 초과 시 wrap (가로 스크롤 미사용). 활성 전략 설명은 그룹 아래에 표시.
4.7.2. **카테고리 라벨 chip**: 줄 시작에 작은 chip(`.strategy-row-label`). 카테고리 색 cascade(`is-stats` / `is-mapping` / `is-random`). 클릭 불가, aria-hidden. 라벨 텍스트는 "통계" / "운세" / "랜덤".
4.7.3. **wrap 정책**: 카테고리 줄 컨테이너는 `display: flex; flex-wrap: wrap`. 한 줄당 1~5개라 데스크톱 / 표준 모바일 모두 한 줄 fit, 좁은 화면에서 자연 줄 바꿈.
4.7.4. **폐기 항목 (S9 이전 가로 스크롤 정책)**: 단일 줄 가로 스크롤이 카테고리 그룹화로 대체되어 다음이 모두 불필요해 폐기: 컨테이너 양옆 mask fade(`is-start` / `is-end`), `scrollLeft` 모듈 변수 보존, 활성 탭 잘림 보정, PC 휠 → 가로 변환. 스크롤바 숨김(`scrollbar-width: none`)은 wrap 컨테이너에 가로 스크롤 미발생이라 자동 무관.
4.7.5. **활성 / 비활성 폭 변동 방지**: 활성 / 비활성 탭 모두 동일 `font-weight: 600`로 클릭 시 폭 변동 0 (S9 이전 정책 유지).
4.8. **다구좌 활성화**: 휠링 탭 진입 시 OFF면 안내 + 켜기 버튼 → 윤리 모달 → ON 후 휠링 본문. 또는 설정 탭에서도 토글 가능.

### 5.1. 캐릭터

- 캐릭터 = **정체성** (이름 + 시드 + 띠 + 별자리 + 이력). 생성 시 1회 결정. (Luck 자산 S089에서 전면 폐기, 2026-05-17.)

- ~~Luck: 가중치 분산도 조절기~~ **S089 (2026-05-17) 전면 폐기**. 사용자 명시 "Luck을 게임요소로 추가하고 싶은 생각 없음" + 낮은 Luck 점수가 부정적 인상 유발. 캐릭터 schema에서 luck 필드 제거 + BLESSED 전략의 luck-비례 boost 폐기 (BLESSED는 시드 6번호 단순 가중치 +0.5로 단순화). ritual 만땅 보상 +5도 폐기 (잠금만 유지).
- 운세 사이클: 회차별 컨디션 등급 (대길 / 길 / 평 / 흉). 캐릭터 12간지(생년 기반) + 회차 일진의 관계로 분포 결정 (5.1.1) + 캐릭터 일주(생년월일) + 회차 일주의 천간 오행 관계로 보정 (5.1.4 참조).

### 5.1.3.0. 추천 알고리즘 architecture (S43, 2026-05-08 재구축)

**핵심 결정**: 사용자 통찰 "알고리즘 근본 자체 잘못. 부분 fix 의미 없다" 반영. 옛 architecture 폐기 + 단일 추첨 architecture 도입.

#### 옛 architecture 결함 (S0~S42 누적 결함)

- 다중 strategy 분배(`distributeCounts`)로 6번호를 N개 strategy에 2개씩 균등 분할 → 각 풀 별도 추첨 → 정렬 시 끝자리 패턴 충돌(2/3/4, 21/22/24 인접 클러스터링).
- 풀 컷팅(`STATS_POOL_SIZE`)으로 풀 좁아 결정론 채택. 풀 안 1-9 비중이 그대로 노출.
- Luck 25배 보너스(`WEIGHT_MAX_BIAS = 50`)로 시드 6번호 채택 84%. 캐릭터 시드에 작은 번호 포함 시 무한 반복.
- balancer 합 필터(121-160 → 100-180)로 작은 번호 자주 통과.
- S33 풀 외 차단으로 학설 풀 외 번호 절대 안 나옴 → 분포 부자연.

#### 새 architecture (S43 단일 추첨)

1. **단일 weight 벡터**: 모든 활성 strategy의 가중을 1-45 base 1.0 + 학설/통계 보너스 +0.3~+0.5로 합성. (S089 전 옛 표현 "Luck 보너스" = BLESSED 시드 6번호 boost를 의미했으나 S089 Luck 폐기로 boost 고정 +0.5).
2. **`weightedSample` 1번 호출**: 6번호 단일 추첨. 인접 클러스터링 자연 해소.
3. **다중 분배 폐기**: `distributeCounts` / `computeStrategyContext` / 잘라쓰기 휴리스틱 모두 우회.
4. **풀 컷팅 폐기**: 학설 = 정의가 아닌 약한 가중치. 1-45 전체 추첨 가능.
5. **BLESSED 시드 6번호 boost**: +0.5 고정 (base 1.0의 1.5배). S089 이전에는 Luck 비례 0~0.5였으나 Luck 폐기로 고정값 채택.
6. **balancer post-filter 폐기**: 합 필터 안 함. 자연 통과 빈도.
7. **풀 외 차단(S33) 폐기**: 학설 가중일 뿐. 풀 외도 base weight로 추첨.

#### 순수 운세(학설) 프리셋 풀 제한 (2026-07-14 복원, 사용자 결정 B안)

**배경 (사용자 보고)**: "운세 프리셋에서 나와서는 안 되는 번호가 추천된다." 진단 결과 - 추첨 자체는 정상(범위/중복/보너스 충돌 0)이나, 순수 학설 프리셋에서 7항(전 범위 추첨)으로 뽑힌 "행운 번호 아닌 번호"가 평균 2.55/6개(42.5%) 발생하고, 이들이 `assignSourcesForNumber` 6번 폴백(`strategyIds[0]`)으로 100% 학설 라벨(별자리 등)을 강제로 달아 "운세가 고른 번호"처럼 오표시됨.

**규칙**: 활성 전략이 **전부 운세 카테고리**(`STRATEGY_CATEGORIES[sid] === 'mapping'`, 즉 별자리/사주/4원소)일 때만, 그 학설들의 행운 번호 합집합(`computePoolForStrategies`) 안에서만 추첨한다. `weightedSample`의 `exclude`로 union 밖 번호를 전부 제외.

**풀 안 순수 무작위 (사용자 결정, 2026-07-14)**: union 안에서는 어떤 가중·조작도 없이 `uniformWeights()`(전부 1.0)로 순수 무작위 추첨한다. 사용자 명시 - "의도 없이 그냥 순수 랜덤, 특정 번호를 몰아주지 마라."

- 검토 후 **미채택**한 대안 (사용자 거부): (1) 다중 학설 매칭 boost(작은 수 과대표집 = '앞번호 반복' 유발)는 조작이라 제거함. (2) 세트별 대역 편중으로 대역 분포를 들쭉날쭉하게 만드는 방식은 "인위적 몰아주기"라 사용자가 명시 거부 → 미채택.
- 순수 무작위의 자연 결과로 남는 특성(조작 아님): 운세 행운 번호가 각 대역에 등차로 고르게 분포하고 union이 좁아(18~23개), 대역 분포가 실제 로또보다 약간 균질하고(진단: 빈 대역 평균 1.07 < 로또 1.23) 좁은 풀에서 특정 번호가 반복될 수 있다. 이는 풀 구조에서 오는 결과이며 추첨 자체는 조작 없는 순수 무작위다. 학설 매칭 색점은 `assignSourcesForNumber`가 lucky 집합 기반 별도 산출이라 무관.

- 학설 + 비학설(통계/랜덤) **혼합** 프리셋은 4항/7항(전 범위 추첨) 유지. 통계·랜덤은 1~45 전체가 정의라 union 개념이 성립 안 함.
- union 크기 < `PICK_COUNT`(6)면 안전하게 전 범위 fallback(실제 학설 최소 9개라 미발생).
- **부수 효과**: 순수 학설 프리셋의 모든 추천 번호가 union 안 → `assignSourcesForNumber`가 항상 학설 매칭 성공 → 6번 "첫 전략 폴백"이 순수 학설 프리셋에서 도달 불가 = 오라벨 문제 동시 해소.
- 보너스 추첨은 union 제한 없음(보너스는 출처 라벨 대상 아님, 본번호 충돌만 회피).

#### 5.2.5.1. 모델

- **조립식 = strategy 조합 + 캐릭터 시드 + 회차**.
- 미리보기 단계 없음. 사용자가 strategy 토글로 조립식 정의 → **"+ 1세트"** 또는 **"+ 5세트"** 클릭 = 즉시 추천1, 추천2, ...로 누적 push.

- 같은 numbers 조합은 자동 skip (중복 차단).
- 결정론: 같은 strategy 조합 + 같은 시점 클릭 = 같은 결과. 미리보기 없어도 strategy 토글 상태가 결과를 1대1 결정.

[캐릭터 슬롯 + 카드]

흐름:
1. 사용자가 전략 한 번 토글(조립).
2. 시선을 위로 살짝 올려 + 버튼 클릭(실행).
3. 결과가 + 버튼 바로 위 추천 리스트에 즉시 누적 = 시선 0 왕복.
4. 더 추가할 거면 + 버튼 반복 클릭 (전략 변경 불필요한 한 + 버튼만 보면 됨).

6. 모두 비우려면 **헤더 우측 "🗑 모두 비우기"** (S095-후속 2: 액션바 → 헤더로 이동, 라벨 정정).

**시각**:
- 추천 리스트 박스 = 위쪽 hero(.home-hero)와 동일 라운드 사각(`--radius-lg` = 16px).
- 헤더 = grid 3열 (spacer / title 가운데 / `.saved-sets-header-right`). 우측 wrapper에 카운터 + "모두 비우기" 작은 ghost 버튼.
- "모두 비우기" 버튼 = 시각 약화 (opacity 0.7, border 없음, dim 색). hover/focus 시 danger 톤.
- 액션바 (S095-후속 2 단순화) = flex column. + 1세트 / + 5세트 가로 가운데. hint 2행 가운데. "전체 삭제" 액션 폐기.

#### 5.2.5.4. Cap + 중복 차단 + 풀 한계 (S32, 2026-05-07 강화)

**Cap**: `SAVED_SETS_CAP = 20`. 도달 시 + 1세트 / + 5세트 비활성. "+ 5세트"는 잔여 < 5면 비활성. 사행성 톤("여러 세트 = 적중 ↑") 회피용.

**중복 차단 (dedupe)**: 같은 numbers 6개 조합은 자동 skip (`hasSameNumbers`). 추첨 결정론 + dedupe 조합으로 누적 리스트 안 본번호 6개는 모두 서로 다른 조합 보장 (보너스 무관 - 본번호 dedupe만으로 시드 자연 분기).

- 비율 필터(토글): 번호합, 홀짝, AC값, 연속수, 끝수합. 다수 영역 수치는 docs/02_data.md.

### 5.3.1. 통계 페이지 갱신 흐름

5.3.1.1. **메타 라인**: 헤더 아래에 "{최신 회차}회까지 반영 · 최근 추첨 YYYY-MM-DD" 표시. 사용자가 한눈에 데이터 신선도 판단.
5.3.1.2. **자동 갱신**: 페이지 진입 시 `syncDrawsIfNewer()` 백그라운드 호출.
- 미러 `latest.json` 한 방 GET → cached max drwNo와 비교.
- 새 회차 있을 때만 정적 `draws.json` 다시 fetch + saveDraws + 재렌더 + 토스트("새 회차 N회 반영됨").
- 새 회차 없으면 무동작 (사용자에게 메시지 없음).
5.3.1.3. **명시 갱신 버튼**: 헤더 우측 "↻ 갱신". 클릭 시 같은 흐름이지만 결과를 항상 토스트로 통보.
- 새 회차 있음 → "새 회차 N회 반영됨 (YYYY-MM-DD)".
- 이미 최신 → "이미 최신 데이터입니다".
- 미러 도달 실패 → "미러 연결 실패. 잠시 후 다시 시도하세요".
- CI 지연 (미러는 새거지만 정적 번들 미갱신) → "미러는 새거지만 정적 데이터가 아직 갱신되지 않았습니다".
5.3.1.4. **트리거 비용**: latest.json은 ~360B 단일 GET. 진입마다 호출해도 부담 없음.
5.3.1.5. **새 회차 없을 때 무동작 원칙**: 사용자가 "날짜 기준 새 정보 있을 때만"이라 명시 (2026-05-01 결정).

### 5.4. 결정론성

- **시드 의존 전략(5개, S34 짝꿍 폐기로 6 → 5)**: 같은 캐릭터 + 같은 회차 + 같은 토글 = 같은 결과. blessed / astrologer / intuitive / zodiacElement / fiveElements.
- **객관 전략(5개, docs/02_data.md 1.5.1)**: 같은 회차 + 같은 통계 = 모든 캐릭터에 동일 결과 (캐릭터 시드 무관). 통계가 객관 데이터이므로 캐릭터에 따라 변하면 안 됨. statistician / secondStar / regressionist / trendFollower / balancer. (S089 Luck 폐기로 "Luck 무관" 조항 자연 흡수.)
- 모든 가중치 토글 OFF + `blessed` = 균등 랜덤.

### 5.4.1. 풀 외 번호 차단 (S33, 2026-05-08)

- **별자리 / 4원소 / 사주 추첨 결과는 학설 풀 안 번호로만 구성된다.** 풀 외 번호는 weight 0 유지로 추첨 자체에서 차단.

**폐기 사유 (사용자 명시)**: "진짜를 돌리고 싶다" + "추천 번호를 직접 선택해야 한다". 백캐스트 = 캐릭터 첫 진입 시 가짜 30회 history 자동 채움 → 사용자 명시 행동 없는 자동 누적이 "내 적중률 N%"라는 진짜 데이터로 오인 = 정직성 부족. S089 Luck 부트스트랩 목적 폐기 후 본 메커니즘 명분도 약함.

**옛 동작 (참조용)**: 캐릭터 첫 추천 탭 진입 시 최근 N회(옛 `BACKFILL_RECENT_COUNT = 30`, S093 cleanup 상수 제거)에 대해 결정론적 추천을 history에 backfill + 매칭.

**폐기 후 흐름**: 신규 캐릭터 = 빈 전적. 사용자가 추천 탭 saved-sets에서 "확정" 클릭 시 history 등록 (5.2.5.9 참조). 매주 발표 후 자동 매칭으로 진짜 적중률 누적.

**옛 데이터 마이그레이션**: `storage.js` `loadCharacters`에서 옛 백캐스트 추정 항목(`history[].createdAt === character.createdAt`) 자동 제거. 잔존 항목은 `source: 'user'`로 보수적 자동 채움.
