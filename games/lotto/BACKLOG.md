# Lotto 다음 버전 업그레이드 백로그

> 2026-06-07 전수 검사 결과 모음. docs/(현재 사양 SSOT) / game-hub ROADMAP(자비스 도메인 마일스톤)과 분리된 게임 자체 업그레이드 후보 목록.

### 2.2. UX-A - "확정" 멘탈 모델 재설계 (UX 디자이너 검사 1순위)

추천 리스트의 체크박스가 "기록 등록"을 의미하는데 화면만으로 안 전달. 사용자는 "체크=항목선택=일괄삭제용"으로 오해. 추천 리스트(임시) vs 기록(공식) 이중 저장 구조가 멘탈 모델로 안 잡힘. 디자인/여정 두 검사 공통 최상위.

- 체크박스 의미 라벨/aria 강화 + hover 피드백
- 확정 직후 토스트 + 기록 탭 연결
- 이중 저장 구조 자체 재정리 검토

### 2.3. UX-B - 추천 탭 정보 계층화

한 화면 9개 섹션 과부하. 라이트 사용자는 프리셋만으로 충분. 캐릭터 카드 기본 접기 / 라이트 모드 검토.

### 2.4. UX-C / 기능 발견성

### 2.10. PD - 제품 전략 관점 (고칠 목록이 아닌 전략 질문)

앞선 3검사와 층위가 다름. 이건 코드/디자인 결함이 아니라 "제품이 답해야 할 질문". 가장 상위라, 이게 먼저 풀려야 아래 우선순위도 방향이 선다.

핵심 진단: 이 제품의 최대 리스크는 버그가 아니라 **정직함이 매력을 죽이는 구조적 긴장**이다. 사행성을 빼면서 로또 앱의 본질 동기("당첨 기대")를 제거했는데, 그 빈자리를 채울 "결과와 무관한 재미"가 아직 약하다.

| ID | 전략 질문 | 본질 | 우선순위 |
|---|---|---|---|
| PD-A | 컨셉-동기 간극 | 캐릭터/운세/사주/리추얼이 전부 "추첨 결과 영향 0"으로 정직하게 분리됨 → 윤리적으로 완벽하나 "그래서 왜 해?"가 남음. 결과 무관 서사를 그 자체로 재미있게 만들 설계가 필요 | 최상 |
| PD-B | 데일리 리텐션 공백 | 로또는 주 1회 사이클. 토요일 발표~다음 추천까지 주중 7일 돌아올 이유가 약함. reveal 게임화는 1회성. 데일리 훅 부재 | 최상 |
| PD-C | 타겟 분열 | 라이트(프리셋 1탭)와 헤비(11전략/휠링/사주오행/4원소)를 동시 추구. 프리셋으로 표면은 단순화했으나 그 아래 깊이를 누가 소비하나 흐릿 | 상 |
| PD-D | 수익화 천장 | BM=캐릭터 슬롯/스킨/이력(코스메틱), 적중률 과금 금지. 캐릭터가 결과 영향 0이면 "애착 기반 과금"만 가능 → 애착(서사/성장)이 약하면 BM 성립 곤란. PD-A와 직결 | 상 |

| PD-F | 장기 콘텐츠 훅 | Luck 성장 폐기(S089)로 캐릭터 성장축 사라짐. 진행=전적 누적만. 시즌/컬렉션/성장 등 확장 여력 빈약 | 중 |

차별성(강점): 캐릭터 시드 + 사주/점성술은 한국 정서에서 통하는 명확한 moat. 시중 번호생성기와 다름. 단 "결과 무관"으로 계속 희석하면 이 강점도 약해짐 - PD-A에서 균형점 찾기.

## 3. 보존 (갈아엎지 말 것)

디자인 시스템 자체는 우수 (UX 디자이너 평가 A-): 디자인 토큰 체계 견고, 번호공 5색 WCAG AA 이상 대비, safe-area 노치 대응 완벽, 상태 시각화(active/hover/disabled/focus) 일관, 사행성 회피 카피 윤리적으로 잘 잡힘. 신규 디자인이 아닌 정제 방향.

## 4. 다음 버전 설계 (사용자 결정 2026-06-07) - "메인 깔끔화"

방향: 메인은 극단적으로 비우고, 중복 제거, 회차마다 바뀌는 운세를 전면에, 당첨 기원은 회차 맥락이 있는 기록 탭으로. UX-B + PD-A/B를 화면 지시로 구현.

### 4.1. 추천 탭 (메인) - 최대 비우기

- 헤더 "Blessed Lotto" 제목 제거 → 설정 탭에 표시.
- 전략 탭(11종) + 프리셋 3슬롯 선택 UI를 설정 탭으로 이동.
- 메인엔: 현재 활성 전략 묶음 이름만 한 줄 작게 표시 + `+1세트`/`+5세트` 버튼 (사용자 결정: 전략 이름 작게 표시).
- 당첨 기원 위젯 제거 → 기록 탭으로 (4.4).

### 4.2. 카운트다운

- 일/시/분/초 단위 라벨을 원 밖 → 원형 숫자 안 하단으로. 원 크기/숫자 비율 조정 (모바일 50px 비좁음 주의).

### 4.3. 캐릭터 카드 (중복 제거 + 운세 전면)

- 접힘: `이름 + 이번 회차 운세`. 옛 별자리·띠·일주 카피("황소.말.을해") 제거, 운세로 대체. 펼치지 않아도 운세 노출.
- 펼침: 이름 제거(접힘과 중복). 별자리/띠/일주 + 행운번호 3종 탭은 그대로 노출 (사용자 결정: 펼치면 다시 노출).
- 원칙: 중복 최대 제거.

### 4.4. 기록 탭

- 당첨 기원을 현재 회차 발표 대기 섹션(5.8.1-A) 아래 배치. 회차 맥락으로 기원 유도.

### 4.5. 영향 spec 절 (docs SSOT 선행 수정 대상)

01_spec 4장(화면흐름) / 5.1.3(전략 위치) / 5.1.5(프리셋 위치) / 5.1.6(캐릭터 카드 접힘) / 5.2.2(카운트다운) / 5.6(당첨 기원 위치) / 5.8(전적 페이지) + 02_data 카피 상수.
