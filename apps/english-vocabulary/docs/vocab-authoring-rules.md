# 단어 데이터 작성 규칙 (뜻·품사·예문·파생형) - 확정본 대기

이 문서는 단어 카드 내용(뜻·품사·예문·파생형)의 **출처와 작성 규칙**을 고정한다. 단어 자체의 선정 근거·전체 계획은 `vocab-master-plan.md`, 원천은 `sources/`를 따른다. 이 규칙이 확정되기 전에는 세트 내용 제작을 시작하지 않는다(사용자 지시 2026-07-24).

적용 대상: 우선 초등(공식 2022 `*` 대표형 800). 이후 중·고에도 동일 적용.

## 1. 카드 단위 (확정)

1.1. **대표형 1개 = 학습 카드 1개.** 공식 2022 개정 별책14 `*`(초등 사용권장) 대표형 800개를 카드로 쓴다.
1.2. 파생형·이형은 별도 카드가 아니라 **그 카드의 보조 정보**로만 넣는다.
   - 파생형(derivatives): 공식 목록이 대표형 옆 `( )`에 표기한 것만(예: act → actual, interact). 임의 추가 금지.
   - 이형(variants): 슬래시로 묶인 동의 이형(예: yes → yeah, yep). 대표형은 첫 형태.
1.3. 보조 정보는 정답 공개(ANSWER) 카드에서 뜻 아래 작게 표시한다(학습 판정 대상은 대표형 뜻).

## 2. 데이터 스키마 (신규 필드 포함)

```
{
  "id": "...", "setId": "...", "level": "elementary",
  "word": "act",              // 공식 대표형
  "pos": "동사",               // 대표 품사 1개
  "meaningKr": ["행동하다"],    // 핵심 뜻 1~2개
  "example": "...", "exampleKr": "...",
  "derivatives": ["actual", "interact"],  // 공식 괄호 파생형(있을 때만)
  "variants": ["yeah", "yep"]             // 슬래시 이형(있을 때만)
}
```
- `derivatives`/`variants`는 없으면 생략 또는 빈 배열.

## 3. 뜻(meaningKr) 출처·규칙

3.1. **출처 기준(고정 대상 - 아래 §확정 필요)**: 공식 목록엔 뜻이 없다. 뜻은 표준 학습용 대표 의미로 작성하고, 공개·검증 가능한 사전(예: 위키낱말사전 등 자유 라이선스, 표준 학습사전 통용 의미)과 대조해 확정한다. 특정 사전을 지정하면 그 기준을 단일 출처로 쓴다.
3.2. 학교급(초등)에 맞는 **가장 기본적인 의미** 우선. 고난도·전문·비유 의미를 대표 뜻으로 삼지 않는다.
3.3. 핵심 뜻 1~2개. 여러 뜻을 무검수로 나열하지 않는다. 품사별로 뜻이 갈리면 대표 품사의 뜻만.
3.4. 오탈자·긴 문법 설명·어원 금지. 짧고 명확하게.

## 4. 품사(pos) 규칙

4.1. 표준 사전 기준 **대표 품사 1개**. 허용 라벨: 명사/동사/형용사/부사/전치사/접속사/대명사/감탄사/한정사.
4.2. 원천에 품사가 없으므로 사전 대조로 채운다. 지어내지 않는다.

## 5. 예문(example/exampleKr) 규칙

5.1. 공식 목록엔 예문이 없다. **예문은 자작이 불가피**하며 사람 검수가 유일한 관문이다.
5.2. 목표 단어(또는 간단한 활용형)를 반드시 포함. 12단어 이하. 초등 수준의 쉽고 자연스러운 문장.
5.3. 대문자·마침표 정상. `exampleKr`은 자연스러운 한국어 해석.

## 6. 세트 구성

6.1. 대표형 800 → **200개씩 4세트**(초등 SET 1~4).
6.2. 세트 배분·세트 내 순서 기준(아래 §확정 필요): 기본은 **알파벳 순**(재현 가능·검증 용이). 빈도·활용도 정렬은 자료 확보 후 옵션으로 후행.
6.3. 초등은 단일 층. 중·고 층 구분은 이 규칙 밖(master-plan).

## 7. ID·데이터 버전·진도 (사용자 지시: 새 ID + 진도 초기화)

7.1. `manifest.json`과 각 세트에 `dataVersion`(예: `"moe2022-elem-1"`)을 둔다.
7.2. 진도 저장 키를 버전 네임스페이스로 분리한다: `deck:moe2022:<setId>`. 기존 `deck:<setId>`(playland 데이터 진도)는 읽지 않으므로 **초등 진도는 자동 초기화**된다(실사용자 없는 초기 단계라 영향 미미).
7.3. 세트/단어 id는 형식(`ev-set-NNN`/`ev-sNN-NNNN`)을 유지하되 공식 데이터로 새로 채번한다(구 데이터와 의미 혼동은 7.2 버전 분리로 차단).

## 8. 2회 내용 검수 절차 (각 세트 필수)

세트별로 아래 두 관문을 모두 통과해야 `available:true`로 전환한다. 검수 없이 앱 적용 금지, 800 일괄 자동 생성 금지.

8.1. **검수 1 - 사실·형식**: 단어가 공식 대표형과 일치하는가 / 뜻이 정확하고 학교급(초등)에 맞는가 / 품사가 맞는가 / 예문이 문법적으로 옳고 목표 단어를 포함하며 12단어 이하인가 / 파생형·이형이 공식 표기와 일치하는가 / `validate-data.mjs` 통과.
8.2. **검수 2 - 가독성·품질**: 뜻·예문이 자연스럽고 초등 학습자에게 쉬운가 / 뜻 중복·품사 편중이 심하지 않은가 / 한국어 표현이 매끄러운가 / 카드로 봤을 때 이해되는가(browser-shot 표본 확인).

## 9. 확정 필요 항목 (사용자 결정 대기)

- (§3.1) 뜻 출처를 "표준 의미 + 공개 사전 대조"로 할지, 특정 사전을 단일 기준으로 지정할지.
- (§6.2) 세트 내 순서를 알파벳 순으로 시작할지.
- (§7.3) 새 ID 채번 방식(형식 유지 + 버전 네임스페이스) 승인 여부.
