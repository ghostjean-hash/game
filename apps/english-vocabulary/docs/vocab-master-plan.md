# 영어 단어장 - 전체 단어 제작 마스터 플랜 (초중고 3000)

이 문서는 english-vocabulary 앱의 단어 데이터 전체 제작 계획이자 단일 진실 source다. 단어를 만들거나 세트를 늘릴 때는 반드시 이 문서의 규칙(2장)을 따른다. 진행 상태는 7장에서 갱신한다.

작성 근거: 사용자 지시(2026-07-24) "초중고 3000단어를 목표로 진행. 전체 단어 작업 계획 작성 및 기록, 앞으로 이 규칙으로 단어 생성."

## 1. 목표

1.1. 최종 목표: 초·중·고 핵심 영어 어휘 **3,000단어**를 사실 근거로 제작해 앱에 탑재.
1.2. 근거 기준: 교육부 영어과 교육과정 **기본어휘목록** 계열(초등 권장 어휘 + 중·고 기본어휘). 자의적 단어 생성 금지.
1.3. 사용자 경험은 기존과 동일 - "모름/알았음" 회상형, 세트별로 0개가 될 때까지 반복 암기. 데이터만 3,000으로 확장.

## 2. 단어 생성 규칙 (SSOT - 앞으로 모든 단어는 이 규칙으로만 만든다)

이 6개 규칙은 앞으로 예외 없이 적용한다. CLAUDE.md §4.7에도 요약 고정.

2.1. **R1 단어 선정 = 검증 가능 출처만.** 단어는 반드시 교육부 기본어휘목록 등 공개·검증 가능한 출처에 실재하는 것만 넣는다. 목록에 없는 단어를 감(感)으로 추가하지 않는다. 각 단어가 어느 출처에서 왔는지 추적 가능해야 한다(출처는 `docs/sources/`에 원본 보존).
2.2. **R2 뜻(meaningKr).** 출처 목록의 뜻을 1차로 쓰되, 표준 학습 사전(옥스퍼드/롱맨 초등 등)으로 대조·교정한다. 핵심 뜻 1~2개(요청서 6장 - 긴 문법·어원 금지). 출처의 오탈자는 반드시 교정.
2.3. **R3 품사(pos).** 표준 사전 기준 대표 품사 1개를 한국어 라벨로(허용: 명사/동사/형용사/부사/전치사/접속사/대명사/감탄사/한정사). 출처에 품사가 없으면 사전 대조로 채우되, 지어내지 않는다.
2.4. **R4 예문(example/exampleKr).** 목표 단어(또는 활용형)를 포함한 짧은 문장(14단어 이하) + 한국어 해석. **예문은 어떤 공식 어휘목록에도 없어 자작이 불가피하다.** 이 부분만은 근거가 "표준 문법·수준 적정성"이며, 반드시 사람 검수 대상으로 둔다(2.6의 검수에서 확인).
2.5. **R5 난이도 배치.** 출처의 층(초등/중학/고등)에 따라 세트에 배분한다(4장). 쉬운 층부터 낮은 세트 번호.
2.6. **R6 검증·검수 게이트.** 세트 완성 시 (a) `tools/validate-data.mjs`로 형식·중복·품사 자동 검증 통과, (b) 사람이 뜻·예문을 훑어 검수, 두 관문을 통과해야 그 세트를 `available=true`로 전환한다. 전체 완성 시 `--strict`(총 3000·세트당 200)로 최종 검증.

## 3. 데이터 출처

3.1. **확보됨(초급층)**: "교육부 지정 초등 필수 영단어 800" - 실제 PDF에서 800단어 + 한글 뜻 채집. `docs/sources/edu-elementary-800.txt`에 원본 보존. 2차 편집본이며 개정 연도 미확정·뜻 오타 일부(3.3 한계). 초급 4세트(약 800단어) 근거로 사용.
3.2. **확보 필요(중·고층, 약 2,200)**: 교육부 영어과 교육과정 기본어휘목록 3,000 중 초등 800을 뺀 나머지. 국가교육과정정보센터(NCIC) 원본 또는 교육부 목록을 명시 인용한 신뢰 2차 출처. Phase 0에서 별도 조사·확보.
3.3. **출처 한계(정직 고지)**: 확보된 800은 교육부/NCIC 원본을 직접 대조한 게 아닌 2차 편집본. 앱·문서에 "교육부 O년 개정 공식"이라 단정하지 않고 "교육부 지정 초등 필수 어휘 기반"으로 표기. 중·고층 확보 시 출처 등급을 다시 기록.

## 4. 세트 구조 (잠정 - 실제 출처 층에 맞춰 확정)

4.1. 총 3,000단어 = **15세트 × 200단어**.
4.2. 난이도 층 배분(잠정, Phase 0에서 실제 목록 층에 맞춰 조정):

| 세트 | 층 | 근거 출처 | 상태 |
|---|---|---|---|
| SET01~04 | 초급(초등) | 초등 800 (확보) | 제작 가능 |
| SET05~10 | 중급(중학) | 중학 기본어휘 (확보 필요) | 출처 대기 |
| SET11~15 | 고급(고등) | 고교 기본어휘 (확보 필요) | 출처 대기 |

4.3. 세트별 200단어는 층 내에서 알파벳/주제 등 재현 가능한 기준으로 나눈다(자의적 취사 최소화, 기준을 세트 메타에 기록).

## 5. 세트 1개 제작 표준 파이프라인

5.1. 출처 층에서 200단어 후보 선정(선정 기준 명시) → 2.1 준수 확인.
5.2. 각 단어에 뜻(R2)·품사(R3)·예문(R4) 부여. 빌더 스크립트로 id·setId·level 자동 부여(형식 오류 0).
5.3. `node tools/validate-data.mjs`로 자동 검증(오류 0).
5.4. 사람 검수(뜻·예문). 필요 시 수정.
5.5. `manifest.json`에서 그 세트 `count`=200·`available`=true 전환.
5.6. `tests/run-node.mjs` + browser-shot 실경로 재생 확인.
5.7. 커밋.

## 6. 실행 순서 (Phase)

6.1. **Phase 0 - 출처 확보**: 중·고층 기본어휘(약 2,200) 검증 가능 출처 조사·확보 후 `docs/sources/`에 보존. 완료 시 4장 층 배분 확정.
6.2. **Phase 1 - 초급 4세트(SET01~04, 800단어)**: 확보된 초등 800으로 5장 파이프라인 실행. SET01부터. (현재 SET01에 있는 임의 200단어는 규칙 위반이라 이 단계에서 출처 기반으로 교체.)
6.3. **Phase 2 - 중급 6세트(SET05~10)**: Phase 0 출처로 제작.
6.4. **Phase 3 - 고급 5세트(SET11~15)**: 동일.
6.5. **Phase 4 - 최종 검증**: 전 세트 `--strict` 통과, 난이도 흐름·중복(세트 간) 최종 점검.

각 Phase는 세트 단위로 쪼개 세션에 나눠 진행. 한 번에 3,000을 만들지 않는다.

## 7. 진행 추적 (여기서 갱신)

- 2026-07-24: 계획 수립. 초등 800 출처 확보(`docs/sources/edu-elementary-800.txt`). manifest 3,000/15세트 구조로 갱신.
- 현재 상태: **Phase 0 대기**(중·고 출처 확보 전). SET01의 임의 200단어는 규칙 위반 상태로, Phase 1에서 초등 800 기반으로 교체 예정.
- 다음 행동(사용자 결정 영역): (a) Phase 0 중·고 출처 조사 착수 (b) 또는 초등 출처가 이미 있으니 Phase 1(SET01 교체)부터 착수.

## 8. ID·스키마 규약 (기존 유지)

8.1. 단어 id `ev-sNN-NNNN`(NN=세트 order 01~15, 뒤 4자리 일련). 세트 id `ev-set-NNN`.
8.2. `word`: `{ id, setId, level, word, pos, meaningKr[], example, exampleKr }`.
8.3. **ID 안정성 주의**: 한번 배포된 단어 id는 다른 단어로 재사용하지 않는다(재사용 시 기존 학습자의 진행이 엉뚱한 단어에 매핑됨). 세트 교체 시 이 점을 고지하고, 필요하면 사용자 "데이터 초기화" 안내.

## 9. 미결정·리스크

9.1. 중·고 기본어휘 원본(3.2) 확보 성공 여부 - NCIC 원본이 로그인/뷰어라 2차 출처에 의존할 수 있음. 확보 실패 시 대체 출처 등급을 사용자와 합의.
9.2. 예문 자작(R4)의 품질 - 자동 검증이 의미 정확성을 못 잡음. 사람 검수가 유일한 관문.
9.3. 세트당 200 균등 분할 시 난이도 편차 - 층 내 세부 난이도까지는 출처가 안 주므로, 알파벳/주제 등 재현 기준으로만 나누고 완벽한 난이도 순은 보장하지 않음.
9.4. 3,000 목표는 교육부 기본어휘목록 규모(약 3,000)와 정합. 실제 확보 단어가 3,000에 못 미치면 목표를 실제 확보량에 맞춰 조정(사용자 확정).
