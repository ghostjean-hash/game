## 2026-07-24 · v0.3 회상형 학습 전환(정답 공개 전 뜻 회상 강제)

- **배경**: 사용자(ChatGPT 작성 요청서) 지적 - 기존 학습 화면이 단어·뜻·예문·판정 버튼을 처음부터 다 보여줘 "정보 열람 화면"이었다. 정답을 본 상태로 외움/모름을 눌러 실제 기억 여부를 확인 못 함. → 정답 공개 전 반드시 한 번 기억에서 뜻을 꺼내게 하는 회상형으로 전면 수정.
- **핵심 변경**: 학습 카드를 QUESTION(단어만) / ANSWER(뜻 공개) 두 상태로 분리.
  - QUESTION: 단어 + 발음 + "뜻 확인" 버튼만. 뜻·예문·판정 버튼은 DOM에 아예 렌더하지 않음(CSS 숨김 아님 → 스크린리더·시각 모두 미노출).
  - ANSWER: 뜻·예문 공개 + 판정 버튼. 이때만 판정 가능.
  - 단어가 바뀔 때마다 QUESTION으로 초기화. 새로고침 복원 시에도 정답 공개 상태는 이어지지 않음(cardView 기본값 question, 미저장 - 편법 방지).
- **버튼·문구**: 모름/외움 → 몰랐음/알았음. 학습·보관함 복습 동일. deck.js의 판정 의미(known=learned·learnedAt, unknown=active 유지+unknownCount)는 요청서와 이미 일치해 로직 무변경(UI만 전환).
- **키보드(상태별)**: QUESTION은 Space/Enter=뜻 확인(판정 키 1·2·←·→ 무시로 오판정 차단). ANSWER는 ←/1=몰랐음, →/2=알았음(Space/Enter는 자동 진행 안 함). 하단 안내도 상태별로 동적 교체. 발음은 버튼 클릭 전담(Space를 뜻 확인에 넘겨 발음 단축키는 제거 - 요청서 10장 대비 유일 의도적 편차, 아래 고지).
- **보관함 복습**: 동일 회상 흐름(단어→뜻 확인→몰랐음/알았음). 몰랐음=active 복귀, 알았음=learned 유지.
- **Undo**: 판정 후에만 생성(기존과 동일). 되돌리면 그 단어를 ANSWER 상태로 복원해 다시 판정 가능. 세트 완료 화면 Undo도 동일.
- **최초 안내**: 첫 학습 진입 시 1회 안내 모달("뜻 먼저 떠올린 뒤 뜻 확인") + [시작하기], settings.introDone 저장으로 재노출 안 함.
- **구조**: 순수 로직 `src/core/viewstate.js` 신설(VIEW·initialCardView·resolveKey) - 카드 상태·키 매핑을 DOM에서 분리해 node 테스트 가능. main.js는 이 모듈 + 공용 헬퍼(buildWordRow/appendAnswerBody)로 학습·복습 렌더 통합.
- **검증**: `run-node.mjs` 83 PASS / 0 FAIL(기존 57 유지 + viewstate·판정 의미 26 추가). browser-shot 실경로 3종 - (1)학습 QUESTION(뜻 DOM 부재 measure.found=false) (2)학습 ANSWER(뜻·예문·판정 노출) (3)보관함 복습 QUESTION(회상형). 모두 콘솔 에러 0, 최초 안내 모달 통과 확인.
- **저장 마이그레이션**: 불필요. deck 직렬화 스키마 무변경, settings에 introDone(기본 false)만 추가 - 기존 사용자는 안내 1회 노출 후 정상. cardView는 런타임 전용(미저장).
- **의도적 편차(고지)**: 요청서 10장 "발음 단축키 기존 유지" 대비, Space가 QUESTION의 뜻 확인으로 재배정돼 발음은 버튼 클릭 전담으로 변경. 요청서 하단 안내 예시(🔊: 발음 = 버튼)와 합치.
