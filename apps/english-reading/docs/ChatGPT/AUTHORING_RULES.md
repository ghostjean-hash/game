# 영어 독해 문제 작성·검수 규칙

## 1. 지문

- 지문 1편은 정확히 5문장이다.
- 다섯 문장은 하나의 주제와 자연스러운 흐름을 가져야 한다.
- 각 문장은 개별 학습도 가능해야 한다.
- 제목과 내용이 일치해야 한다.
- 첫 문장은 주제를 시작하고 마지막 문장은 자연스럽게 정리한다.
- 관련 없는 예문 다섯 개를 모은 형태는 허용하지 않는다.

## 2. 난이도

단어 수는 대표 범위이며 강제 기준이 아니다.
절 수, 수식 구조, 추상성, 생략, 관계절, 비교 구조를 함께 판단한다.

### Level 1
- 대체로 7~11단어
- 일상, 감정, 습관, 기본 여행
- 단순 현재·과거, 기본 조동사
- 문장당 핵심 구조 1개
- 긴 관계절과 복합 비교절은 피한다.

### Level 2
- 대체로 8~14단어
- that절, to부정사, 동명사, 간단한 비교
- 짧은 관계절과 간접의문문 허용
- 복잡한 구조를 여러 개 중첩하지 않는다.

### Level 3
- 대체로 9~18단어
- 관계절, 비교절, 수동태, 분사 수식, 간접의문문
- 쉬운 영어 설명 영상 수준
- 과도한 절 중첩과 전문용어는 피한다.

단어 수가 대표 범위를 벗어났다는 이유만으로 바로 오류로 판정하지 않는다.
실제 난이도는 구조·어휘·추상성을 기준으로 판단한다.

## 3. 영어 원문

- 문법적으로 정확하고 실제 영어에서 자연스러워야 한다.
- 한국어 직역투를 피한다.
- 대명사의 대상이 문맥에서 분명해야 한다.
- 시제와 화자 관점이 자연스럽게 이어져야 한다.
- 문법 목표를 넣기 위해 문장을 부자연스럽게 만들지 않는다.

## 4. 청킹 `chunks`

- `chunks[].en`을 공백으로 합치면 `text`와 같아야 한다.
- 원문의 단어를 누락·추가·변형하지 않는다.
- 영어 어순대로 의미를 누적해 이해하기 좋은 단위로 나눈다.
- 짧은 문장은 한 청크여도 된다.
- 모든 문장을 억지로 여러 청크로 만들지 않는다.

원칙적으로 다음 내부는 끊지 않는다.

- 관사·형용사 + 명사
- 전치사 + 목적어
- 조동사 + 본동사
- 진행형·완료형
- to + 동사원형
- 동사 + 필수 전치사
- 짧은 주어 + 동사
- be동사 + 짧은 보어
- 하나의 비교 의미 단위
- help/make/ask/tell + 목적어 + 동사

추천 경계:

- 긴 문두 시간·조건 표현 뒤
- 긴 주어 뒤
- 주절과 종속절 경계
- 접속사 앞
- 긴 관계절 앞이나 뒤
- 새로운 정보가 시작되는 지점

`that`, `who`, `which` 앞을 무조건 분리하지 않는다.

## 5. 직독직해 `chunks[].kr`

- 해당 영어 청크의 의미를 영어 순서대로 보여준다.
- 뒤 청크의 의미를 앞 청크에 미리 넣지 않는다.
- 원문에 없는 정보나 감정을 추가하지 않는다.
- 청크 간 의미를 중복하지 않는다.
- 이해하기 어려운 기계 번역투를 피한다.

## 6. 자연 번역 `naturalTranslation`

- 모든 문장에 필수다.
- 자연스럽고 완전한 한국어 문장이어야 한다.
- 직독직해 조각을 기계적으로 이어 붙이지 않는다.
- 원문의 강도, 원인, 결과, 감정을 바꾸지 않는다.

## 7. `breakRules`

boundary는 0-based 단어 사이 인덱스다.

예:
`["We", "believe", "that"]`

- boundary 0 = We | believe
- boundary 1 = believe | that

### allowed
- 대표 chunks 경계는 아니지만 자연스러운 대안 분할
- 모든 문장에 억지로 넣지 않는다.

### discouraged
- 강하게 결합된 구조를 갈라 이해를 방해하는 위치
- 모든 gap을 채우지 않는다.
- 구체적인 `reason`을 쓴다.

금지:

- allowed와 discouraged의 동일 boundary 중복
- 같은 배열 안 boundary 중복
- 대표 청킹 경계와 discouraged 중복
- 문장 범위를 벗어난 boundary
- 이유 없는 항목

## 8. `wordOrderPoint`

- 문장마다 정확히 1개
- 가장 먼저 이해해야 할 핵심 어순 또는 패턴
- `title`은 짧은 구조명
- `explanation`은 실제 원문 표현과 연결한다.
- 문법 용어만 나열하지 않는다.

## 9. `grammar`

- 문장마다 최소 1개
- 핵심적인 문법·구조만 설명한다.
- `label`은 문법 또는 구조 이름
- `note`는 실제 원문 표현과 연결된 한 줄 설명
- 잘못된 문법 설명은 반드시 수정한다.

## 10. `words`

- 어려울 가능성이 있는 단어만 선택한다.
- 쉬운 단어를 개수 채우기 위해 넣지 않는다.
- 원문에 등장한 표면형을 사용한다.
- 활용형을 임의로 기본형으로 바꾸지 않는다.
- 뜻은 해당 문맥에 맞춘다.

## 11. `insight`

- 선택 필드다.
- 한 지문에서 최대 3문장까지만 사용한다.
- Level 3 또는 구조적으로 어려운 문장에서만 사용한다.
- 일반 grammar 설명을 반복하지 않는다.
- 문장 이해에 실질적으로 도움이 되는 심화 설명만 넣는다.
- 필요하지 않으면 필드를 만들지 않는다.

## 12. 공식 topic

- Critical Thinking
- Daily Life
- Games
- Health
- Language
- Psychology
- Relationships
- Science
- Self Development
- Study
- Technology
- Travel

새 topic이 필요하면 앱 규약과 스키마를 함께 갱신한다.

## 13. 중복

자동 검증 대상:

- 동일 ID
- 동일 또는 앱 정규화 기준상 동일한 제목
- 완전히 같은 문장

출제·검수 단계에서 추가로 경계할 항목:

- 거의 같은 소재
- 비슷한 전개
- 표현의 과도한 반복
- 의미상 유사한 문장

의미 유사도는 자동 검증이 아니라 검수 참고 항목으로 본다.

## 14. 오류 판정

### 형식 오류
- 필수 필드 누락
- chunks 결합 불일치
- boundary 범위·중복 오류
- 허용되지 않는 level 또는 topic
- 스키마 불일치

처리:
자비스 검증 결과에 따라 수정 후 재검증한다.

### 내용 오류
- 영어 문법 오류
- 의미가 달라진 번역
- 잘못된 문법 설명
- 부적절한 청킹
- 레벨의 큰 불일치
- 지문 흐름 불량

처리:
수정 후 전체 문장을 다시 검수한다.

### 개선 권장
- 제목이 다소 평범함
- 설명이 장황함
- 단어 선정이 약간 쉬움
- 번역이 조금 어색하지만 의미는 정확함

## 15. 출제 전 자체 점검

1. 정확히 5문장인가
2. 영어가 자연스러운가
3. 레벨에 맞는가
4. 공식 topic인가
5. chunks가 원문과 일치하는가
6. 강하게 결합된 구조를 잘못 나누지 않았는가
7. 직독직해가 영어 순서를 유지하는가
8. 자연 번역이 정확한가
9. boundary가 논리적으로 맞는가
10. wordOrderPoint와 grammar가 정확한가
11. words가 원문 표면형인가
12. insight가 정말 필요한 문장에만 사용되었는가
13. 기존 콘텐츠와 명백하게 중복되지 않는가
14. JSON 구조가 스키마와 일치하는가
