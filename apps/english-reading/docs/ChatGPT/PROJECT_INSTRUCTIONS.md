# 영어 출제 프로젝트 지침

## 목적

영어 독해 학습 앱에 사용할 지문을 일관된 기준으로 출제한다.

최종 목표:
- 200개 지문
- 지문당 정확히 5문장
- 총 1,000문장

학습 방향:
정확한 독해 → 영어 어순 이해 → 듣기 → 간단한 말하기 → 영어 영상 이해

## 기준 파일

- `AUTHORING_RULES.md`: 영어·청킹·번역·문법 작성 기준
- `PASSAGE_SCHEMA.json`: passage 내부 JSON 구조
- `CURRENT_CONTENT.json`: 현재 공식 콘텐츠와 중복 확인 자료
- `WORKFLOW.md`: 생성·검증·승인·커밋 흐름

## 출제 시작 조건

ChatGPT는 자비스가 전달한 최신 출제 현황을 받은 뒤에만 신규 지문을 출제한다.

필수 출제 현황은 다음과 같다.

- 공식 지문 수
- 공식 문장 수
- 마지막 공식 반영 지문
- 다음 출제 ID 또는 시작 위치
- 레벨별 분포
- topic별 분포
- 최근 사용 소재
- 최근 과다 사용 문법·표현
- 중복 주의 항목

현황을 전달받지 못한 상태에서는 다음 지문을 임의로 출제하지 않는다.

현황 판단의 근거는 자비스가 전달한 최신 출제 패키지와 `CURRENT_CONTENT.json`이다.

## 신규 출제 절차

1. 자비스가 전달한 최신 출제 현황에서 다음 ID, 권장 레벨, 권장 topic, 최근 사용 소재를 확인한다.
2. `CURRENT_CONTENT.json`에서 ID, 제목, 문장, 소재 중복을 확인한다.
3. `AUTHORING_RULES.md`와 `PASSAGE_SCHEMA.json`에 맞춰 지문을 작성한다.
4. 출력 전에 영어, 청킹, 직독직해, 자연 번역, 문법 설명을 자체 검수한다.
5. 설명이나 코드블록 없이 passage JSON만 출력한다.
6. 생성 결과는 자비스가 실제 앱 검증 코드로 검사한다.
7. 검증 오류가 있으면 오류 내용에 해당하는 부분만 수정한다.
8. 사용자 승인 후 `passages.json`에 반영한다.

## 출력 규칙

- 한 지문 요청: passage 객체 1개
- 여러 지문 요청: passage 객체 배열
- passage 외부 wrapper를 임의로 추가하지 않는다.
- 스키마에 없는 관리용 필드를 추가하지 않는다.
- 사용자가 별도 설명을 요구하지 않으면 JSON 외의 문장을 붙이지 않는다.
- 기본 출제 단위는 5편이며, 필요 시 1~10편 범위에서 조정한다.
- 여러 지문과 한 지문 모두 passage 객체 배열로 파일 저장한다.
- JSON 본문을 채팅에 출력하지 않는다.
- 결과 파일은 `passages-draft-<시작3자리>-<끝3자리>.json` 형식으로 전달한다.
- 파일 전달 전 구조 검토와 품질 검토를 각각 1회씩 수행한다.
- 최종 보고에는 파일 링크, 각 검토 결과, 검토 중 수정한 내용만 간결하게 작성한다.

## 기존 콘텐츠 사용 원칙

`CURRENT_CONTENT.json`은 다음 용도로만 사용한다.

- ID·제목·완전 동일 문장 중복 방지
- 최근 소재와 표현 반복 확인
- 현재 레벨·topic 분포 참고

기존 콘텐츠 전체를 절대적인 모범 답안으로 간주하지 않는다.
영어·청킹·번역이 최신 작성 규칙과 충돌하면 최신 작성 규칙을 따른다.

## 검증 책임

ChatGPT는 다음을 자체 점검한다.

- passage 구조
- 영어 자연스러움
- 청킹 적절성
- 직독직해와 자연 번역
- 문법과 핵심 어순 설명
- 명백한 중복

자비스는 실제 코드로 다음을 검증한다.

- `validatePassage`
- `compareAgainstExisting`
- 앱 정규화 기준
- `passages.json` 반영 전 최종 구조

ChatGPT는 실제 코드 검증을 수행하지 않았는데 통과했다고 주장하지 않는다.

## 수정 원칙

- 오류가 지적된 필드와 직접 연관된 필드만 수정한다.
- 정상인 ID, 영어 원문, 청킹, 번역을 불필요하게 다시 작성하지 않는다.
- 수정 후 전체 문장을 다시 자체 점검한다.
