영어 독해 사다리 기획서

공부를 정말 못하는 사람이 영어 지문 한 편을 스스로 읽어내게 하는 독해 훈련 앱
버전 v0.1 초안  ·  작성 2026-07-07  ·  상태 방향 확정, 세부 규격 초안  ·  앱 game-hub 허브의 첫 학습 앱

1개요

1.1 정의

공부를 정말 못하는 사람이 영어 지문 한 편을 스스로 읽어내는 경험을 매번 완결하게 해 주는 독해 훈련 앱.

1.2 핵심 속성

  • 막히지 않음 - 모르는 단어는 언제든 도움받고, 틀려도 벌 없이 넘어감.
  • 매 문장 성취 - 한 문장을 스스로 읽어낸 감각을 매번 줌. 중학 수준 학습자가 가장 못 받아본 경험임.
  • 성장이 보임 - 어제 막힌 단어를 오늘 맞히면 앱이 기억해 보여줌.

1.3 가치 제안

  • 학습자 - 평생 못 받아본 "나도 읽어낸다"는 성공 경험. 좌절 없이 매일 한 편씩.
  • 운영자 - 이탈이 적은 꾸준한 학습 습관과, 눈에 보이는 성장 지표.

2타겟과 전략

2.1 타겟

영어 독해가 중학생 수준이고, 단어도 긴 문장도 글 전체 요지도 세 군데 다 벅찬 학습자. 학습 실패가 쌓여 "나는 원래 못해"라는 좌절이 깔린 사람.

2.2 문제 진단

이 사람들이 그만두는 진짜 이유는 머리가 아니라 막히는 순간의 좌절임. 모르는 단어 하나, 긴 문장 하나에서 멈추면 그날로 앱을 닫음. 그런데 학교와 기존 앱은 독해력의 맨 위층인 요지 찾기와 문제 풀이부터 시켜서 계속 실패시킴.

2.3 독해력의 세 층

독해력은 세 층으로 쌓인 건물임. 맨 아래가 단어, 중간이 문장을 끊어 읽는 힘, 맨 위가 글 전체 의미 잡기. 아래 두 층이 무너진 사람에게 맨 위층을 시키면 무너짐. 그래서 아래층부터 밟게 하는 것이 이 앱의 뼈대임.

2.4 핵심 전략 - 단어를 뒤로

단어 뜻을 앞에서 떠먹이지 않음. 먼저 문맥으로 뜻을 추측하게 한 뒤 나중에 확인시킴. 이는 취향이 아니라 어휘 습득 연구에 기댄 결정임.

  • 문맥으로 뜻을 추측한 학습자가 다른 방법보다 더 오래 기억함. 중학생 대상 실험으로 확인된 결과임.
  • 스스로 추측하는 예측 행위 자체가 기억을 강화함. 답을 미리 보면 그냥 흘려보내게 됨.
  • 다만 추측만 하고 끝내면 효과가 약함. 추측 뒤에 실제 뜻을 확인하고 나중에 다시 떠올리는 과정이 붙어야 진짜 실력으로 남음. 그래서 확인과 복습을 학습 흐름에 반드시 포함함.
  • 규칙 암기보다 이해되는 글을 많이 읽어 문맥으로 익히는 쪽이 효과적임. 이 방식으로 성공한 대표 학습 서비스로 Readlang과 LingQ가 있음.

3핵심 기능

지문 한 편을 세 단계로 통과함. 각 단계가 작아서 무너지지 않음. 중학 수준의 짧은 글 한 편이 한 번의 학습 단위임.

3.1 읽으며 추측

문장을 의미 단위로 끊어 읽으며 직독직해 감각을 잡음. 지문에는 이 학습자가 모를 법한 단어 몇 개가 표시되지만 뜻은 주지 않음. 사용자는 문맥으로 그 뜻을 추측함. 정말 막히면 힌트로 품사나 문맥의 방향만 받고, 완전한 뜻은 아직 받지 않음.

3.2 무슨 얘기였나

지문 전체를 읽은 뒤 "이 글은 무슨 내용인가"를 쉬운 보기로 확인함. 정답과 그럴듯한 오답 두세 개 중 고름. 틀려도 벌 없이 정답을 알려 준 뒤 넘어감.

3.3 단어 확인과 내 단어장

이제서야 앞에서 추측한 단어들의 실제 뜻을 공개함. 내 추측이 맞았는지 대조함. 애매했거나 힌트를 쓴 단어는 내 단어장에 자동으로 담기고, 다음 학습에서 복습으로 다시 나옴. 추측하고 확인하고 다시 떠올리는 세 박자가 여기서 완성됨.

단계하는 일채우는 층무너지지 않는 이유
① 읽으며 추측끊어 읽으며 표시된 단어의 뜻을 문맥으로 추측단어와 문장긴 문장을 덩어리로 쪼갬, 힌트 상시
② 무슨 얘기글 요지를 쉬운 보기로 확인글 전체 의미오답에 벌점 없음, 정답 안내 후 진행
③ 단어 확인추측 단어의 실제 뜻 공개와 대조, 막힌 단어 수집단어 정착맞혔다는 성취 + 복습으로 재등장

4사용 흐름

  • 진입 - 오늘 읽을 지문 한 편을 바로 제시함. 복습할 단어가 있으면 먼저 복습 카드 몇 장을 끼워 줌. 고를 것 없이 시작 한 번으로 학습에 들어감.
  • 지문 읽기 - 문장을 끊어 읽으며 표시된 단어의 뜻을 추측하고, 막히면 힌트를 받음.
  • 요지 확인 - 지문 전문과 요지 질문을 보고 보기를 고름. 고르면 정답과 한 줄 해설을 보여줌.
  • 단어 확인 - 추측한 단어의 실제 뜻을 공개하고, 맞힘과 새로 알게 됨을 구분해 성취를 보여줌. 막힌 단어는 단어장에 담겼다고 안내함.
  • 내 단어장과 복습 - 담긴 단어를 모아 두고, 문맥 문장에서 뜻을 떠올려 보는 복습을 제공함.
  • 공통 - 막히지 않기 - 어느 화면에서든 오답에 벌점이 없고, 정답을 강요하지 않고 다음으로 넘어갈 수 있으며, 도움이 늘 열려 있고, 진행은 자동으로 저장됨.

5상태와 성장

5.1 내 단어장

학습 중 막힌 단어를 자동으로 모음. 단어와 그 단어가 나온 문맥 문장을 함께 보관해, 나중에 그 문장 속에서 다시 떠올리게 함.

5.2 복습

담긴 단어를 나중 학습에서 다시 만남. 맞히면 다음에 만나는 간격이 길어지고, 틀리면 짧아짐. 자주 틀리는 단어를 더 자주 보게 만드는 방식임.

5.3 진행 지표

읽어낸 글 수, 연속 학습일, 정착한 단어 수를 보여줌. 같은 글을 다시 읽을 때는 지난번보다 막힌 단어가 몇 개 줄었는지 짚어 줌. 점수나 등급이 아니라 "어제보다 늘었다"를 체감하게 하는 것이 목적임.

6약점 진단과 리포트

학습 데이터를 모아 어디가 약한지 짚어 주는 층. 단어만 모으면 무슨 단어를 모르는지까지만 알지만, 문법 이름표를 붙이면 어떤 문장 구조에서 무너지는지가 드러남. 독해가 안 되는 사람은 단어보다 문장 구조에서 막히는 경우가 더 많음. 이 층이 그냥 문제 푸는 앱과 제대로 도움되는 앱을 가름.

6.1 무엇을 모으나

  • 자주 틀리는 단어 - 내 단어장에 다시 담긴 횟수로 순위를 냄.
  • 약한 문법 - 어떤 문법이 든 문장이나 문항에서 자주 막히는지 이름표별로 집계함.
  • 추이 - 지난주와 견줘 나아졌는지.

6.2 문법 이름표

독해에서 문장 구조를 못 잡게 만드는 문법에 이름표를 붙임. 유추 단어와 요지 문항마다 이름표를 달아 두고, 막히거나 힌트를 쓸 때마다 그 이름표에 기록이 쌓임. 처음 여섯 가지로 시작하고 콘텐츠가 늘면 넓힘.

문법 이름표왜 독해에 중요한가
주어·동사 찾기문장의 뼈대. 이게 안 되면 아무리 읽어도 무슨 동작인지 모름

시제·수동태·비교의미를 뒤집는 지점. 당했다인지 할 것인지를 놓치면 오독

6.3 리포트로 돌려주기

모은 데이터를 이번 주 약한 문법 세 가지와 자주 틀리는 단어로 보여줌. 한 발 더 나가면 약한 문법이 든 지문을 더 자주 내주는 되먹임도 가능함.

이 진단은 화면 뒤에서 조용히 쌓는 것이라 문법을 강의하지 않음. 문법 강의를 만들지 않는다는 원칙은 그대로 두고, 어디서 막히는지 짚어 주기만 함.

7콘텐츠 규격

  • 지문 - 중학 수준의 짧은 글. 흥미를 끄는 주제로 고름.
  • 난이도 - 완만한 계단으로 올라감. 문장 길이와 추측 단어의 밀도로 가르고, 단어 수만으로 가르지 않음.
  • 추측 단어 선정 - 문맥에 뜻을 짐작할 단서가 실제로 있는 단어만 대상으로 삼음. 단서 없이 못 맞힐 단어는 대상에서 빼서 좌절을 막음.
  • 요지 문항 - 오답은 글을 안 읽으면 그럴듯하지만 읽으면 아닌 수준으로 만듦. 함정이 아니라 이해를 확인하는 용도임.
  • 문법 이름표 - 유추 단어와 요지 문항마다 6.2의 이름표를 달아 약점 집계의 근거로 삼음. 콘텐츠를 만들 때 함께 붙임.
  • 제작 주체 - 초기 지문은 자비스가 만들고, 사용자가 뒤이어 보강함.

8만들 것과 안 만들 것

8.1 첫 버전에 넣는 것

  • 중학 수준 지문 열에서 열다섯 편.
  • 추측, 확인, 복습으로 이어지는 세 단계 학습 전체.
  • 문장 끊어 읽기.
  • 내 단어장 자동 수집과 복습.
  • 자주 틀리는 단어와 약한 문법을 짚어 주는 기본 리포트.
  • 읽은 글 수, 연속 학습일, 정착 단어 수 지표.

8.2 나중으로 미루는 것

  • 약한 문법이 든 지문을 더 자주 내주는 되먹임.
  • 같은 글 재도전 시 지난번과의 향상 비교.
  • 사용자 수준에 맞춘 지문 자동 추천.
  • 음성으로 읽어 주기.
  • 주제와 수준을 직접 고르는 화면.

8.3 만들지 않는 것

  • 타이핑이나 서술형 답. 처음부터 끝까지 눌러서 진행함.
  • 서버, 계정, 결제.
  • 어두운 화면 전환. 밝은 학습 화면만 둠.
  • 문법 규칙 강의. 문맥으로 익히는 것이 원칙임.

9개발 참고

아래는 목업과 구현을 위한 최소 정책 메모임. 기획 본문이 아니라 제작 시 참고용임.

  • 지문 한 편은 여러 문장으로 이루어지고, 각 문장에 끊어 읽기 단위와 추측 대상 단어를 둠. 추측 단어의 실제 뜻과 힌트는 따로 보관했다가 세 번째 단계에서 공개함.
  • 유추 단어와 요지 문항에 문법 이름표를 붙이고, 막힘과 힌트 사용, 오답을 이름표별로 집계해 저장함. 이 집계가 약점 리포트의 재료임.
  • 지금 있는 끊어 읽기 처리를 그대로 재활용하고, 그 위에 요지 확인과 단어 확인, 복습을 얹음.
  • 단어장과 진행 기록, 복습 일정, 문법별 집계는 서버 없이 기기 안에 저장함.
  • 검증은 실제 학습 경로를 처음부터 끝까지 밟아 확인함. 화면만 보고 됐다고 판단하지 않음.
