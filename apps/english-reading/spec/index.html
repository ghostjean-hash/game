영어 하이브리드 독해 앱 기획서

다독의 흐름과 정독의 깊이를 한 화면에 합친 리딩 도구
버전 v0.4 초안  ·  작성 2026-07-08  ·  상태 방향 전환 확정, 사용자 검토 대기  ·  앱 game-hub 허브의 학습 앱

1개요

1.1 정의

이 앱은 사용자를 공부하게 만드는 퀴즈 앱이 아니라, 재미있는 글의 바다에 몰입하게 만드는 리딩 도구임. 평소에는 다독의 흐름을 방해하지 않고 시원하게 읽어 내려가고, 사용자가 원하는 순간에만 끊어 읽기 가이드·단어 수집·문법 현미경을 손끝으로 꺼내 씀. 타이핑과 강제 문제 풀이는 일절 없음.

1.2 핵심 철학

  • 흐름이 기본값 - 화면의 기본 상태는 지문 전체가 깔끔하게 놓인 읽기 화면 하나. 배너·문항·카운트다운 같은 학습 장치가 먼저 나서지 않음.
  • 깊이는 선택 - 막히는 문장, 모르는 단어, 궁금한 구조에서만 사용자가 직접 열어 봄. 열지 않으면 아무것도 나타나지 않음.
  • 수집은 자동 - 사용자가 연 것들은 도구가 알아서 기억함. 단어를 눌러 뜻을 본 순간 그 단어는 읽던 문장과 함께 단어장에 쌓임.

1.3 왜 방향을 뒤집는가

이 앱은 두 차례 시안을 거쳤고, 각 시안의 한계가 이번 방향의 근거임.

1차 시안 - 독해 사다리2차 시안 - 문법 스캔이번 안 - 하이브리드 독해
기본 흐름지문을 문장 단위로 끊고 매 단어마다 객관식 추측을 강제문장 하나를 화면에 띄우고 문법 단어 찾기를 반복지문 전체를 그냥 읽음
깊이 도구추측 보기·구조 문항·요지 문항정답 배너·구조 해부 카드끊어 읽기·단어 뜻·문법 현미경을 원할 때만
한계문제 풀이가 읽기 흐름을 계속 끊음읽는 재미가 없고 답이 뻔해짐흐름과 깊이의 충돌 자체를 제거

1.4 가치 제안

  • 학습자 - 공부한다는 부담 없이 흥미로운 글을 읽는데, 읽고 나면 단어장과 구조 이해가 남아 있음.
  • 콘텐츠 제작자 - 지문 한 편에 끊어 읽기·해설·단어를 미리 심어 두면, 도구가 알아서 학습 장치로 바꿔 줌. 문항 출제가 필요 없어 제작 부담이 낮음.
  • 허브 - 게임 모음 허브에 성격이 다른 학습 도구를 더해 방문 이유를 넓힘.

2타겟과 전략

2.1 타겟

단어와 문장 구조에서 막히지만, 문제집 스타일 앱에는 지친 성인 학습자. 시험 대비가 아니라 읽는 힘 자체를 키우려는 사람.

2.2 전략

  • 다독이 기본, 정독은 손끝에 - 읽기를 막는 요소를 화면에서 제거하고, 깊이 도구는 전부 클릭 한 번 거리에 둠.
  • 맥락 속 단어 - 단어는 항상 그 단어를 만난 문장과 함께 저장하고 복습함. 맥락 없는 단어 목록을 만들지 않음.
  • 회독으로 각인 - 같은 지문을 서너 번 다시 읽는 것을 앱이 권장하고 횟수를 세어 줌. 단어와 구조를 아는 상태의 재독이 뇌에 각인되는 순간임.

3다섯 가지 핵심 기능

3.1 몰입형 리딩

메인 화면에는 흥미로운 지문 한 편 전체가 깔끔하게 노출됨. 제목·본문·회독 버튼 외에는 아무것도 없음. 심리학 법칙, 흥미로운 스토리처럼 내용 자체가 궁금한 소재만 씀.

3.2 터치형 끊어 읽기 가이드

읽다가 해석이 막히는 문장을 클릭하면, 그 문장 바로 아래에 직독직해 덩어리들이 부드러운 애니메이션과 함께 나타남. 영어 덩어리와 우리말 해석이 쌍으로 놓여 어순 감각을 도움. 같은 문장을 다시 클릭하면 접힘.

3.3 맥락 기반 단어 자동 수집

본문 속 주요 단어를 클릭하면 뜻이 그 자리에서 노출됨. 이때 도구가 단어 + 뜻 + 방금 읽던 원문 문장을 묶어 내 단어장에 즉시 자동 저장함. 저장 버튼을 따로 누르지 않음. 단어는 항상 맥락 속에서 복습해야 하므로 원문 문장이 예문으로 함께 남는 것이 이 기능의 핵심임.

3.4 선택형 문법 현미경

모든 문장에 문법 설명을 노출하지 않음. 구조적으로 짚을 가치가 있는 문장 끝에만 작은 🔬 아이콘을 두고, 사용자가 그 아이콘을 클릭할 때만 해설이 열림. 해설은 공식 - 왜 이 구조인가 - 이렇게 쓰면 틀린 문장 - 자연스러운 해석의 네 단으로 구성해, 읽기 이해를 넘어 작문 감각까지 닿게 함.

3.5 다회독 반복

지문 끝에 회독 완료 버튼을 둠. 누를 때마다 그 지문의 회독 수가 1 올라가고 기기에 기억됨. 지문 목록에서 회독 수를 보여 주어, 단어와 구조를 인지한 상태로 같은 지문을 3~5번 다시 읽는 루프를 유도함.

4사용 흐름

  • 1 지문 고르기 - 목록에서 궁금한 제목을 고름. 각 지문에 회독 수가 붙어 있음.
  • 2 그냥 읽기 - 막히지 않으면 끝까지 읽어 내려감. 아무 장치도 개입하지 않음.
  • 3 막히면 열기 - 문장이 막히면 문장을 클릭해 끊어 읽기를, 단어가 막히면 단어를 클릭해 뜻을, 구조가 궁금하면 🔬를 클릭해 해설을 봄. 연 단어는 단어장에 자동 수집.
  • 4 회독 완료 - 끝까지 읽으면 회독 완료를 누름. 같은 지문을 바로 다시 읽거나 목록으로 돌아감.
  • 5 단어장 복습 - 단어장 화면에서 수집된 단어를 원문 문장과 함께 다시 봄.

5화면 구성과 손끝 규칙

5.1 화면 두 개

  • 읽기 화면 - 지문 목록과 본문 읽기. 앱의 기본 화면.
  • 내 단어장 화면 - 수집된 단어 목록. 단어를 누르면 뜻과 원문 예문이 펼쳐짐.

5.2 손끝 규칙

본문 위에서 클릭 대상이 세 가지라, 겹치지 않게 역할을 나눔.

누르는 곳일어나는 일
주요 단어뜻이 그 자리에 노출되고 단어장에 자동 저장
문장의 나머지 부분그 문장의 끊어 읽기 가이드가 아래에 펼쳐지거나 접힘
문장 끝 🔬그 문장의 구조 해설이 펼쳐지거나 접힘
  • 주요 단어 표시 - 클릭할 수 있는 단어는 아주 옅은 점선 밑줄로만 구분함. 몰입을 깨지 않는 선에서 발견 가능해야 함. 실물 확인 후 농도를 조정함.
  • 열림은 하나씩 - 끊어 읽기와 구조 해설은 문장마다 독립으로 열고 닫음. 다른 문장을 연다고 이미 연 것이 닫히지 않음.

6상태와 저장

  • 회독 수 - 지문별로 기기에 저장. 앱을 껐다 켜도 유지.
  • 단어장 - 단어·뜻·원문 문장·출처 지문을 묶어 기기에 저장. 같은 단어를 다시 눌러도 중복으로 쌓이지 않음.
  • 계정 없음 - 서버와 로그인이 없으므로 모든 기록은 이 기기 안에만 있음.

7콘텐츠 규격

  • 소재 - 내용 자체가 궁금한 글만 씀. 심리학 법칙, 뇌과학, 흥미로운 실화 등. 교과서식 지문 금지.
  • 분량 - 한 편 6~10문장, 성인이 2~3분에 읽는 길이.
  • 문장마다 - 직독직해 끊어 읽기 쌍을 필수로 심음. 우리말 해석은 구조가 드러나는 직독직해로 쓰되 어색한 번역투는 피함.
  • 어려운 문장에만 - 구조 해설을 심음. 가목적어 it, 명사를 뒤에서 꾸미는 to부정사, trick A into B 같은 핵심 구조가 대상. 한 지문에 2~4개가 적정.
  • 주요 단어 - 문장마다 0~3개. 그 글을 읽는 데 실제로 걸림돌이 되는 단어만 지정하고, 쉬운 단어까지 늘어놓지 않음.
  • 시작 물량 - 심리학 소재 고품질 지문 1~2편으로 시작함. 스포트라이트 효과가 첫 지문. 루프가 검증되면 늘림.

8만들 것과 안 만들 것

8.1 이번에 만드는 것

  • 읽기 화면과 내 단어장 화면.
  • 다섯 가지 핵심 기능 전부 - 몰입 리딩, 문장 클릭 끊어 읽기, 단어 클릭 수집, 🔬 구조 해설, 회독 카운터.
  • 심리학 소재 지문 1~2편.
  • 더블클릭으로 열리는 단일 파일 버전.

8.2 나중으로 미루는 것

  • 지문 확대와 소재 다양화.
  • 단어장 복습 모드 심화 - 가리고 떠올리기 같은 인출 연습.
  • 읽기 통계 - 회독 추이, 자주 연 단어 유형.

8.3 안 만드는 것

  • 타이핑 입력과 서술형 답안.
  • 강제 객관식·문제 풀이·정답 판정. 이전 두 시안의 핵심 장치였으나 이번 철학과 충돌해 폐기.
  • 서버·계정·랭킹.
  • 다크 테마 토글.
  • 자동 복습 일정 알고리즘 - 회독은 사용자가 버튼으로 직접 돌리는 수동 루프까지만.

9개발 참고

기획 본문과 분리해 꼭 필요한 개발 정책만 남김.

  • 허브 방침대로 빌드 없는 바닐라 웹으로 만들고, 리액트 같은 프레임워크를 쓰지 않음. 단일 파일 버전은 원본에서 자동 재조립함.
  • 지문 데이터는 문장 단위로 원문·끊어 읽기 목록·구조 해설·주요 단어 목록을 유기적으로 묶음. 전체 텍스트를 따로 저장하지 않고 문장 목록에서 조합함 - 같은 내용을 두 곳에 두면 어긋나기 때문.
  • 구조 해설은 공식·왜 이 구조인가·틀린 문장 대비·자연스러운 해석의 네 필드를 필수로 채움. 2차 시안의 해부 카드 데이터를 형식 그대로 이식함.
  • 회독 수·단어장은 기기 저장소에 남기고, 단어장 항목은 단어 기준으로 중복을 막음.
  • 데이터 무결성은 자동 테스트로 검증함 - 주요 단어가 원문에 실재하는지, 끊어 읽기를 이어 붙이면 원문과 일치하는지, 해설 필수 필드가 채워졌는지.
  • 검증은 실제 사용 경로를 처음부터 끝까지 클릭으로 밟아 확인함. 화면만 보고 됐다고 판단하지 않음.
